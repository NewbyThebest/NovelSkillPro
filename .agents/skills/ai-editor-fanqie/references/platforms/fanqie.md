# 番茄小说共用审稿口径

本文件只提供男女频共用的平台留存口径和榜单证据解释。数值规则以 `scoring.md` 为准，频道差异读取对应频道参考，具体题材按 `genre` 路由。移动端清楚不等于内容低幼或只能写直给爽文；复杂题材也必须让当前事件和追读理由可理解。

## 榜单证据解释

- 优先使用 `<SKILL_DIR>/scripts/fetch_fanqie_rank.py` 产生的番茄网页公开榜单结果或缓存。
- 分类热度只使用同频道、同榜型、相同 `per-category` 参数的完整全分类扫描。`scan.complete` 不为 `true` 时，不得使用 `heat_percentile` 计分。
- `heat_rank` 是分类在本次扫描中的热度名次，`category_count` 是参与比较的分类数，`heat_percentile` 越低表示相对位置越靠前。
- 单分类 TOP10 或 TOP30 只能解释头部规模、常见包装和竞争结构，不能代替全分类热度位置。
- 网页榜单中的书名、简介可能受 PUA 字体混淆影响。无法确认文字时，优先使用分类、排名、在读人数、链接和其他可读字段，不猜测乱码内容。
- 用户提供的 App 榜单或其他公开资料与脚本结果不一致时，在报告中并列说明来源和时间，不擅自宣布其中一方绝对正确。
- 第三方标签或热度接口不能作为番茄官方数据，也不能替代本技能的榜单主证据。用户明确要求时，只能作为附录参考。
- 没有完整可用扫描时，按 `<SKILL_DIR>/references/scoring.md` 将榜单热度记为 `N/A`，继续完成其余诊断，不凭印象补分。

## 男女频共用留存基准

### 1. 首屏建立阅读理由

- 第一章前部尽快出现事件、异常、关系变化、利益压力、身份变化、明确目标或有效谜团。
- 先让读者知道主角当前面对什么，再把世界观、等级、家族和关系网分散到事件中交代。
- 核心卖点可以是能力、身份、关系、职业、斗争目标、现实痛点、生存问题或悬疑异常，不强制要求金手指。

### 2. 题材承诺清楚

- 读者应能概括这本书主要提供什么持续阅读回报。
- 复杂世界观和人物关系可以存在，但前三章至少要让读者理解当前事件、关键人物和下一步期待。
- 熟悉套路不直接扣分。只有缺少具体切口、实际兑现或长期变化能力时，才降低对应组件。

### 3. 剧情产生变化

- 情绪、信息和设定必须推动选择、行动、关系或局势，不能长期停留在心理独白和背景解释。
- 前三章至少形成一次可指认的结果，例如收益、反制、证据、关系变化、身份变化、地位变化或目标升级。
- 不要求所有题材立即打脸或反击。策略性忍耐、情绪蓄力和悬念铺设可以成立，但主角或核心视角必须持续观察、判断、选择或布局。

### 4. 移动端表达清楚

- 段落、句子和信息密度应适合手机阅读，关键人物、动作和结果第一遍能看懂。
- 环境、修辞、心理和专业信息可以使用，但必须服务当前事件，不遮住行动与因果。
- 对话和人物反应要符合身份、处境和信息边界。

## 共用高风险写法

以下属于留存或兑现风险，不是平台政策红线，也不能脱离题材直接判死刑：

1. **慢开局且没有有效承诺**：前三章持续铺背景，却没有明确事件、关系变化、异常、目标或问题。
2. **抽象表达挤占剧情**：大量议论、感慨或概念解释，没有人物行动和结果。
3. **群像开局失焦**：频繁切换视角，却没有建立核心人物、核心关系或共同事件。
4. **持续受压没有主体性**：主角连续受损，却不判断、不选择、不布局，也不形成新目标。
5. **设定只存在于说明中**：核心能力、关系、职业或规则没有转化为选择、冲突和结果。
6. **频道或题材错配**：使用与作品实际承诺无关的标准，例如要求所有男频必须有系统，或要求所有女频必须有恋爱线。
