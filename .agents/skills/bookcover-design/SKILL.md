# 番茄移动端书封设计

## 边界与路径

- 将本技能目录记为 `SKILL_DIR`，包含 `.agents/` 的项目根目录记为 `WORKSPACE_ROOT`。技能参考从 `SKILL_DIR` 解析，小说材料和输出从 `WORKSPACE_ROOT` 或用户指定路径解析。
- 默认按番茄小说移动端口径设计，兼顾男频和女频。用户明确指定其他平台时，按目标平台调整，不继续套用番茄判断。
- 本技能负责封面定位、视觉方案、提示词、旧封面诊断和封面图生成，不负责重新起书名或改简介。
- 项目任务默认把方案写入 `<WORKSPACE_ROOT>/2-设定/封面设计方案.md`；生成的图片保存到 `<WORKSPACE_ROOT>/2-设定/封面/`，工具无法指定路径时记录实际输出位置。

## 任务模式

1. **探索模式**：用户没有确定视觉方向时，生成 3 个差异明确的方案，分别侧重题材识别、核心关系或新鲜感。
2. **改版模式**：用户提供现有封面或已选方向时，先诊断，再给 1 个主改方案；只有存在明显取舍时补 1 个备选，不强制凑满 3 个。
3. **提示词模式**：用户只要提示词时，使用完整中文自然语言描述画面，不调用图像生成工具。只有用户明确使用需要特殊参数的工作流时，才补充该工具要求的参数。
4. **出图模式**：用户明确要求直接生成封面图片时，按已选方向或主推方向生成带书名的成品封面；完成后检查原图和移动端缩略图，发现主体、裁切、文字或题材信号问题时修正后再交付。

## 材料准备

按以下顺序定位材料，只读取形成封面所需文件：

1. 用户明确提供的设定、简介、书名或封面图。
2. `<WORKSPACE_ROOT>/2-设定/候选书名与简介.md`。
3. `<WORKSPACE_ROOT>/2-设定/2.1_新书设定案.md`、`2.3_主要角色设定表.md` 和必要的金手指或关系设定。
4. 只有封面卖点仍不清楚时，补读 `<WORKSPACE_ROOT>/3-大纲/3.1_全书结构总纲.md` 的开篇承诺和第一阶段目标。

提取书名、频道、题材、目标读者、核心卖点、主角或关系视觉特征、情绪承诺和代表场景。作者名仅在用户要求放入封面时需要；书名未定时可使用占位标题，不因此阻塞概念设计。核心题材或卖点仍无法判断时，只询问一次。

## 设计流程

1. 完整读取 `<SKILL_DIR>/references/cover-framework.md`，根据频道和题材选择视觉主轴，不默认男频单人、女频双人，也不强行使用人物封面。
2. 先确定一个主要识别信号：主角、关系、关键道具、能力表现、危机场景或身份符号。其他元素只作辅助，避免多个卖点抢视觉中心。
3. 确定一条主风格路线，分别说明表现媒介、文化或美术方向、材质质感和镜头光影。用户未指定时按题材与读者预期选择，不堆叠互相冲突的风格标签。
4. 使用竖版 `3:4` 构图，默认设计并生成书名，按移动端小图检查书名、主体比例、明暗对比和题材可读性。
5. 不询问文字模式。只有用户明确要求“不要文字”“无字底图”或“后期排版”时，才生成无字画面并预留书名安全区。书名直出后逐字检查，出现乱码、错字或多余文字时继续修正；不得因文字生成困难自行改交无字版。
6. 用户提供旧封面时，从题材识别、主体尺度、视觉层级、书名可读性、色彩对比、裁切安全和生成瑕疵七项诊断，不只给审美评价。

## 输出与交付

- 每个设计方向包含：封面定位、视觉主体、风格与质感、构图与色彩、缩略图识别点、书名设计、绘图提示词和必要的生成约束。用户明确要求无字底图时，书名设计改为安全区说明。
- 提示词默认使用现代图像模型可理解的完整自然语言；只有用户明确使用特殊工作流时才补充专属参数。
- 不使用平台 Logo、水印、真实明星肖像或可识别的第三方版权角色；避免以“电影感、大片感、高清细节”等空词替代具体画面。
- 不承诺某个封面必然提高点击率。说明设计依据和最大风险即可。
- 对话中提供方案文件或图片路径、主推方向和不超过三条关键判断，不重复整份方案。
