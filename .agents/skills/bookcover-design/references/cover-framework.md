# 番茄移动端书封参考

## 一、构图基准

- 使用竖版 `3:4`。成图应保持同一比例，导出尺寸可用 `600×800` 或更高分辨率。
- 主要主体通常占画面高度的 `35%–65%`，确保缩小到约 `90×120` 时仍能识别。
- 书名区保持背景相对干净，避开人脸、武器尖端、关键道具和高对比边缘。
- 同一封面只保留一个视觉中心。人物通常不超过两名；群像题材也要确定主次。
- 亮度、轮廓和色彩对比服务于缩略图识别，不要依赖放大后才能看到的纹理和小道具。

## 二、书名与文字

- 默认直接生成准确书名，并说明位置、层级、颜色和字体气质；作者名只在用户要求时加入。
- 只有用户明确要求“不要文字”“无字底图”或“后期排版”时，才不生成文字，并预留书名安全区。
- 出图后逐字核对书名。错字、乱码、笔画粘连、额外文案或文字遮挡主体都视为不合格，继续修正后再交付，不自行改成无字版。

## 三、风格控制

`2D`、`3D` 和写实属于表现媒介，国风、赛博、暗黑奇幻等属于美术方向，不能当作同一层级互相替代。每个方案按以下四层选择，通常每层只保留一个主要判断：

1. **表现媒介**：二维插画、三维渲染、写实摄影感、平面海报或混合媒介。
2. **文化与美术方向**：国风、东方奇幻、现代商业、复古年代、暗黑悬疑、未来科技等。
3. **材质与完成度**：赛璐璐、厚涂、水墨晕染、工笔细节、真实皮肤与织物、雕塑或游戏角色质感等。
4. **镜头与光影**：近景或中景、低机位或平视、柔光或硬光、轮廓光、明暗反差和景深。

用户明确指定风格时优先遵循；没有指定时，根据题材、核心卖点和目标读者选择。不要同时堆叠“写实摄影、二维动漫、三维游戏、水墨国风”等互相冲突的主风格。

常用组合示例：

- **二维国风插画**：二维商业插画 + 东方国风 + 工笔或厚涂 + 清晰人物轮廓，适合古言、仙侠、历史幻想。
- **三维东方幻想**：三维角色渲染 + 东方奇幻 + 游戏角色质感 + 强轮廓光，适合高武、玄幻、系统文。
- **现代写实商业**：写实摄影感 + 现代商业 + 真实服装材质 + 清晰棚拍或城市光线，适合都市、职场、现言。
- **悬疑平面海报**：平面海报 + 暗黑悬疑 + 克制纹理 + 高反差局部光，适合悬疑、规则怪谈和灵异。
- **青春二维插画**：二维插画 + 清新现代 + 柔和厚涂或赛璐璐 + 明亮自然光，适合校园、青春和轻甜现言。

## 四、题材视觉路由

| 题材方向 | 优先信号 | 可用主体与场景 | 常见误区 |
|---|---|---|---|
| 都市、神豪、职场 | 身份、资源、阶层反差 | 人物近景、城市、办公室、豪车或关键职业物件 | 金钱符号堆满、通用西装男 |
| 高武、玄幻、仙侠 | 力量体系、身份、危机 | 人物动作、武器、能力效果、宗门或战场 | 主体过小、特效遮脸、空灵风景 |
| 末世、求生 | 生存压力、资源、灾变 | 废墟、载具、物资、异常天气或怪物轮廓 | 画面过暗、血腥直给、背景过乱 |
| 历史、朝堂、武侠 | 时代身份、权力或行动 | 官服、宫门、军阵、刀剑、文书或城池 | 影楼古装、身份不明、只剩空景 |
| 悬疑、灵异、规则怪谈 | 谜题、规则、异常物件 | 人物局部、门窗、走廊、标识、关键物件 | 全黑看不清、廉价鬼脸、直接剧透 |
| 现言、豪门、校园 | 关系位置、身份压迫、情绪距离 | 单人或双人关系构图、合同、婚戒、校园或职业场景 | 通用情侣写真、人物没有关系动作 |
| 古言、宫斗、宅斗 | 身份、婚姻、权力与关系博弈 | 古装人物、宫墙宅院、婚服、诏书、珠钗 | 服饰时代混乱、多人挤满、纯唯美空景 |
| 脑洞、系统、游戏、科幻 | 高概念机制、界面或异常规则 | 能力视觉化、弹幕、排行榜、装置、异世界反差 | UI 小字过多、概念无法一眼理解 |

频道只能辅助判断，不能替代题材证据。男频可以使用关系构图，女频也可以使用单人主体或场景主体。

## 五、提示词结构

使用完整中文自然语言，按以下顺序写，删除不适用字段：

```text
生成番茄小说移动端商业书封，竖版 3:4。
频道与题材：[频道、题材]
核心阅读承诺：[一句话卖点]
主要主体：[人物、关系、道具或场景]
人物特征：[年龄感、外貌、服装、身份和情绪；无人像时删除]
动作与关系：[动作、站位、视线、距离或冲突]
环境：[具体场景和时代信号]
表现媒介：[二维插画 / 三维渲染 / 写实摄影感 / 平面海报 / 混合媒介]
美术方向：[国风、东方奇幻、现代商业、复古年代、暗黑悬疑、未来科技等]
材质与完成度：[厚涂、赛璐璐、水墨、工笔、真实材质、游戏角色质感等]
构图：[近景或中景、主体占比、视觉中心和文字区域]
色彩与光影：[主色、对比色、光源方向和轮廓要求]
画面要求：[缩略图清晰、题材明确、边缘干净、主体不被裁切]
书名设计：准确生成书名“[书名]”，说明位置、层级、颜色和字体气质；仅当用户明确要求无字底图时，改为不生成任何文字并预留书名安全区
```

优先用正向表述说明目标画面，例如写“背景简洁、街道空旷、主体完整”，不要堆叠“不要复杂背景、不要车辆、不要裁切”等长串否定词。确有必要时，在提示词末尾补充少量硬约束：

```text
画面中不出现平台 Logo、水印、第三方版权角色或书名以外的文字；人物数量和时代服饰必须符合设定。
```

## 六、输出格式

探索模式的每个方向使用：

```markdown
## 方向一：[方向名]

**封面定位**：[题材信号 + 核心卖点]
**视觉主体**：[主体、动作或关系]
**风格与质感**：[表现媒介 + 美术方向 + 材质完成度 + 镜头光影]
**构图与色彩**：[景别、位置、主色和对比色]
**缩略图识别点**：[小图首先看见什么]
**书名设计**：[书名文字、位置、层级、颜色和字体气质；用户明确要求无字时写安全区]
**绘图提示词**：[完整中文自然语言提示词]
**生成约束**：[只列必要的硬约束]
**主要风险**：[最可能出现的问题]
```

旧封面诊断使用：

```markdown
## 诊断结论

| 检查项 | 当前表现 | 影响 | 修改动作 |
|---|---|---|---|
| 题材识别 | [表现] | [影响] | [动作] |

## 主改方案

[按探索模式的单方向格式输出。]
```
