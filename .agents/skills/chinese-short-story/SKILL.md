# 中文短篇小说创作 V1.0

中文短篇狗血爽文的完整创作流程（10–20 章，每章 800–1500 字）。专为快节奏、强情绪、多反转的爽文风格设计，源于真实社会矛盾（婆媳、重男轻女、凤凰男、职场剥削等），极致狗血展开，难受但欲罢不能。

***

## 执行任何任务前：阅读 SOLOENT.md

若存在 `SOLOENT.md`，请阅读并提取：

- **第 8.1 待办列表** — 当前待处理任务是什么？
- **第 7.1 当前状态** — 当前处于哪一步/哪一章？
- **第 7.4 待续线索** — 活跃钩子与下一章目标
- **第 3 角色索引** — 角色语言风格摘要是否可用？

若 `SOLOENT.md` 不存在，直接进入第一步。用户确认核心设定后，执行 `/init` 创建并填充 `SOLOENT.md`，再开始第二步。

***

## 使用方式

每当进行中文短篇狗血爽文创作时使用本技能，涵盖从一句话脑洞到成稿章节的完整流程。每步需用户确认后再进入下一步。

**开始新故事**：提供一句话脑洞或基础设定，技能将引导你完成第 1–4 步；第 5、6 步为可选，按需执行。

触发本流程的表述：

- 直接故事想法：「婆婆把拆迁款全给了小叔子…」「老公出轨对象竟是我闺蜜…」
- 意图表述：「我想写一篇短篇小说」「我有个故事脑洞」「用短篇工作流」
- 显式调用：「用短篇技能」「开始创作工作流」

**继续已有故事**：打开已有 `SOLOENT.md` 的项目，说「继续写」「接着上次的」— 技能读取当前状态并从正确步骤恢复。

**跳转到指定步骤**：直接说出步骤名称（如「执行第四步写第三章」「审查第五章」），技能从该步骤开始。

**提示词模板为可选**：若存在 `docs/提示词-0X-*.md` 文件则加载；否则使用内置说明与检查清单执行。

## 六步工作流

### 第一步 — 核心设定脑暴

**输入**：用户提供的一句话脑洞或基础设定
**输出**：核心设定 + 故事走向（3 个以上大情节转折）

若存在 `docs/提示词-01-核心设定脑暴.md`，加载并遵循。否则按下方输出规格与检查清单执行。

**进入第二步前的检查清单**：

- [ ] 是否有明确的社会矛盾切入点？
- [ ] 是否设计了 3 个以上大情节转折？
- [ ] 狗血程度是否足够？
- [ ] 第一人称叙事是否合理？

**等待用户确认。**

→ 确认后，执行 `/init` 初始化 `SOLOENT.md`，再进入第二步。
→ 生成文件：`概念/核心设定.md` — 完整第一步输出（脑洞、故事走向、反转地图）。

***

## /init — 初始化 SOLOENT.md

在第一步确认后、第二步开始前自动执行。

**操作**：

1. 在项目根目录创建 `SOLOENT.md`。
2. 用第一步输出填充以下字段：
   - **第 1 节 项目 DNA**：类型、核心主题、情绪承诺
   - **第 5 节 风格指南**：基调、语言风格、节奏说明（由核心设定推导）
   - **第 7.1 当前状态**：故事标题、故事时间、当前氛围
   - **第 8.1 待办列表**：勾选 `[x] 确认核心设定（第一步）`，其余待办保持未勾选
   - **第 8.2 里程碑日志**：添加 `[已完成] 第一步 — 核心设定确认：[脑洞摘要]`
3. 向用户确认：`SOLOENT.md 已初始化 — 可进入第二步。`

***

### 第二步 — 角色设计

**输入**：已确认的核心设定
**输出**：主要人物 + 角色语言风格摘要

若存在 `docs/提示词-02-角色设计.md`，加载并遵循。否则按下方输出规格与检查清单执行。

**规则**：

- 在接触大纲之前，先完成并确认角色
- 必须输出**角色语言风格摘要**：每个角色 1–2 句（说话特点 + 口头禅/惯用句式）— 粘贴到每一步第四步的提示中

**进入第三步前的检查清单**：

- [ ] 主角是否主动型？有明确任务/目标驱动剧情？
- [ ] 每个角色是否体现某种价值、欲望、恐惧或道德立场？
- [ ] 主角语言是否鲜明、有层次、会随剧情进化？
- [ ] 角色语言风格摘要是否已输出、可供第四步使用？
- [ ] 核心角色 ≤ 4 个？
- [ ] 人物关系是否能自然制造冲突？

**等待用户确认。**

→ 生成文件：每个角色一个 `角色/[姓名].md` — 完整档案（背景、价值/主题角色、外貌、语言习惯、口头禅、弧光摘要）。
→ 更新 `SOLOENT.md`：

- **第 3.1 角色索引**：添加每个角色条目（姓名、年龄、角色、指向 `角色/[姓名].md`）
- **第 3.2 关系网络**：填写关键关系
- **第 7.2 角色快照**：填写当前角色状态
- **第 8.1**：勾选角色设计待办，添加 `[ ] 完成故事大纲`
- **第 8.2**：记录 `[已完成] 第二步 — 角色确认：[姓名列表] / 语言风格摘要就绪`

***

### 第三步 — 大纲与章纲

**输入**：已确认的核心设定 + 已确认的角色 + 章节数（默认 10–20）
**输出**：三幕大纲 + 逐章章纲

若存在 `docs/提示词-03-大纲与章纲.md`，加载并遵循。否则按下方输出规格与检查清单执行。

**章纲格式**（须与第四步输入一致）：

- 情节点：每章 3–5 条
- 虐点：注明触发方式（对话 / 行动 / 发现）
- 结尾钩子：注明悬念类型（反转 / 揭秘 / 威胁 / 失联）
- 无预设时间戳过渡

**进入第四步前的检查清单**：

- [ ] 章节数是否已与用户确认？
- [ ] 第一章是否快速进入核心矛盾？
- [ ] 第一章虐点是否足够强、能锁住读者？
- [ ] 每章是否含：情节点 + 虐点（触发方式） + 结尾钩子（悬念类型）？
- [ ] 反转节点是否清晰标注？
- [ ] 虐点是否分布合理、不集中堆叠？

**等待用户确认。**

→ 生成文件：

- `大纲/全文大纲.md` — 三幕结构，含幕摘要与反转标记
- `大纲/章纲.md` — 完整逐章计划（每章情节点、虐点、结尾钩子）
  → 更新 `SOLOENT.md`：
- **第 4.1 故事结构**：填写章节范围与幕划分
- **第 4.2 大纲索引**：添加指向 `[故事大纲] -> 大纲/全文大纲.md`
- **第 8.1**：勾选大纲待办，添加 `[ ] 开始写第一章`
- **第 8.2**：记录 `[已完成] 第三步 — 大纲确认：[X] 章 / [幕结构摘要]`

***

### 第四步 — 章节创作 + 自动审核

**输入**：当前章纲 + 角色语言风格摘要；从第二章起还需粘贴前 1–2 章正文
**输出**：章节正文（800–1500 字）+ 自动审核报告

若存在 `docs/提示词-04-章节创作.md`，加载并遵循。否则按下方输出规格与检查清单执行。

**前置加载（每章创作前必须执行）**：

读取以下三个规则文件，创作与审核全程遵守：

- `.agents/rules/anti-ai-expression.md` — 表达克制层（词汇/句式黑名单、修饰语克制、对话提示语规范）
- `.agents/rules/anti-ai-rhythm.md` — 语言韵律层（动作链韵律、句式呼吸感、视角锚定、段落节奏）
- `.agents/rules/character-realism.md` — 人物真实感（动机私心、弱点瑕疵、细节习惯、情感递进）

**每章流程**：

1. 第一章：粘贴章纲 + 角色语言风格摘要
2. 第二章起：粘贴章纲 + 角色语言风格摘要 + 前 1–2 章正文
3. 运行 `python3 scripts/count_words.py 第X章.md` 获取字数 — 不以模型自估为准
4. 正文完成后立即输出自动审核

**自动审核清单**（每章完成后执行，逐项标注 ✅/❌）：

**▎基础与大纲**

- [ ] 字数 800–1500（通过 `python3 scripts/count_words.py` 验证）？
- [ ] 情节点是否按顺序执行？虐点是否按正确触发方式落实？
- [ ] 结尾钩子是否有力？无哲理总结 / 发散思考 / 感悟式收尾？
- [ ] 第一人称 POV 是否干净 — 无「他心里想 / 她感到」等越界？
- [ ] 是否无连续 3 句以上「我想/我觉得/我心里」式独白？

**▎反 AI 表达（anti-ai-expression）**

- [ ] 是否无词汇黑名单词（心理套路词、神态堆砌、形容词滥用、比喻陈词）？
- [ ] 是否无句式黑名单（并列定语顿号、「带着…某种…」、从句嵌套超两层）？
- [ ] 每个名词/动词前是否最多一个修饰语？每章比喻句是否 ≤ 3 个？
- [ ] 对话提示语是否简洁（优先「他说」「她问」，无长修饰提示语）？
- [ ] 是否无抒情说教、总结升华式收尾？章末是否以悬念/动作/未尽对话结尾？

**▎反 AI 韵律（anti-ai-rhythm）**

- [ ] 是否无动作节拍等长（连续动词短语字数结构相同）？
- [ ] 是否无三句以上等长排比？长短句是否交错？
- [ ] 关键句是否独立成行？段落是否控制在手机两三行以内（≥ 70% 两行以内）？
- [ ] 每个场景是否有明确视角锚定？是否无无源监控镜头？
- [ ] 每句对话是否独立成行？长对话是否拆分并插入动作/反应？

**▎人物真实感（character-realism）**

- [ ] 对话是否推进剧情（≥ 60%）？每个角色说话是否符合其语言风格摘要？
- [ ] 同一角色同类意思是否换了说法、无重复话术？
- [ ] 角色行为是否有合理动机？是否无「因为剧情需要」式行动？
- [ ] 是否无时间戳式场景开头（「第二天…」「一周后…」）？
- [ ] 年龄/职业/背景首次提及后是否改为侧面表达？

→ 生成文件：`章节/第[XX]章.md` — 仅正文（干净，不含审核内容）。
→ 每章完成后更新 `SOLOENT.md`：

- **第 7.1**：将「当前章节」更新为下一章编号
- **第 7.3 近期情节流**：替换为本章 3–5 个关键事件
- **第 7.4 待续线索**：更新「紧急钩子」「活跃伏笔」「下一章目标」
- **第 7.5 执行审核**：粘贴自动审核清单结果 + 字数
- **第 8.1**：勾选当前章节待办，添加下一章待办
- **第 8.2**：记录 `[已完成] 第[X]章 — [一句话情节摘要]`

> 若需深度审查指定章节，使用第五步（可选）。

***

### 第五步 — 章节审查（可选）

**输入**：用户指定的一章或多章
**输出**：详细审查报告 + 修改建议

若存在 `docs/提示词-05-章节审查.md`，加载并遵循。否则按下方检查清单执行。

**前置加载（审查前必须执行）**：

读取以下三个规则文件，审查全程以此为判定标准：

- `.agents/rules/anti-ai-expression.md` — 表达克制层
- `.agents/rules/anti-ai-rhythm.md` — 语言韵律层
- `.agents/rules/character-realism.md` — 人物真实感

**适用场景**：

- 用户对某章质量有疑虑
- 反转章需要精修
- 多章联审检查连贯性
- 跨章节风格一致性检查

**检查清单**：

**▎基础与大纲**

- [ ] 字数（800–1500）+ 大纲符合度（情节点 + 虐点）？
- [ ] 叙事逻辑连贯、文风统一、人物语言符合设定？

**▎反 AI 表达（anti-ai-expression）**

- [ ] 是否存在词汇黑名单词？逐条列出违规词及位置。
- [ ] 是否存在句式黑名单？（并列顿号、「带着…」、从句嵌套超两层）
- [ ] 修饰语是否克制（每名词/动词前 ≤ 1 个）？比喻句是否 ≤ 3 个/章？
- [ ] 对话提示语是否简洁？是否存在长修饰提示语？
- [ ] 章末是否以悬念/动作/未尽对话结尾？是否存在抒情说教或总结升华？

**▎反 AI 韵律（anti-ai-rhythm）**

- [ ] 是否存在动作节拍等长？是否存在孤立动作罗列（无视角感知绑定）？
- [ ] 是否存在三句以上等长排比？长短句交错是否合理？
- [ ] 段落长度是否达标（≥ 70% 两行以内、无大段堆砌、无单字成段）？
- [ ] 视角是否一致锚定？是否存在无源监控镜头？
- [ ] 对话排版是否规范（独立成行、长对话拆分插动作）？

**▎人物真实感（character-realism）**

- [ ] 角色行为动机是否合理？是否存在「因为剧情需要」式行动？
- [ ] 角色对话是否符合各自语言风格摘要？是否存在话术重复？
- [ ] ⚠️ 标签化重复：年龄/职业 ≤ 2 次；时间戳开头 = 0；专属细节是否过度使用？
- [ ] 角色是否展现认知局限（非全知全能）？情感递进是否自然？
- [ ] 剧情逻辑是否自洽？关键选择前是否有内心挣扎？

→ 生成文件：`审查/第[XX]章-审查.md` — 完整审查报告（清单结果、发现的问题、修改建议）。
→ 更新 `SOLOENT.md`：

- **第 7.5 执行审核**：记录审查发现与修订状态
- **第 8.1**：若适用则勾选审查待办

### 第六步 — 标题与引言（可选）

**输入**：已完成的全文章节（或核心设定 + 关键情节摘要）
**输出**：100–150 字引言 + 3 个标题选项

若存在 `docs/提示词-06-标题与引言.md`，加载并遵循。否则按下方输出规格与检查清单执行。

**仅在用户请求时执行。** 建议全部章节完成后再生成 — 从成稿中提炼冲突与悬念。

**检查清单**：

- [ ] 引言是否 100–150 字、点明核心冲突、制造悬念？
- [ ] 3 个标题是否风格各异（冲突角度 / 悬念角度 / 氛围角度）？
- [ ] 标题是否适合中文短篇平台？

→ 生成文件：`标题与引言.md` — 3 个标题选项 + 引言（100–150 字）。
→ 更新 `SOLOENT.md`：

- **第 7.1**：将「当前章节」设为「完成」
- **第 8.1**：若存在则勾选第六步待办
- **第 8.2**：记录 `[已完成] 第六步 — 标题与引言已交付 / 项目完成`

## 核心创作原则

- **钩子优先**：开篇必须制造即时张力 — 一个问题、一种威胁、一种读者无法忽视的欲望
- **情绪强度优先**：用强烈情绪驱动故事 — 欲望、恐惧、执念、悲痛、愤怒 — 而非为狗血而狗血
- **递进 stakes**：每一幕都比读者预期更进一步 — 在亲密、危险、揭秘或道德代价上
- **主动型主角**：故事引擎是主角*主动选择做什么*，而非被动承受
- **角色体现价值**：每个角色代表一种欲望、恐惧、道德立场或权力形态
- **3 个以上大反转**，分布全文
- **第一人称叙事**，强化代入
- **狗血是校准工具**：程度在第一步设定 — 根据故事需要调整
