# 打斗描写

## 使用原则

把打斗当成剧情兑现器，不要当成招式流水账。每一场战斗都要同时完成三件事：

1. 推动外部局势：胜负、积分、伤势、任务、追杀、资格、惩罚、奖励。
2. 推动人物关系：羞辱、压服、试探、藏锋、救人、结仇、被注意、被误判。
3. 制造阅读爽点：压迫、反杀、技巧、打脸、藏拙、结算、代价、悬念。

## 总流程

1. 先判断战斗类型和隐藏任务。
2. 按需读取 `references/` 中的对应模板。
3. 锁定起因、战场、规则、对手压力、主角策略和结算结果。
4. 用动作链写清交锋因果：进攻 -> 防御/闪避/硬接 -> 反击 -> 结果 -> 反应 -> 调整。
5. 在危险、命中、反转、结算处加入五感细节。
6. 结尾必须改变某种状态：名声、评分、伤势、线索、关系、敌意、后续压力。

## 按需读取指南

根据用户需求只读取相关文档，避免一次加载全部模板。

- 写不同类型的战斗：读 [fight-types.md](references/fight-types.md)。
- 主角要藏实力、控分、假败、不能赢太明显：读 [hidden-strength.md](references/hidden-strength.md)。
- 需要打得聪明、有战术、有信息差：读 [tactics.md](references/tactics.md)。
- 想增强爽点、打脸、压迫、结算、藏锋：读 [payoff-types.md](references/payoff-types.md)。

常见组合：

- **规则擂台 + 藏拙：** `fight-types.md` + `hidden-strength.md` + `payoff-types.md`。
- **追杀反杀：** `fight-types.md` + `tactics.md`。
- **越级硬战：** `fight-types.md` + `payoff-types.md`。
- **群战：** `fight-types.md` + `tactics.md`。
- **斗法/异能：** `fight-types.md`。
- **润色一段已有打斗：** 按问题补读其他模板。

## 基础动作链

优先写清因果，不要只堆华丽招式名。

```text
对手带着明确目的出手。
主角读出压力，或先承受压力。
主角格挡、闪避、硬接、换伤、诱敌，或借地形化解。
交锋造成可见、可听、可感的结果。
对手、系统、裁判、亲友、围观者或高手重新判断局势。
下一拍改变距离、节奏、风险或策略。
```

基础动作语法：

- **攻击：** 劈、斩、刺、挑、扫、砸、贯、钩、踢、肘击、肩撞、掌拍。
- **防御：** 格挡、横架、卸力、侧身、矮身、翻滚、后撤、硬接、借势、以伤换位。
- **反击：** 踹腹、击腕、砸颌、断筋、折兵、锁臂、贴身、攻盲区、借环境反杀。
- **结果：** 血线、手臂发麻、地砖开裂、枯叶倒卷、袖口撕开、耳中嗡鸣、刀锋弯折、满场死寂。

## 五感使用

五感只放在关键点，不要平均铺满。

- **危险前：** 破风声、汗毛竖起、视野一暗、腥甜味上涌。
- **命中时：** 闷响、骨裂、火辣、麻痹、血味、衣料炸裂。
- **反转时：** 突然安静、脚步错位、对手瞳孔变化、兵器哀鸣。
- **结算时：** 红光亮起、系统播报、尘土落下、血滴在地、全场沉默。

## 润色检查

### 删改前的三问（每处修改都要过）

动手改任何词句之前，必须先问：

1. **这个词/句在承载什么？** 节奏/人物气场/画面感/情绪递进/对比反差？
2. **删了，这些会不会一起没了？** 比如"慢慢转一圈"承载的是掌控感，删了等于削气场。
3. **念出声，改后版本卡不卡嘴？** 读出来不顺，就是改错了。

**铁律：功能优先于精简。承载节奏和人物的词，哪怕看着"多余"，也不删。**

### 正文检查（按顺序）

1. 开打前的矛盾是否够热，读者是否觉得必须打。
2. 战斗条件是否清楚：地形、规则、倒计时、胜负标准、不能做什么。
3. 对手是否靠动作显强，而不是旁白说强。
4. 主角是否有策略，而不是被动挨打或机械闪避。
5. 每个动作是否有后果：伤、退、乱、断、扣分、误判、暴露。
6. 节奏是否有松紧变化：短句爆发，中句判断，长句结算。
7. 结尾是否带来新的剧情压力或人物变化。

## 输出方式

- **设计打斗：** 给起因、战场、规则、双方目标、战斗节拍、反转、高潮、余波。
- **直接写正文：** 按用户要求的风格、视角和字数写成可读段落，注意移动端节奏。
- **润色原文：** 保留原剧情结果，增强清晰度、压迫感、五感、爽点和结算。
- **诊断问题：** 先指出最大问题，再给具体修法，必要时附示范改写。
