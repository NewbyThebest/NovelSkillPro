# 番茄人物活人感

这是一个只处理 `人物真实度 / 角色记忆点 / 成长逻辑 / 配角灵魂` 的技能。

它不负责整本书节奏，也不主做开篇拉新，而是专门回答：

- 这个人物为什么像工具人
- 主角为什么不让人信
- 配角为什么只有功能没有人味
- 人物成长为什么像作者口头宣布
- 为什么读者很难代入主角
- 角色团为什么记不住

## 适用场景

- 人物立不住
- 角色像纸片人
- 配角只是推动剧情的工具
- 人设有，但写出来不真
- 帮我把人物写活一点
- 为什么读者记不住我的角色

## 输入类型

- 角色设定
- 角色小传
- 角色出场章节
- 多角色对手戏
- 用户对人物问题的描述

如果用户只给一句“人物不行”，优先要求补 `角色设定 + 一段实际正文`，否则只能讲原则。

## 核心任务

1. 让角色有过去，不是一张白纸标签
2. 让角色有具体反应，不只会说设定台词
3. 让角色有变化轨迹，不靠作者口播成长
4. 让配角有独立欲望，不只是围着主角转
5. 让读者更容易代入主角处境和情绪

## 工作流

### 1. 先抓角色当前功能和当前欲望

先分清两件事：

- 这个角色在故事里的 `剧情功能`
- 这个角色此刻最想要的 `个人欲望`

如果只有剧情功能，没有个人欲望，人物很容易发空。

### 2. 用四根支柱检查活人感

#### 第一根：过去感

重点判断：

- 人物有没有被过去塑形
- 过往经历是不是只写在设定表里，没有渗进正文

修改原则：

- 不一次性端出完整身世
- 用选择、习惯、偏见、禁忌、熟练度，慢慢露出过去

#### 第二根：反应感

重点判断：

- 人物遇事时有没有具体动作、微表情、心理偏移
- 不同角色的反应是否长得一样

修改原则：

- 少用笼统结论，改成可见反应
- 同一件事，让不同角色有不同关注点和不同应对方式

#### 第三根：成长感

重点判断：

- 角色变化有没有前因后果
- 是真的被困境改造，还是作者突然宣布“他成熟了”

修改原则：

- 用 `困境 -> 挣扎 -> 改变 -> 沉淀`
- 每次变化都尽量落到一个事件、一场失败、一次代价上

#### 第四根：独立灵魂

重点判断：

- 配角离开主角后，是否仍然像一个会活的人
- 角色团里每个人是否有自己的算盘、偏执、软肋、坚持

修改原则：

- 至少给核心配角一个独立目标
- 让配角不是只负责吹主角、帮主角、恨主角

### 3. 检查人物记忆点是不是落在事件里

人物记忆点不要只写成标签词，比如：

- 冷
- 狠
- 傲
- 温柔

要改成可复现的行为模式、决策倾向、情绪触发点。

如果要做角色团，额外补一层：

- 给角色做 `反差标签`
- 但反差不能只是设定词反着贴，必须在事件里兑现

例如：

- 平时怂，关键时刻敢豁出去
- 嘴毒，但对某类人会异常心软
- 看着散漫，碰到底线立刻翻脸

### 4. 补代入感

如果用户的问题是“人物有设定，但读者不代入”，重点检查六件事：

1. 主角当下想要什么是否清楚
2. 主角痛点是否够具体
3. 主角的得失是否能被读者快速理解
4. 情绪反应是不是可感，不是空标签
5. 场景压力是不是落到了人物身上
6. 读者是否能看见人物做选择，而不只是听作者解释

代入感的底层，不是把角色写得惨，而是让读者知道他为什么急、怕、想赢、舍不得。

## 输出方式

### 诊断模式

1. `这个人物最不真的地方`
2. `问题出在过去感 / 反应感 / 成长感 / 独立灵魂 的哪几项`
3. `最该补的一个事件或一类反应`
4. `必要时给一段改写示例`
5. 如果读者代入弱，再补 `最该增强的得失点`

### 改稿模式

优先顺序：

1. 先补人物欲望
2. 再补具体反应
3. 再补成长链
4. 最后补标签和语言风味

### 生成模式

如果用户要直接生成人设，至少给出：

- 角色当前目标
- 角色隐性缺口
- 角色高压下的反应
- 角色最容易暴露过去的一种行为
- 角色与主角的独立利益冲突

## 改稿原则

- 人物不是设定表，人物是会在事件里暴露自己的人
- 配角不是功能插槽，必须有一点自己的命
- 人物成长不靠总结陈词，靠反复碰壁后的变化
- 代入感不是讲道理讲出来的，而是靠明确欲望、具体得失、真实反应撑起来的
- 所有“活人感”最终都要落回正文动作、选择和关系上

## 与其他技能协同

- 开篇人物出不来时，可先接 `fanqie-opening-hook`
- 单章里人物反应假、冲突不够时，可接 `fanqie-pacing-conflict`
- 如果问题还混着题材、视角、人称、开篇雷点，可先走 `fanqie-newbie-guardrails`
