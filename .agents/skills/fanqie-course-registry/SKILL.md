# 番茄课堂台账

这是一个专门维护 `番茄课堂文章 -> 仓库技能` 映射关系的技能。

它不直接负责写作改稿，而是负责防止重复劳动，帮助后续代理先查清：

- 这篇文章有没有整理过
- 已经整理进了哪个技能
- 是并入老技能，还是应该拆新技能
- 为什么某篇文章被暂时跳过

## 适用场景

- 查一下这篇番茄课有没有做过
- 更新番茄课堂台账
- 避免重复整理
- 帮我登记新文章和技能映射
- 这篇文章该并进哪里
- 偏题材的文章要不要拆成题材技能

## 核心工作流

### 1. 先查台账

优先读取：

- [references/ARTICLE_REGISTRY.md](references/ARTICLE_REGISTRY.md)

先确认文章当前状态：

- `已整理`
- `待拆题材技能`
- `暂不整理`
- `待评估`

### 2. 再判断文章类型

把新文章先归到四类之一：

1. `通用基础写作`
2. `长篇控盘 / 结构方法`
3. `强题材专用`
4. `运营 / 版权 / 改编 / 时效风向`

### 3. 决定处理动作

处理动作只用下面四种：

- `并入现有技能`
- `新建独立技能`
- `登记为题材技能候选`
- `登记为暂不整理`

### 4. 更新台账

每次处理完都要回写 [references/ARTICLE_REGISTRY.md](references/ARTICLE_REGISTRY.md)，至少补这几项：

- 文章标题
- 栏目
- 链接
- 当前状态
- 目标技能
- 处理理由

## 题材技能的判断规则

偏题材文章可以整理成技能，但要满足两个前提：

1. 文章方法对该题材有持续复用价值
2. 技能描述里要明确写清 `只在该类小说使用`

优先适合拆题材技能的方向：

- 仙侠 / 玄幻
- 悬疑
- 新媒体
- 男频脑洞
- 女频脑洞
- 宫斗宅斗
- 末世
- 科幻

如果一篇文章只是短期风向、标签热度、月度改编口味，不要急着做技能，先登记为 `时效观察`。

## 输出方式

当用户问“有没有做过”时：

1. 直接回答 `做过 / 没做过 / 部分吸收`
2. 给出对应技能
3. 如果没做，说明原因和下一步建议

当用户问“该怎么处理这篇”时：

1. 先给文章分类
2. 再给处理动作
3. 最后更新台账

## 改稿原则

- 先查台账，再动技能
- 不凭记忆判断是否做过
- 如果一篇文章同时影响多个技能，要记录 `主归属技能`，其他写成 `辅助影响`
- 对跳过的文章，也要留下跳过原因

## 什么时候读取额外参考

- 需要确认当前已整理范围时，读取 [references/ARTICLE_REGISTRY.md](references/ARTICLE_REGISTRY.md)
