# 番茄长篇大纲 (Outline)

## Usage
触发指令：`加载大纲`、`写大纲`、`大纲阶段`、`番茄大纲`、`番茄长篇大纲`

## 目标
将设定落地为适合番茄长篇阅读节奏的章节施工图：全书总纲 → 单卷卷纲 → 标准化章纲（含场景拆解）；重点控制前 200 字首屏、前三章承诺兑现、5-10 章短循环、20-30 章阶段高潮、30-50 章留存爆点与章尾追读；支持动态修订。

---

## 初始化（最先执行）

1. 在**项目根目录**创建目录 **`3-大纲`**（若不存在）。

2. 进入大纲阶段前，建议先打开 `预期.md`，确认大纲方向与你的初始预期（题材边界、核心剧情执念、番茄读者情绪、投流卖点等）是否一致，再开始规划。

---

## 📋 设定回溯时机与范围

> **设定回溯（总原则）**：撰写大纲时若发现与既有设定不一致或需要细化，须回到设定部分更新对应文件，再继续或落稿。设定服务于剧情，不是一成不变。

| 时机 | 建议检视并可能更新的内容 |
|------|------------------------------|
| **总纲完成后** | **设定**：与总纲冲突或需对齐的设定（主线/地图/换图逻辑等），**以 `SOLOENT.md`指针与已读设定为准** |
| **每卷卷纲完成后** | **大纲**：对照 **`3-大纲/3.1_全书结构总纲.md`**，若本卷卷纲导致全书结构/分卷/锚点需调整，则修订总纲。**设定**：与本卷剧情、人设、金手指相关的设定文件（路径见「路径说明」） |
| **章纲每批次后**（如每 10–15 章） | **大纲**：对照 **本卷 `3-大纲/XX卷_完整卷纲.md`**，若本批次章纲导致卷级战略/事件流/节奏需调整，则修订卷纲（必要时再联动总纲）。**设定**：与本批次剧情变动相关的设定文件（同上） |

## 📚 步骤 1: 全书总纲规划 (Master Outline)

### 输入


1. **项目根目录下所有与设定相关的文件**  
   扫描并读取与创作设定、边界、预期相关的 `.md`（含根目录散落文件，以及若存在的约定子目录如 `2-设定/`、`1-边界/` 等；**不假设**固定文件夹名，以仓库实际为准）。
2. **和/或** 项目根 **`SOLOENT.md`**（若存在）  
   通读其**正文**，并依据其中 **索引/指针**（如 `->` 路径、各 `§` 登记的链接）**逐条打开并读取**被引用的设定、边界、大纲类文件。

> **步骤 1 写作侧重**：在以上输入中，优先消化「世界观 / 主线推演 / 地图与换图 / 节奏与分卷」等信息；若有样板书拆解，则**重点对标**其中的 **全书框架** 类材料（常见文件名如 `1-边界/1.4_全书框架.md`，**以实际路径为准**）。缺关键材料时须提示用户或确认后再生成总纲。

### 参考标准

> **🎯 强制对标原则**：新书的总纲必须在格式、结构、节奏密度上严格对标 `1.4 全书框架.md`。（如果有）

**包含的核心要素**：
1. **全书剧情单元总览**：按【地图 + 身份/阶级 + 核心目标】划分阶段
2. **核心节奏公式**：样板书的标准循环公式（如：发现危机→试错→重开→布局→收网）
3. **换地图逻辑**：何时换地图、为什么换（战力崩坏/资源枯竭/主动扩张）
4. **节奏密度统计表**：样板书各卷的字数占比、章节范围、核心事件密度
5. **番茄留存节点**：首屏承诺、前三章兑现、5-10 章短循环、20-30 章阶段高潮、前 30-50 章追读爆点、可包装投流素材

### 执行方式
**交互式共创（Iterative Dialogue）** - 这是一场持续的深度对话

> **⚠️ 重要**：
> - 不要一次性生成完整总纲。必须逐项与用户讨论、确认后再继续。
> - 每一轮提议时，必须明确说明「如何对标」**已识别的全书框架类材料**（若有）；若无，则说明如何与已读设定/预期对齐。

### 动作流程

#### 前置步骤：加载全量输入与提炼对标参考
- **AI 动作 1（加载输入）**：严格执行本步骤的【输入】要求，扫描并读取项目下所有设定文件（或经 `SOLOENT.md` 索引加载），确保世界观、人设、主线等设定信息已在上下文中。**禁止跳过此步直接写大纲。**
- **AI 动作 2（提炼对标）**：在已读材料中定位「全书框架/样板节奏」类内容（如 `1-边界/1.4_全书框架.md`，如有）。向用户摘要展示样板书的总卷数、核心主题、节奏密度统计表、换地图逻辑。
- **若无样板书框架**：向用户说明总纲将从零构建，仅与已读设定、预期自洽，并确认是否可接受。

#### 第1轮：分卷数量与字数规划
- **AI 提议**：根据全书目标字数（询问用户）提出分卷方案，并对标说明
- **等待用户确认** → 确认后进入下一轮

#### 第2轮：各卷核心主题与节奏对标
- **AI 提议**：定义每一卷的核心主题，强制对标样板书节奏分布
- **等待用户确认** → 确认后进入下一轮

#### 第3轮：大事件锚点分配与换地图逻辑
- **AI 提议**：确立全书超级高潮点，分配到具体卷中，并说明换地图逻辑、追读钩子和情绪回报
- **等待用户确认** → 确认后进入下一轮

#### 第4轮：核心节奏公式提炼
- **AI 提议**：根据**前置步骤已识别的全书框架类材料**（若有）提炼新书的标准循环公式；若无样板书框架，则根据已读设定与分卷方案归纳可复用节奏模板
- **等待用户确认** → 确认后进入下一轮

#### 第5轮：番茄留存与创作原则检查
- **AI 检视**：检查每一卷的核心高潮是否符合项目宪法，并检查首屏承诺、短循环回报、阶段爽点和投流素材是否成立
- **等待用户确认** → 确认后生成文件

### 输出
- `3-大纲/3.1_全书结构总纲.md`

#### 输出格式要求

**必须包含以下五大板块**（若有样板书「全书框架」类材料则**格式与密度上对标**；否则按板块含义填满，并与已读设定一致）：

1. **全书剧情单元总览**：表格形式（章节区间 | 地图 | 身份/阶级 | 核心目标 | 故事梗概）
2. **核心节奏公式**：新书的标准循环公式与可复用节奏模板
3. **节奏密度统计表**：各卷字数占比、章节范围、核心事件密度
4. **番茄留存设计**：首屏承诺、前三章兑现、5-10 章短循环、20-30 章阶段高潮、前 30-50 章追读爆点、投流素材
5. **对新书的启示**：开局节奏、中期布局、后期格局

> **CRITICAL**: 必须将总纲注册到 `SOLOENT.md §4.2 Outline Index`：
> `[总纲] -> 3-大纲/3.1_全书结构总纲.md`

### 用户确认
- **【强制暂停】** 输出以下提示后，**禁止继续执行任何后续步骤，等待用户明确回复**：
  > 「已生成 `3-大纲/3.1_全书结构总纲.md`，请打开文件仔细阅读，确认全书结构与节奏符合预期后回复『继续』。」
- 收到用户『继续』（或等效肯定）后，方可进入步骤 2。

### 设定回溯（可选）
总纲落定后，检视**新书设定案**（见上文「路径说明」）中的主线推演、地图路线图、换图逻辑是否与总纲一致；若有细化或修正，则更新后再进入步骤 2（卷纲）。

## 📖 步骤 2: 单卷完整卷纲规划 (Volume Outline)

### 目标
将战略规划（卷纲）与战术落点（细纲）合二为一，形成既有宏观指导又有微观执行的作战地图。

### 触发时机
- 新书开坑
- 或上一卷正文即将写完时

### 参考依据
- `3-大纲/3.1_全书结构总纲.md`
- `1-边界/1.4_全书框架.md`（如有）
- **新书设定案**（**重点参考核心创作指导**；路径见「路径说明」）
- `1-边界/1.5_微观节奏拆解.md`（如有）

### 执行方式
**交互式共创（Iterative Dialogue）** - 这是一场持续的深度对话

> **⚠️ 重要**：不要一次性生成完整卷纲。必须逐层与用户讨论、确认后再继续。

### 动作流程

#### 前置步骤：加载参考依据
- **AI 动作**：在提议卷纲前，必须先读取并消化 `3-大纲/3.1_全书结构总纲.md`、**新书设定案**（核心创作指导）以及 `1-边界/1.4_全书框架.md`（如有）。**禁止未读取总纲与设定就直接生成卷纲。**

#### 第1轮：战略层 - 核心设计
- **AI 提议**：本卷预估字数、核心矛盾（明暗双线）、爽点节奏曲线、每 5-10 章的短回报
- **等待用户确认** → 确认后进入下一轮

#### 第2轮：战略层 - 人物链
- **AI 提议**：本卷的 BOSS（经验包）、盟友（渠道）、工具人及功能定位
- **等待用户确认** → 确认后进入下一轮

#### 第3轮：战术层 - 剧情事件流
- **AI 提议**：按阶段（如每 10-20 章）规划详细剧情流，并标出章尾钩子、阶段爽点兑现和追读问题
- **等待用户确认** → 确认后进入下一轮

#### 第4轮：战术层 - 金手指植入
- **AI 提议**：明确每个关键剧情节点主角如何利用金手指破局，精确到词条功能与代价
- **等待用户确认** → 确认后进入下一轮

#### 第5轮：质检与创作原则
- **AI 检视**：对照项目宪法确保符合项目专属创作指导
- **等待用户确认** → 确认后生成文件

### 输出
- `3-大纲/XX卷_完整卷纲.md`

> **CRITICAL**: 必须将卷纲注册到 `SOLOENT.md §4.2 Outline Index`：
> `[第XX卷卷纲] -> 3-大纲/XX卷_完整卷纲.md`

### 用户确认
- **【强制暂停】** 输出以下提示后，**禁止继续执行任何后续步骤，等待用户明确回复**：
  > 「已生成 `3-大纲/XX卷_完整卷纲.md`，请打开文件仔细阅读，确认本卷卷纲无误后回复『继续』。」
- 收到用户『继续』（或等效肯定）后，方可进行设定回溯并进入步骤 3。

### 设定回溯
本卷卷纲确认后，须检视并必要时更新**实际项目中的**对应设定文件（路径见「路径说明」），通常涉及：
- **新书设定案**（关键事件、地图、关系博弈等与本卷一致）
- **金手指设定**（本卷金手指植入是否需补丁或新功能）
- **主要角色设定表**（本卷新配角、主角成长或关系变化）

（旧流程约定文件名常为 `2.2` / `2.3` / `2.4` 等，**仅作参考**；无 `2-设定/` 时以 SOLOENT 与用户文件为准。）

更新完毕并经用户确认后，再进行步骤 3（章纲）。

## 🔨 步骤 3: 标准化章纲细化 (Chapter Specify)

### 目标
将卷纲拆解为具体的章节施工图，包含场景拆解和写法指导。

### 参考依据
- `3-大纲/XX卷_完整卷纲.md` - 卷纲
- `1-边界/1.5_微观节奏拆解.md` - 场景与写法原子级对标（如有）

### 执行方式
**分批交互式细化** - 不要一次性完成全卷章纲

> **⚠️ 重要**：按批次进行（每次 5-10 章），每批次完成后必须与用户确认，确认无误后再继续下一批。

### 动作流程

#### 前置步骤：加载参考依据
- **AI 动作**：在提议章纲前，必须先读取本卷的 `3-大纲/XX卷_完整卷纲.md` 以及 `1-边界/1.5_微观节奏拆解.md`（如有）。**禁止未读取卷纲就直接生成章纲。**

#### 步骤 1: 结构对标与场景拆解（每批次重复）

**约束**：**单章字数锚定 2300-3300 字**，每章必须有明确推进、情绪波动和章尾追读点；开篇章纲必须标注前 200 字首屏抓力。

**第1轮：AI 提议本批次章节与场景拆解**
- **章节范围**：明确本批次涵盖第 X 章到第 Y 章
- **结构对标**：参考样板书的分章节奏
- **完整章纲**：为本批次每一章提供完整章纲（章节梗概、本章番茄目标、场景拆解、爽点/虐点、伏笔与线索、结尾钩子）
- **场景拆解要求**：**尽可能多地拆解场景**，不要把一章写成两三个大场景

**等待用户确认本批次** → 确认后进入步骤 2

#### 步骤 2: 设定回溯与动态修正 (Feedback Loop)

**触发时机**：每完成一批（如 10-15 章）章纲后，批量执行一次。

**AI 检视**（以下设定文件若有与章纲不一致或需细化的地方，须更新后经用户确认再继续；**具体路径见「路径说明」**）：
- **设定案检视**：本批次剧情是否导致**新书设定案**需要补充或修正？
- **金手指检视**：现有金手指机制能否支撑本卷爽点？若不能，需修改**金手指设定**对应文件。
- **人设检视**：本卷是否需要新配角？主角性格是否有成长？若有，需更新**角色设定表**对应文件。

若发现本批次章纲与原定卷纲/总纲严重冲突，立即跳转至 **步骤 5: 动态修订**。

**等待用户确认** → 确认后继续下一批章纲（回到步骤 1），或完成本卷章纲。

**用户确认（最终产出）**：
- **【强制暂停】** 输出以下提示后，**禁止继续执行任何后续步骤，等待用户明确回复**：
  > 「本卷章纲已全部完成，已生成 `3-大纲/XX卷_章纲.md`，请打开文件仔细阅读，确认全卷章纲无误后回复『继续』。」
- 收到用户『继续』（或等效肯定）后，注册到 SOLOENT.md 并结束本步骤。

### 输出
- `3-大纲/XX卷_章纲.md`

#### 输出格式示例

```markdown
第X章：[章节标题]

**章节梗概**：[一句话概括]

**本章番茄目标**：[首屏问题/冲突升级/爽点兑现/追读钩子]

**场景拆解**：
- 场景1：[发生了什么] + [写法指导：情绪基调、细节要求]
- 场景2：[发生了什么] + [写法指导：情绪基调、细节要求]
- 场景3：[发生了什么] + [写法指导：情绪基调、细节要求]
- 场景N：...

**伏笔与线索**：
- 埋入：[新伏笔内容]（预计第Y章收回）
- 推进：[推进的已有线索]
- 收回：[收回的伏笔]（呼应第Z章）

**结尾钩子**：[悬念点]
```

> **CRITICAL**: 必须将章纲注册到 `SOLOENT.md §4.2 Outline Index`：
> `[第XX卷章纲] -> 3-大纲/XX卷_章纲.md`

## 🔄 步骤 4: 动态修订 (Dynamic Revision)

> **场景**：正文写作中涌现新灵感、剧情自然偏离原定大纲，或发现原大纲逻辑不通时使用。

### 目标
保持"大纲-正文"的一致性，确保后续剧情在新的逻辑上继续展开，防止崩书。

### 触发时机
- **每一卷章纲创作完成时**：自动回溯检视
- **正文创作里程碑**（每卷完结或每 10 章节点）：自动检视正文走向
- **用户主动指令**："修订大纲"、"剧情偏了"

### 执行流程

#### 情况 A：正文跑偏，回溯大纲 (Bottom-Up)
1. 读取最新完成的正文章节、当前卷的 `XX卷_完整卷纲.md`
2. AI 总结"原定走向"与"实际走向"的差异点
3. 推演蝴蝶效应（对本卷结局及后续卷的影响）
4. 执行修订：更新章纲 → 更新卷纲 → 必要时更新总纲

#### 情况 B：总纲调整，重做细纲 (Top-Down)
1. 读取 `3.1_全书结构总纲.md` 和下一卷的旧大纲（如有）
2. 按"步骤 2：单卷完整卷纲规划"重新生成下一卷卷纲
3. 确认后覆盖旧文件，并更新 SOLOENT 索引

### 输出
- 更新后的 `3-大纲/XX卷_章纲.md`
- 更新后的 `3-大纲/XX卷_完整卷纲.md`
- 更新后的 `3-大纲/3.1_全书结构总纲.md`

### 用户确认
- **【强制暂停】** 输出以下提示后，**禁止继续执行任何后续步骤，等待用户明确回复**：
  > 「大纲修订已完成，请打开对应文件确认修订内容无误后回复『继续』。」
- 收到用户『继续』（或等效肯定）后，方可继续创作。

## ⏭️ 下一步（灵活选择）

大纲的三个模块（总纲、卷纲、章纲）**可独立调用、按需生成**。
根据进度选择：

- ✅ **渐进式**：写完第一卷章纲 → 加载番茄长篇“正文”技能包写第一卷正文 → 写完后回来写第二卷卷纲
- ✅ **批量规划式**：先写完所有卷的卷纲和章纲 → 再加载番茄长篇“正文”技能包开始正文创作
- ✅ **快速启动式**：只写第一卷的卷纲和前几章章纲 → 立即加载番茄长篇“正文”技能包 → 边写边补充后续章纲

**推荐**：大部分作者采用「渐进式」，可以根据写作进度动态调整大纲。
