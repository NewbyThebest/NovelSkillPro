# 番茄长篇控盘

这是一个专门处理 `十万字以上长篇续航` 的技能。

它重点回答：

- 十万字后为什么开始难写
- 百万长篇为什么容易疲软
- 中段为什么像在原地踏步
- 怎么安排阶段目标、阶段高潮、阶段结算

## 适用场景

- 百万长篇怎么写
- 十万字后开始卡文
- 中段容易崩
- 怎么稳住长篇节奏
- 追更读者为什么会掉
- 怎么防注水和重复感

## 输入类型

- 整书主线
- 卷纲
- 分阶段大纲
- 当前字数和卡点说明
- 连续数章正文

如果用户只说“长篇写不住”，优先要求补 `主线一句话 + 目前写到哪 + 最近一个卡点`。

## 核心任务

1. 拆出阶段目标，别让长篇变成一条平线
2. 设计阶段高潮和阶段结算
3. 轮换主线、支线、升级线，避免单线疲劳
4. 处理十万字后常见卡顿和百万字常见失速

## 工作流

### 1. 先判断现在卡在哪一层

优先区分四种问题：

- `主线太虚`：整本书不知道长期在追什么
- `阶段太平`：最近一段没有清晰目标和阻碍
- `回报太少`：读者追了很多字，却吃不到回报
- `重复太重`：每段都像上一段换皮

### 2. 把长篇拆成阶段

默认不要把百万字当一整块写。

至少拆成：

1. 开书拉新段
2. 第一阶段立主线段
3. 中段扩张段
4. 中后段变局段
5. 大后期总清算段

每个阶段都要有：

- 阶段目标
- 阶段敌人或阻碍
- 阶段回报
- 阶段结算
- 下一阶段的新期待

### 3. 处理十万字卡文

十万字左右常见问题：

- 初始设定写完了，不知道接下来写什么
- 角色关系没有继续变化
- 爽点模式开始重复

优先补法：

1. 给主角换一个更高层目标
2. 给旧关系加新变量
3. 给原有爽点换兑现方式，不只重复原模板

### 4. 处理百万字疲软

百万长篇常见风险：

- 只会叠事件，不会做阶段结算
- 主线和支线抢戏
- 升级、打脸、闯关机械重复
- 阶段高潮之后没有新局面

修改原则：

- 每个大高潮后都必须出现局势变化
- 不让支线长期脱离主线价值
- 让主角的目标、代价、对手跟着体量升级

### 5. 轮换读者回报

长篇不要只靠一种回报活着。

常用回报轮换：

- 升级回报
- 打脸回报
- 关系推进回报
- 真相推进回报
- 地位变化回报

如果连续很多章都只有一种回报，疲劳感会很快上来。

## 输出方式

### 诊断模式

1. `当前长篇最大失速点`
2. `问题属于主线 / 阶段 / 回报 / 重复 的哪一类`
3. `最该补的阶段目标或阶段高潮`
4. `下一轮续航方案`

### 规划模式

1. `整本书主线一句话`
2. `阶段拆分`
3. `每阶段的目标 / 阻碍 / 回报 / 结算`
4. `中段防疲劳设计`

### 修复模式

如果用户已经写到中段或后段发虚，优先输出：

1. `哪里开始掉劲`
2. `哪几段在重复`
3. `该砍的重复模块`
4. `该补的新变量、新敌人、新回报`

## 常用公式

### 公式一：阶段推进

`阶段目标 -> 阶段阻碍 -> 阶段高潮 -> 阶段结算 -> 新阶段期待`

### 公式二：十万字破卡

`旧目标完成或失效 -> 新变量进入 -> 关系重排 -> 新目标抬升`

### 公式三：防注水轮换

`主线推进 -> 小回报 -> 支线反扑 -> 大回报 -> 局势换档`

## 改稿原则

- 长篇不是把短篇爽点无限重复
- 阶段结算和新期待同样重要
- 任何一段如果既不推进主线、也不制造新变量、也不给回报，就该警惕注水
- 百万字控盘的关键，不是写得更多，而是让每一段都知道自己在服务什么

## 与其他技能协同

- 开书前 3 到 5 章抓力问题，可接 `fanqie-opening-hook`
- 单章太平、高潮无余波、过渡章发虚，可接 `fanqie-pacing-conflict`
- 如果还混着新手通病、题材包装、人称视角问题，可先接 `fanqie-newbie-guardrails`
