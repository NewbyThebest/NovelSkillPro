# 番茄开篇钩子

这是一个专门面向 `开篇设计 / 前三章重做 / 第一卷起手搭建` 的技能。

它不负责泛泛判断“有没有问题”，而是直接回答：

- 这个故事应该从哪里切入
- 开篇第一刀该砍在哪
- 怎么让读者先被吸住
- 前三章怎么排，钩子怎么留

## 适用场景

- 帮我设计开篇
- 这个开头不抓人
- 我不知道该从哪一幕开始写
- 前三章怎么排更容易留人
- 按番茄方法给我重做开篇
- 这个设定怎么切成读者愿意看的第一章
- 第一次打开第一页为什么留不住人

## 输入类型

- 故事设定
- 主线一句话概括
- 第一卷卷纲
- 现有开篇正文
- 仅有一个核心梗

如果用户完全没有主线，只给了零散设定，先帮他压出一句话主线和第一卷目标，再做开篇。

## 核心任务

1. 选择切入点
2. 主动引爆剧情
3. 设计好奇升级链
4. 控制前五章角色数量
5. 给出前三章节奏排布
6. 保证开篇每句话尽量有功能
7. 保证第一页先把读者拽住

## 工作流

### 1. 先判断用哪种切入

主要有两种：

- `剧情切入`
- `场景切入`

默认原则：

- 新手优先 `剧情切入`
- 文笔、氛围掌控、场景调度很强时，才考虑 `场景切入`

如果用户没有明确要求，默认按 `剧情切入` 设计。

### 2. 先压缩主线与第一卷目标

在做开篇前，先明确：

1. 整本书主线一句话是什么
2. 第一卷主角最迫切的事是什么
3. 主角当前身份、处境、行动动机是什么

如果这三件事不清楚，开篇再花也容易空转。

### 3. 找最适合引爆的切口

不是找“最早发生的事”，而是找“最适合第一时间炸开的事”。

优先选择：

- 当前危机
- 关系冲突
- 人性抉择
- 反差场面
- 异常事件

少用：

- 大段背景介绍
- 过远的前史
- 纯设定展示
- 大而空的生死危机

额外硬要求：

- 优先做到 `首页见山`
- 第一屏就让读者知道：有人要出事、主角有麻烦、局面有反差，至少占一个
- 如果最有吸引力的场面在后面，优先考虑前置，而不是老老实实按时间线排队

### 4. 设计好奇升级链

开篇不只是“有一个钩子”，而是要有一串升级动作：

`小异常 -> 好奇 -> 更怪的细节 -> 引爆 -> 留坑`

让读者不是只被震一下，而是被一步步拽进去。

拉新向开篇额外检查三件事：

1. `能不能点进来`
2. `能不能读下去`
3. `能不能顺手点下一章`

如果三件事里只完成了一件，这个开篇就还没搭稳。

### 5. 控制角色与信息量

开篇前五章内：

- 不出现没有意义的人
- 每个角色出场都要有作用
- 主角要有清晰能动性
- 支线、设定、人物数量都要克制

### 6. 输出前三章结构

必要时直接给出：

- 第一章钩子
- 第二章伏笔升级
- 第三章或第三至四章的第一次高潮 / 爽点 / 新坑

同时遵循几个硬要求：

- 尽量从第一个字开始下钩子
- 三段话以内，必须有重点
- 300 字内，尽量进入主题
- 第一页优先抛痛点、异常、反差或风险，不把准备动作放在最前面
- 任何一句话都优先承担信息、情绪、悬念或推进功能

如果是强拉新场景，可直接按下面顺序压：

1. 先抛异常或当前麻烦
2. 再补最少量必要背景
3. 迅速把矛盾升级
4. 章末留一个比开头更大的问号

## 输出方式

### 诊断模式

适用于用户已经写了开篇。

输出顺序：

1. `这个开头最大的问题是什么`
2. `切入点选错了，还是引爆不够`
3. `第一页有没有真正抓住人`
4. `前三章当前心理路径是否成立`
5. `重做建议`
6. `给一版更合适的切入点`

### 设计模式

适用于用户只有设定或只有一个梗。

输出顺序：

1. `主线一句话`
2. `推荐切入方式`
3. `第一章开头怎么炸`
4. `前三章排布`
5. `每章结尾钩子`

### 重构模式

如果原开篇已经写散，直接输出：

1. `原开篇为什么不抓人`
2. `新的切入点`
3. `新的前三章推进链`
4. `一个开头样段`

## 常用公式

### 公式一：剧情切入

`主角当前麻烦 -> 细节异常 -> 危机升级 -> 留下更大的疑问`

### 公式二：关系冲突切入

`熟悉关系 -> 矛盾抛出 -> 必须选择 -> 代价显现`

### 公式三：反差锚点切入

`正常预期 -> 反常行为 -> 读者错愕 -> 追问为什么`

## 改稿原则

- 开篇先抓人，再慢慢给设定
- 比起“前三章都不错”，番茄拉新更看重第一页有没有抓手
- 冲突优先小而准，不优先大而空
- 主角必须有能动性，不能像被剧情拖着走
- 读者前三章最想知道的是“接下来怎么炸”，不是“作者还藏了多少背景”
- 对开篇废话零容忍，只要拖节奏、削吸引，就删

## 与其他技能协同

- 如果要先判断开篇有没有普遍性问题，可先走 `fanqie-newbie-guardrails`
- 如果已经确认开篇问题集中在钩子、切入、前三章布局，转到本技能
- 如果开篇已经有了，但单章内部节奏、冲突、爽点还弱，可接 `fanqie-pacing-conflict`
- 如果需要严格式评分审稿，可接 `golden-three-chapters-review`
