# 番茄节奏冲突爽点检查清单

来源：番茄课堂《节奏混乱？冲突乏力？爽点不足？你的问题，本篇为你解决！》

## 1. 节奏

### 重点判断

- 有没有迟迟不进主线
- 有没有无效情节占比过高
- 场景、情节切换是否突兀
- 规则、能力、设定是否前后打架

### 命中雷点的典型信号

- 该推进主线时却在写琐事
- 上一段还在 A 地，下一段毫无过渡到了 B 地
- 世界规则前面讲一套，后面用一套

### 修改原则

- 先删无效内容，再谈补细节
- 每章尽量有明确目标，不要漫游
- 转场要给行动、心理或事件上的衔接
- 关键设定整理成清单，写关键情节时回查

## 2. 冲突

### 重点判断

- 主角前进时有没有真实阻碍
- 高潮前有没有足够铺垫
- 高潮后有没有持续影响

### 命中雷点的典型信号

- 主角一路顺风顺水
- 最终爆点像突然从天上掉下来
- 高潮结束后像什么都没发生

### 修改原则

- 给目标加敌人、代价、时间压力、误判、身份掣肘
- 高潮前 3 到 5 章埋线索、抬情绪、加压迫
- 高潮后 1 到 2 章补变化：主线推进、人设变化、关系变化、格局变化

## 3. 爽点

### 重点判断

- 爽点类型是否单一
- 爽点是否缺少铺垫
- 爽点是否拖太久
- 爽点后是否没有反馈
- 看点是否稀薄，读者是否长时间吃不到东西

### 命中雷点的典型信号

- 全书只会打脸
- 爽点一来就赢，没有压制过程
- 读者等了几章还没兑现回报
- 爽完没人震惊、没人付代价、主线也没动

### 修改原则

- 爽点要多样：打脸、升级、逆袭、智斗、情感满足都可以用
- 爽点前用 `激烈矛盾 + 递进铺垫 + 细节压迫`
- 爽点尽量密集化：`小冲突 -> 快解决 -> 强反馈`
- 每 3 到 5 章至少要有一个小回报点
- 拉新向章节可额外参考：`300字1个小看点，500字1个小钩子，1章1个悬念`

## 快速改稿顺序

1. 先找本章目标
2. 删掉与目标无关的段落
3. 给目标加阻碍
4. 提前爽点或增强爽点落地
5. 补高潮后果
6. 最后修语言
