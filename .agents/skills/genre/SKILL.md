# 番茄题材写作

处理**具体题材**的写作方法：这个题材写什么、怎么开头、爽点从哪来、人设金手指怎么做、新人容易踩什么坑。

只管题材写法。跨题材的通用技法（开篇钩子、单章节奏冲突、人物活人感、长篇结构）不在这里处理，别在本技能里越界去改。

## 怎么用这个技能

1. **先定题材**：从下面路由表认出用户写的是哪个题材，读对应的 `references/<题材>.md`。题材有明确子类型时，再读该子类型参考；例如都市高武先读 `urban.md`，再读 `urban-high-martial.md`。
2. **题材不明 / 跨题材**：先帮用户判断核心卖点更像哪个题材（路由表的"易混淆"列给了岔路），定了再读。一本书可能沾两个题材，但**核心追读预期只归一个**，按那个走；另一个当辅助参考。
3. **材料不足时**：题材选型或新书构思只需补核心卖点、目标读者或现有设定；诊断“写不好”的具体问题时，才要求提供题材和一段正文或设定。缺关键材料只问一次，不假装能判断具体短板。

## 每个 reference 的统一结构

13 个主题材及其子类型 reference 都按同一套结构写，读的时候对应着看：

1. **边界与易混淆**：本题材范围，和哪些题材易混、该转哪
2. **流派/分类选型**：题材内部分几路，先定路再动笔
3. **核心爽点/抓手**：这个题材的爽从哪来（一句话公式）
4. **开篇切口**：怎么起手、前几章给什么
5. **人设与金手指**：主角底盘、金手指怎么做
6. **常见误区**：这个题材新人最容易踩的坑
7. **题材特有约束**（有才写）：如悬疑的合规与线索逻辑、历史的史料与边界、科幻的科技树自洽

## 给设定流程的路由结果

当 `webnovel-ideation` 调用本技能时，读完主参考和适用的子类型参考后，返回：

1. 核心题材参考与已读取的辅助参考。
2. 会影响当前项目的题材特有约束。

`webnovel-ideation` 只消费这份路由结果，不自行增加、改写或猜测题材维度。

## 通用输出方式

所有题材通用，读完对应 reference 后按这个出：

**用户给的是设定：**
1. 判断题材和流派
2. 定主角底盘和第一阶段目标
3. 定开篇切口
4. 指出最该补的一处（金手指创新 / 关系链 / 爽点兑现，看题材）

**用户给的是正文：**
1. 判断更像哪一路
2. 找开篇是否拖、是否迟迟不进入卖点
3. 找核心线（升级线 / 资源线 / 冲突线 / 情感线，看题材）是否成立
4. 给改法或样段

**用户要选题材：**
1. 比较最接近的 1-3 个题材及其读者预期
2. 推荐一个核心题材，说明辅助题材只承担什么作用
3. 指出当前卖点最需要补的题材信号

**用户只问一个具体问题：**
只回答对应部分，例如开篇、金手指、关系线或爽点，不强制输出完整题材方案。

## 通用心法（所有题材都成立）

- **先定边界，再动笔**：写之前先认清题材和流派，别一边想写传统扎实、一边又用短平快脑洞节奏，两头不靠。
- **爽点先于剧情**：先想清这个题材的爽从哪来、怎么持续兑现，再铺世界和设定。
- **开篇别先讲世界**：先让读者看到主角的处境和第一个钩子，设定名词往后放。
- **核心差异化抓手要创新**：有金手指就优先创新其玩法、限制和兑现；没有金手指时，创新关系结构、职业/资源规则、谜题或题材特有机制。差异化要能在正文持续兑现，不能只停在设定名词。
