# 番茄主角设计

设计适合番茄小说/短故事的主角。重点不是复杂人设，而是读者第一章能不能代入，前三章能不能相信主角会持续制造爽点、情绪和反转。

## 适用场景

- “帮我设计一个主角/男主/女主”
- “主角不够有魅力”
- “主角动机太弱”
- “读者代入不进去”
- “主角太被动”
- “按番茄风格做人设”
- “生成角色文档并写入 SOLOENT”

## 番茄主角核心

主角必须具备 6 个可见要素：

1. **直给欲望**：主角想要什么，一句话说清。
2. **强压困境**：谁/什么正在压主角，代价是什么。
3. **身份反差**：废物其实是天才、底层藏资源、弱者有底牌、好人被逼黑化等。
4. **首章动作**：主角第一章必须做一件主动的事，哪怕只是拒绝、取证、试探、逃跑或反击一句。
5. **爽点接口**：主角凭什么持续带来爽感，金手指、能力、资源、人脉、智商、身份或关系都可以。
6. **情绪钩子**：读者为什么替他生气、心疼、期待或想看他打脸。

## 主角设计公式

```text
我是 [身份/处境]，
我想 [具体目标]，
但 [压迫/敌人/规则] 正在逼我 [损失/屈服/死亡/失去尊严]，
我表面 [弱点/低位/误解]，
实际 [底牌/反差/金手指]，
所以我会用 [行动方式] 一步步拿回 [成就/爽点/关系主动权]。
```

## 题材校准

- **都市/神豪/战神**：主角要有低位开场 + 身份/财富/能力反差 + 当场打脸能力。
- **高武/玄幻/仙侠**：主角要有升级目标 + 明确强敌 + 可视化金手指/天赋反馈。
- **末世**：主角要有求生目标 + 物资/安全区/队伍领导力 + 冷硬选择。
- **悬疑/灵异**：主角要有调查动力 + 认知优势/特殊感知 + 被规则追杀的压力。
- **现言/古言/世情短篇**：主角要有关系压迫 + 私人伤口 + 反击证据/身份反转/离开能力。
- **历史/朝堂/权谋**：主角要有身份位置 + 权力目标 + 破局手段，不要只“隐忍观察”。

## 优化弱主角

如果用户提供现有人设，先诊断：

- **欲望虚**：目标抽象，如“变强”“活下去”“追求自由”，但没有具体抓手。
- **困境轻**：压迫不痛，读者不急。
- **反差弱**：主角没有可传播标签。
- **太被动**：前三章只挨打、解释、旁观。
- **爽点接口断**：不知道主角以后凭什么赢。
- **情绪不聚焦**：读者不知道该心疼、愤怒还是期待。

然后给出：

```markdown
### 问题诊断
[逐条指出]

### 番茄化改造
[重写六要素]

### 首章登场改法
[给出 300-600 字登场片段或节拍]
```

## SOLOENT.md 指针联动

当用户要求生成角色文档或当前项目需要落盘时：

1. 在合适目录创建角色文档，如 `角色/[角色名].md` 或用户指定路径。
2. 文档包含：一句话人设、六要素、前三章表现、长线弧光、语言行为标签、关系线。
3. 在文档末尾添加返回指针：

```markdown
> [!note] 实体归属
> 本角色已注册于全局实体库：[[SOLOENT.md]]
```

4. 若根目录存在 `SOLOENT.md`，在角色索引或主角区块新增 Markdown 链接：

```markdown
- **[角色名](角色/角色名.md)**：一句话人设摘要。
```

5. 若 `SOLOENT.md` 不存在，询问是否创建；用户授权或工作流要求创建时再创建。
