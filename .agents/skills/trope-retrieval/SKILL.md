# 番茄套路检索与改编

从本地套路库检索可用素材，并改编成适合番茄小说/短故事的剧情方案。重点不是“列套路”，而是把套路落到首屏、单章爆点、短循环结算和章尾钩子里。

## 适用场景

- “卡文了，不知道接下来写什么”
- “给我几个打脸/装逼/反转套路”
- “这个开局怎么更番茄”
- “帮我找适合这个题材的爽点”
- “这里需要一个关系爆点/金手指桥段”
- “推荐几个能接下一章的钩子”

## 前置读取

执行前：

1. 若存在 `SOLOENT.md`，读取题材、主角、人设、当前章节、待续线索。
2. 明确用户当前需要的是：
   - 开局钩子
   - 爽点结算
   - 反转揭秘
   - 关系爆点
   - 金手指兑现
   - 章尾钩子
   - 长线新事件
3. 检索 `docs/tropes_library.md`，不要整篇读入。优先用 `rg` 搜关键词。

## 检索策略

优先搜索这些库区和关键词：

- **开局类情节**：穿越开局、重生开局、赘婿开局、退婚流、危机开局、继承遗产/债务开局
- **成长/逆袭类**：废柴崛起、逆袭、扮猪吃虎、打脸、升级流、莫欺少年穷
- **桥段级爽点**：打脸、救场登场、最后一分钟营救、掉马甲、隐藏身份、剧情反转、多重反转
- **中文网文特有**：系统流、签到流、马甲流、直播流、赘婿文、规则类怪谈、神豪主角
- **关系情感类**：契约婚姻、先婚后爱、追妻火葬场、破防、团宠、团欺、盟友背叛
- **题材专用**：末世、宫斗、宅斗、朝堂斗争、无限流、任务系统、学院流、比武大会

避免优先推荐太泛的结构母题，如“三幕式结构”“英雄之旅”，除非用户明确问全书结构。

## 筛选标准

每次从库中筛出 3-5 个套路，按番茄适配度排序。

筛选时判断：

- **题材是否匹配**：不要给古言硬塞系统面板，除非用户本来就是脑洞古言。
- **能否首屏使用**：能不能前 200 字就让读者感到冲突或好奇。
- **能否快速兑现**：1-3 章内是否能产生爽点、反转或关系变化。
- **是否有持续产能**：后续能不能反复制造打脸、升级、修罗场、揭秘。
- **是否有投流画面**：能不能形成一句话文案或高光场面。

## 改编规则

绝对不要复制素材库原文。输出时必须改成用户故事里的：

- 具体人物
- 具体场景
- 具体利益
- 具体冲突动作
- 具体章尾钩子

套路必须服务主角目标，不要为了套路而套路。

## 输出格式

```markdown
基于当前作品的设定（[题材/主角/当前矛盾]），我从套路库里筛了 [3-5] 个番茄适配方向。

## 推荐 1：[套路名]

- **番茄适配度**：[高/中/低]，[一句话理由]
- **底层爽点**：[读者获得什么情绪回报]
- **适用位置**：[首章开局/第 X 章小高潮/章尾钩子/阶段高潮]
- **在本作中的改编**：[结合用户角色和世界观具体落地]
- **首屏用法**：[前 200 字怎么打出来]
- **章尾钩子**：[本章最后一句/最后一个事件怎么接]
- **风险**：[老套/不合题材/兑现太慢/需要铺垫]

## 推荐 2：[套路名]
...

## 主推方案

我建议优先用：[套路名]

理由：
1. [最匹配当前矛盾]
2. [最快兑现爽点]
3. [最容易接下一章]

## 可直接使用的章纲
第 X 章：[标题]
- 开场钩子：
- 冲突升级：
- 爽点/反转：
- 主角动作：
- 章尾钩子：
```

## 按需求快速映射

如果用户没给关键词，可以按问题类型主动检索：

- **开头不抓人**：退婚流、危机开局、赘婿开局、重生开局、隐藏身份
- **主角不爽**：扮猪吃虎、打脸、掉马甲、救场登场、神豪主角
- **中段疲软**：任务系统、限时任务、盟友背叛、身份暴露危机、多重反转
- **关系线没爆点**：契约婚姻、先婚后爱、追妻火葬场、破防、团欺/团宠
- **章尾没钩子**：悬念/断在关键处、剧情反转、红鲱鱼、掉马甲、死亡 FLAG
- **题材要新鲜感**：规则类怪谈、直播流、马甲流、迪化流、系统窗口
