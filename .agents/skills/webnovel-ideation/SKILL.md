# 创意与设定

## 执行流程

先读取 `1-预期/预期.md`。文件不存在时停止，说明缺少方向锚点。

## 步骤 1：新书设定案

先执行 `genre` 识别题材，读取对应参考。逐项讨论，一次聊一项，每项深入聊透再确认：

- **主角处境与目标**：主角当前的处境、目标和不能失去的对象
- **开篇承诺与读者留存**：读者最先看到什么，为什么继续看
- **力量体系（根据题材按需）**：能力来源、境界、战力构成等
- **核心矛盾与冲突**：最持续的冲突是什么
- **资源与组织规则**：物资、组织、利益分配规则
- **核心关系与势力**：势力层面的合作、竞争、对立，个人关系留到角色设定表

全部确认后按模板写入 `2-设定/2.2_新书设定案.md`：

```markdown
# 新书设定案

## 主角处境与目标

## 开篇承诺与读者留存

## 力量体系（按需）

## 核心矛盾与冲突

## 资源与组织规则

## 核心关系与势力
```

完成后进入下一步。

## 步骤 2：核心机制或金手指（按需）

先从预期和设定案判断是否有独立金手指。没有则创建 `2-设定/2.3_金手指设定.md`，注明"无独立金手指"及核心驱动。有则逐项讨论，一次聊一项，每项深入聊透再确认：

- **本质与来源**：它是什么，主角为何拥有
- **可见规则**：触发条件、作用范围、已知效果
- **限制与风险**：成本、条件、副作用、暴露风险
- **演变或揭示路线（按需）**：成长、升级或逐层揭谜

题材已有公共力量体系时，金手指只写相对公共体系的特殊部分。全部确认后按模板写入 `2-设定/2.3_金手指设定.md`：

```markdown
# 金手指或核心机制设定

## 状态

## 核心驱动

## 规则与限制（按需）

## 演变或揭示路线（按需）

## 剧情影响与不可违背的边界
```

## 步骤 3：主要角色设定

先读取 `character-vitality`，按角色层级区分核心角色和功能配角。一次聊一个角色，每个角色逐项确认后再进入下一个：

- **基本信息**：年龄、外貌
- **身世与过去**：出身、关键经历、塑造他的事件
- **可识别特征与风格**：能力、性格基调、说话风格、行为习惯
- **写作红线**：角色不能被写成什么

所有角色确认后，再讨论核心关系。功能配角只记录场景功能、当下得失和差异化反应。全部确认后按模板写入 `2-设定/2.4_主要角色设定表.md`：

```markdown
## 主角

- **写作红线**：

## 核心配角

### [角色名]

- **基本信息**：
- **身世与过去**：
- **可识别特征与风格**：
- **写作红线**：

## 核心反派

### [角色名]

- **基本信息**：
- **身世与过去**：
- **可识别特征与风格**：
- **写作红线**：

## 功能配角

- 角色名、场景功能、当下得失、差异化反应

## 核心关系

- 合作、依赖、竞争、对立

## 全局写作红线
```

## 步骤 4：种子文风文档

先询问用户有没有可以参考的小说素材，直接复制粘贴到窗口发送过来，素材越多越好，最好能有 3 章以上正文。没有完整素材时，可提供一段自己认为不错的段落。都没有则从零开始讨论，给出风格选项供选择（如简洁快速型、细腻画面型、轻松吐槽型等）。

有素材时，先按素材风格预填各维度，再逐项询问用户是否可行。无素材时逐项讨论，一次聊一项，每项深入聊透再确认：

- **用词与句段节奏**：用词偏好、句子长短、段落节奏
- **场景、动作与信息释放**：场景描写密度、动作表达方式、信息释放节奏
- **对白与人物语言差异**：核心人物的说话风格见角色设定表，文风只记录整体对白原则
- **情绪基调与爽点表达**：整体情绪氛围、爽点的表达方式
- **本书特有的避免项**：不用的表达方式、不出现的套路

全部确认后按模板写入 `2-设定/2.5_文风.md`，写入时只保留维度标题和内容，不保留模板标记和举例：

```markdown
# 文风

## 用词与句段节奏

## 场景、动作与信息释放

## 对白与人物语言差异

## 情绪基调与爽点表达

## 本书特有的避免项
```

**下一步建议**：完成后，执行 `webnovel-outline` 写大纲。
