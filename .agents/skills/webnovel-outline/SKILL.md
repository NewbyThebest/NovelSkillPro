# 长篇大纲

## 目标与边界

生成 `3-大纲/3.1_全书结构总纲.md`，让后续正文能判断当前阶段该推进什么、该给读者什么回报，以及什么不能写偏。

总纲采用滚动规划：终点方向与持续追读引擎负责守住全书承诺，每次只规划一个当前阶段。未来阶段不提前列出，等当前阶段由定稿正文实际收束后再补下一阶段。

总纲不是卷纲或逐章施工图。当前阶段内部的短期推进由正文阶段的十章章纲和单章章纲即时生成。

## 初始化、输入与续写

1. 确保项目根目录存在 `3-大纲/`，只创建缺失目录，不覆盖文件。
2. 完整读取 `1-预期/1.1_预期.md`、`2-设定/2.1_新书设定案.md`、`2-设定/2.2_金手指设定.md`、`2-设定/2.3_主要角色设定表.md`。核心关系文件存在时一并读取 `2-设定/2.3.2_核心关系.md`；当前或近端阶段涉及已有阶段角色时，再读取 `2-设定/2.3.1_阶段与功能角色记录.md` 的相关条目。缺少会影响主线判断的文件时，说明缺口并先补足，不凭空推演。
3. 已有 `3.1_全书结构总纲.md` 时，先读取当前阶段。当前阶段尚未完成时，只调整当前阶段和入口承诺，不预写下一阶段。
4. 当前阶段已经达到完成标志时，再读取 `控制台/进度.md`、相关十章章纲和阶段收束部分的定稿正文，先用正文事实总结实际结果。用户确认阶段收束后，将其压缩为已完成阶段摘要，再共创新的当前阶段。
5. 已完成阶段摘要只记录正文已经发生的主要结果，不保留当初未实现的预测，也不扩写成分卷回顾。

## 共创原则

- 每轮提议都要说明怎样对齐预期、设定和角色利益；待定项明确标记，不能用假设填满大纲。
- 追读承诺和阶段回报的有效形式由本书决定，不套固定爽点公式。
- 每一轮确认一组会改变全书方向的决策，不机械要求逐条回复"继续"。
- 终点方向不等于阶段路线。除当前阶段外，不为未来人物、势力、入口、敌人顺序和关系变化安排具体剧情。

## 第 1 轮：终点方向与持续追读引擎（首次创建）

先提出并确认两项：

1. **终点方向**：故事最终要达成的状态、关系、身份、真相、目标或代价，不强制写成"主角变强"。
2. **持续追读引擎**：概括本书反复如何产生"目标或问题 -> 阻碍与选择 -> 结果或代价 -> 新目标或新问题"的因果循环。它可以由成长、关系、破局、案件、事业、求生、身份揭露等驱动，不规定单章或 5-10 章的固定公式。

确认后进入下一轮。滚动补充后续阶段时默认保留这两项，只有定稿事实或用户方向发生实质变化才重新讨论。

## 第 2 轮：当前阶段

只规划当前正在开始或即将开始的一个阶段，记录：

- **起点处境**：主角和关键关系处于什么位置。
- **核心问题或矛盾**：这一阶段最需要解决、承受或看清什么。
- **阶段目标与完成标志**：阶段结束时必须发生什么可确认的变化，用于定稿阶段判断何时滚动更新。

当前阶段内部若有多个小单元，留给正文阶段按十章章纲滚动处理，不在总纲中提前拆成后续阶段。确认后进入下一轮。

## 第 3 轮：当前阶段入口承诺

确认阶段入口承诺：第一阶段明确首屏与前三章要让读者看到什么；后续阶段明确进入新阶段后最先兑现或提出的问题。只定目标，不拆逐章场景，也不预测阶段内部的关键进展与后续问题。阶段内部路线由正文流程按定稿事实滚动生成十章章纲。

确认后生成总纲。

## 输出结构

```markdown
# 全书结构总纲

## 终点方向与核心读者承诺

## 持续追读引擎

## 已完成阶段摘要（没有则省略）

### 阶段[序号]：[名称]
- 实际结果：

## 当前阶段

### 阶段[序号]：[名称]
- 起点处境：
- 核心问题或矛盾：
- 阶段目标与完成标志：

## 当前阶段入口承诺
```

写入后等待用户确认。后续正文阶段以当前阶段确定方向，再生成十章章纲和单章章纲。不得因为总纲没有未来阶段便自行补写。

## 阶段滚动更新

`webnovel-dinggao` 判断当前阶段已经收束后，重新执行本技能：

1. 以定稿正文和控制台状态说明阶段完成标志实际如何达成，以及有哪些原计划未发生。
2. 用户确认后，将实际结果写入已完成阶段摘要。
3. 根据此时的角色状态、关系、线索、资源和终点方向，只共创下一个当前阶段。
4. 重新确认该阶段的入口承诺，然后回到正文流程。

不能由上一个阶段的旧计划直接推导下一阶段，也不能在定稿回写时自动生成未经用户确认的新阶段。

## 设定冲突处理

若大纲推演发现设定、角色利益或核心机制无法支撑已确认方向，先说明"现有设定 vs 拟定大纲"的差异与影响。未经用户确认，不直接改写 `2-设定/` 文件，也不靠新增设定掩盖矛盾。

**下一步建议**：首次总纲完成后，执行 `webnovel-title-intro` 生成书名简介；滚动补完下一阶段后，回到 `webnovel-zhengwen` 继续正文。
