# 短篇方向大纲

## 输入与目标

读取预期文档、必要设定和用户指定篇幅。方向大纲只确定短篇从开局承诺走到结算的因果和情绪路径，不拆逐章场景。

## 设计要点

1. **开局承诺**：第一章让读者看见的具体压力、异常、关系问题或目标。
2. **核心问题**：主角必须解决、承受或看清什么；不解决会失去什么。
3. **变化链**：至少安排一次由人物选择、证据或规则推动的局面变化。反转数量由故事长度和题材决定，不为凑数安排。
4. **结算**：核心问题如何得到回应，主角、关系或局面留下什么结果与余波。
5. **线索与边界**：哪些信息要先露出、哪些不能临场新增，避免结局靠巧合。

## 输出结构

```markdown
# 短篇方向大纲

## 篇幅与核心读者承诺

## 开局承诺与核心问题

## 主要变化与因果链

## 结算与情绪余波

## 线索与伏笔
```

确认方向后写入 `3-大纲/短篇方向大纲.md`。逐章章纲留给正文阶段即时生成。
