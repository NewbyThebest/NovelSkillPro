# 正文润色

## 目标与边界

在不改变已确认的剧情事实、人物关系、视角、伏笔方向和设定边界的前提下，让正文更清楚、准确、顺畅并贴合本书文风。

- 上位规则是项目根目录 `AGENTS.md`；排版、去 AI 味和人物真实性与其冲突时，以该文件为准。
- 本技能处理表达和段落功能，不替用户擅改剧情。发现需要调整章纲、设定或已定稿事实的问题时，指出影响并交由相应流程决定。
- 将本技能目录记为 `SKILL_DIR`，包含 `.agents/` 的项目根目录记为 `WORKSPACE_ROOT`。比较页脚本从 `<SKILL_DIR>/scripts/` 解析，输出到 `<WORKSPACE_ROOT>/.temp/`。

## 任务模式

先识别用户所需力度，不默认生成三份全文：

| 模式 | 触发 | 输出 |
| :--- | :--- | :--- |
| 简洁 | 用户要精炼、压字数、去重复 | 一份收紧版 |
| 调整 | 用户说润色、优化、改顺 | 一份常规精修版，默认模式 |
| 强化 | 用户要重写、补足场面或明显增强 | 一份结构性加强版，先说明会调整的层面 |
| 对比 | 用户明确要三档、多个版本或想比较取舍 | 简洁、调整、强化三版，均从原文独立生成 |
| 诊断 | 用户只问哪里不好、怎么改 | 问题和修改方案，不重写全文 |

## 执行流程

### 1. 读取上下文

读取用户提供的段落或草稿。项目写作中，按需补读本章章纲、`2-设定/2.4_文风.md`、最近定稿和相关角色状态；涉及主要角色时读取 `2-设定/2.3_主要角色设定表.md` 的对应条目，涉及核心关系时读取 `2-设定/2.3.2_核心关系.md` 的对应条目，涉及阶段或功能角色时读取 `2-设定/2.3.1_阶段与功能角色记录.md` 的对应条目；缺少会影响事实判断的材料时只问一次。

先确定原文段落承担什么功能：推进剧情、转场、冲突、人物、关系、伏笔、情绪落点、情绪回报或信息交代。只补强它实际承担的功能。

### 2. 判断该改什么

按原文需要检查以下方面，不逐项硬改：

1. **信息与视角**：主语、动作对象、结果和人物可知信息是否清楚。
2. **动作与因果**：关键动作是否有起因、过程或结果；转折是否跳步。
3. **人物与关系**：判断依据、情绪反应、语言和选择是否符合人物处境，不用额外悲惨经历或内心独白硬补。
4. **场景与线索**：物件、空间、环境或异常细节是否真正提供信息；不为画面感堆气味、五感和比喻。
5. **语言与节奏**：删重复解释、压缩短语、虚词、套话和无效身体反应；保留角色专属腔调与必要留白。

需要增强紧张感时，优先写准确的动作、代价和局面变化，不机械添加副词、感官或冲突。需要比喻时，只有比直写更清楚才保留。

### 3. 按模式生成

- **简洁版**：删除重复、弱修饰和拖慢推进的解释。原文本已精炼时少动，不为“更短”删掉事实、情绪落点或伏笔。
- **调整版**：保留原文结构和事实，修正词语、句段、动作链、判断依据和必要的段尾落点。默认像作者自己精修后的版本。
- **强化版**：在不改剧情结论的前提下，重排镜头或段落节奏，补足必要动作链、冲突后果、判断链或关系反应；不堆辞藻、不机械拉长。
- **对比模式**：三版均从原始原文独立生成，力度须有真实区别。没有适合某一档的改法时明确说明，不为凑齐差异虚构内容。

新增内容不得改变人物决定、能力设定、关键道具、关系结论、谜底或后续伏笔。遇到这些需求时停在诊断和建议层。

### 4. 输出与自查

单版模式默认输出：原文功能判断、最多三条关键修改说明和润色后的正文。用户只要成稿时，只交付润色正文。

对比模式先检查各版本是否符合对应力度、是否守住相同事实边界，再进入比较页输出。不要以“改动数量”判断质量；无效改动应撤回。

## 三档比较页（仅对比模式或用户明确要求时）

在 `WORKSPACE_ROOT` 下，以实际绝对路径执行：

```bash
python "<SKILL_DIR>/scripts/render_compare_html.py" --stdin --output "<WORKSPACE_ROOT>/.temp/润色结果预览页.html"
```

将占位符替换为实际路径。用标准输入传入内存中组织的 JSON，不创建临时 JSON 文件。数据包含 `source`、`function`、三个 `versions` 和 `recommendation`；三个版本名称固定为“简洁版”“调整版”“强化版”。

脚本不可用时说明原因，并在聊天中提供三版正文及推荐理由；不要手写或伪造 HTML 预览页。

比较完成后提供预览页的实际绝对路径、推荐版本和不超过三条理由。用户明确要求时再在聊天中完整展示多版正文。
