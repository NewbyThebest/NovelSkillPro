# character-design

协助用户创建或更新详尽的角色设定文件，确保角色弧光与核心故事目标一致，并同步更新角色索引。

## 步骤

### 第一步：读取上下文

1. 读取 `SOLOENT.md（第 1 节：项目基因）` 以了解核心概念和故事目标。
2. 读取 `SOLOENT.md（第 3 节：角色索引与关系）` 以对齐现有角色的成长弧线。

---

### 第二步：信息收集与确认

主动向用户询问缺失的关键角色细节：
- 姓名与年龄
- 外貌特征
- 性格特点
- 背景与过往历史
- 动机与目标
- 角色弧光（起点 → 转折 → 终点）
- 与其他角色的关系
- CP 路线（仅限主角）
- 系统或超能力设定
- 其他相关细节

---

### 第三步：生成角色档案

创建一份全面的角色档案。确保关键信息根据故事类型有所侧重（例如：升级流注重战力等级和成长潜力；言情向注重情感弧线和关系动态）。使用清晰的 Markdown 标题进行逻辑排版。

**输出**：将档案保存至 `characters/` 目录，文件名为：`{character-name}.md`。

---

### 第四步：更新 SOLOENT 总控面板索引

1. 读取 `SOLOENT.md（第 3 节：角色索引与关系）`。
2. 在相应部分（主角或配角）为新角色添加条目：

```markdown
### [角色姓名] (characters/xxx.md)
- 角色定位：[主角/反派/配角]
- 核心特征：[3-5个关键特征]
- 当前状态：未出场 / [当前章节状态]
- 角色弧光：[起点] → [转折] → [终点]
```

3. 如有必要，更新角色关系部分。
4. 保存更新后的索引文件。
