# 三层大纲设计

通过"全书大纲 → 分卷大纲 → 章节大纲"的三层结构，明确创作边界，构建完整的推演链条，确保长篇连载不崩盘、不烂尾。

## 步骤

### 第一步：全书大纲

**输入**：作者的初步脑洞或核心设定。

**处理逻辑**：
- **确定本体**：明确题材、类型、流派。用 10 句话以内直白描述创作背景，或用一句话概括全篇的核心情绪底色。
- **推演链条**：明确主角的"初始匮乏"（开局缺什么）和"终极目标"（最终对抗什么/成为什么）。
- **刚性原则**：全书大纲一旦确定，即为"刚性穹顶"，后续所有设计不得跨维度扩张或偏离核心认知。

**输出格式**：
```
【全书大纲】
- 题材/类型/流派：_____
- 核心情绪底色：_____
- 主角及初始匮乏：_____
- 终极目标与对抗：_____
- 升级路径规划（预计分为几个大剧情块）：_____
```

输出后暂停，等待用户确认。确认后此框架不可再动。

---

### 第二步：分卷大纲

**输入**：第一步确认的全书大纲 + 当前卷的字数预期（建议 12-16 万字）。

**处理逻辑**：
- **叙事诉求**：明确这一卷主角必须达到的阶段性成果。
- **核心阻碍**：设计阻止主角完成目标的障碍（人、事、环境）。
- **破局方法**：推演主角如何利用金手指或智慧翻越障碍。
- **情感锚点**：设计本卷的核心反派或配角，作为读者的情感投射点。设计"遗憾"或"情感遗物"作为情绪加速器。

**输出格式**：
```
【第 X 卷大纲】（预计 XX 万字）
- 叙事诉求（本卷目标）：_____
- 核心阻碍（反派/困境）：_____
- 破局方法（金手指应用）：_____
- 情感锚点（遗憾/遗物设计）：_____
- 预期剧情节点（3-5个大高潮）：_____
```

输出后暂停，等待用户确认或修改。卷纲允许预判性微调。

---

### 第三步：章节大纲

**输入**：第二步确认的分卷大纲 + 当前剧情节点。

**处理逻辑**：
- **极简概括**：必须能用 1-3 句话极简概括这一章要写什么。如果做不到，说明这章"水"了。
- **有效信息点**：提炼 2-3 个确保读者能 get 到的有效信息点（如：暴露性格缺陷、埋下悬念、为战斗做情绪铺垫）。
- **伏笔管理**：记录本章埋下的伏笔，并规划在几章内揭开（最简单的伏笔：上一章写，下一章揭）。

**输出格式**：
```
【第 X 章章纲】
- 核心概括（1-3句话）：_____
- 核心叙事价值（选其一：暴露缺陷/铺设悬念/战斗铺垫）：_____
- 有效信息点（读者必须get到的点）：_____
- 伏笔与回收：_____
```

按批次（如 5-10 章）输出，等待用户确认。章纲在写作中可灵活调整。

---

## 大纲自检清单

每一层大纲输出后，必须依次核查以下五条：

1. ❌ 是否偏离了全书的"刚性穹顶"？（如：凡人流突然变成了无敌流，科幻突然加入了修仙）
2. ❌ 是否出现了"机械降神"？（高位实体是否过度干预了低维角色的剧情，导致战力崩塌？）
3. ✅ 主角的初始匮乏是否得到了阶段性的满足？（分卷大纲是否解决了主角当前的痛点？）
4. ✅ 章纲的叙事价值是否明确？（这章删掉是否影响主线？如果没影响，必须重写章纲）
5. ✅ 情感连接是否断裂？（换地图时，是否带上了上一卷的"情感遗物"？）
