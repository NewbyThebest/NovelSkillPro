# 番茄长篇综合诊断

## 范围与规则来源

- 将本技能目录记为 `SKILL_DIR`，包含 `.agents/` 的项目根目录记为 `WORKSPACE_ROOT`。reference 和脚本从 `SKILL_DIR` 解析；稿件、缓存和报告从 `WORKSPACE_ROOT` 解析。脚本以 `WORKSPACE_ROOT` 为工作目录，路径占位符必须替换为实际绝对路径。
- `<SKILL_DIR>/references/scoring.md` 是分值、权重、公式、复核和评级的唯一来源。平台参考提供共用口径，频道和题材参考只补充证据标准，报告模板只定义输出结构。
- 本技能只处理“大纲 + 完整前三章”的番茄长篇综合诊断。材料不足时按下文路由，短故事不使用本技能。
- 按“用户明确说明 > 作品已有信息 > 文本读者承诺”确定 `CHANNEL=male/female`。仍无法判断时只询问一次。读取对应频道参考；需要题材校准时，再按 `genre` 的题材路由读取对应参考。

## 编辑标准

- 依据稿件证据、榜单数据和题材承诺判断，不因作者身份、投入程度或个人偏好改变稿件分。
- 表达直接、具体、尊重；指出问题时说明位置、影响和修改动作，不辱骂、不强制称呼。
- 区分事实、推断和缺失信息。商业潜力是概率判断，不承诺上榜、赚钱或必然失败。

## 材料准备

1. 按“用户指定路径 > `<WORKSPACE_ROOT>/3-大纲/` 与 `4-正文/` > 限定候选搜索”定位稿件；`2-设定/` 只补充书名、简介和频道信息。
2. 限定搜索使用 `rg --files`，排除 `.agents/`、`.claude/`、`.codebuddy/`、`.git/`、`AI 责编/` 以及可识别的缓存、备份、旧稿、报告和临时目录。只按“大纲”“第一章”“chapter01”等明确语义筛选，不把纯数字文件名当作证据，也不批量读取所有候选。
3. 完整诊断必须有可判断长线主线的大纲和完整第一至第三章。不设字数门槛，但在报告中记录实际字符数；章节缺失、截断或只有摘要时不得生成综合报告。
4. 材料不足时直接路由：仅首章、前两章或前三章使用 `golden-three-chapters-review`；仅正文使用 `webnovel-review`；短故事改用相应短故事审查或工作流。
5. 只有多个稿件组合无法消歧、频道无法判断、文件为空或不可读时才询问。

## 执行流程

### 1. 加载依据与证据

- 完整读取 `<SKILL_DIR>/references/scoring.md`、`<SKILL_DIR>/references/platforms/fanqie.md`、当前频道参考和必要的题材参考。
- 按 `scoring.md` 建立统一证据索引，记录稿件位置、榜单字段和材料缺失；每个组件必须引用证据编号或缺失原因。

### 2. 获取平台信号

1. 先判断目标分类；名称或 ID 不确定时才运行榜单脚本的 `--list-categories`。
2. 运行一次 `--scan --gender <CHANNEL> --rank-type read --per-category 10`。只有 `scan.complete=true` 时才使用热度字段计分。
3. TOP10 和作品 URL 只作解释。仅在用户要求深度赛道分析或扫描信息不足时，额外运行 `--category <分类> --limit 30`。
4. 只有用户明确要求实时数据时才使用 `--refresh`，同一查询最多强制刷新一次。缓存、旧数据兜底、置信度和 `N/A` 全部按 `scoring.md` 处理。

### 3. 评分与报告

- 严格按 `scoring.md` 计算全部组件、五维总分、高分复核和评级。同一问题只影响最直接组件，不增加印象分、补偿分或隐藏扣分。
- 按 `<SKILL_DIR>/references/report_template.md` 生成 `<WORKSPACE_ROOT>/AI 责编/[书名]_番茄诊断报告.md`。
- 运行 `python "<SKILL_DIR>/scripts/validate_report.py" "<WORKSPACE_ROOT>/AI 责编/[书名]_番茄诊断报告.md"`。修正全部 `ERROR` 后再交付；长度 warning 不阻塞，但需确认符合用户要求。

### 4. 交付

- 对话中只提供报告路径、评审范围、最终总分和不超过三条最高优先级问题，不重复整份报告或追加新的评分理由。
- 报告交付后默认结束任务。用户明确要求时再进入修改方案、选题孵化或下一轮审稿；不主动询问无关个人信息或推荐外部服务。
