# 番茄长篇综合诊断评分规则

本文件是 `ai-editor-fanqie` 的唯一计分来源。男频与女频使用同一套分值结构，再由频道参考和题材参考解释什么证据算高分。

## 一、总原则

- 五个维度各自满分 100 分，所有组件分数相加得到该维度分。
- 禁止在组件总分之外继续叠加“加分”“扣分”“补偿分”或“作者资历分”。
- 同一问题只影响最直接的一个组件。长段落只影响文笔的移动端排版，不得再去前三章节奏重复扣分。
- 每个组件都必须有可定位的稿件、榜单或材料缺失证据。没有证据，不得给分。
- 除平台分中的固定档位外，其余组件按四档取值：
  - **充分**：证据清楚，且在正文中已经兑现，取该组件满分。
  - **部分**：方向成立，但兑现不完整、只出现一次或主要停留在大纲，取组件满分的 60%，四舍五入。
  - **微弱**：只有苗头，存在明显矛盾或需要审稿人替作者补全，取组件满分的 30%，四舍五入。
  - **缺失**：没有证据，或文本表现与目标相反，取 0 分。

### 证据规则

- 先建立统一证据索引。文本证据写明材料、章节和位置，短引文不超过 40 个中文字符；榜单证据只记录必要字段、时间和缓存状态。
- 同一证据可支持多个直接相关组件，但必须分别说明证明内容，也不能借复用重复降分。
- 0 分组件写明材料未提供、文本未出现或表现相反，不得留空。

## 二、平台匹配度，满分 100 分

平台匹配度只判断频道、分类、赛道信号、包装表达和合规风险，不重复评价正文节奏与文笔。

### 1. 分类匹配，满分 25 分

- 25 分：书名、简介、大纲和正文共同指向一个番茄分类。
- 18 分：没有完全对应分类，但能命中相邻分类，且主要读者承诺一致。
- 10 分：只能命中宽泛频道或情绪类型，分类依据较弱。
- 0 分：频道或题材判断互相矛盾，无法确定目标读者。

### 2. 分类榜单热度，满分 20 分

必须使用同一频道、同一榜型、相同 `per-category` 参数执行全分类扫描。只使用扫描结果中的 `heat_percentile`，数值越低表示热度位置越靠前。

- `0 <= heat_percentile <= 20`：20 分。
- `20 < heat_percentile <= 40`：16 分。
- `40 < heat_percentile <= 60`：12 分。
- `60 < heat_percentile <= 80`：8 分。
- `80 < heat_percentile <= 100`：4 分。

单分类 TOP10/TOP30 数据只能用于举例和解释，不能代替全分类热度位置。

7 天内完整扫描可以直接参与计分。网络刷新失败时，30 天内的完整旧扫描可以作为兜底参与计分，但必须标记低置信度和实际缓存年龄。残缺扫描、超过 30 天的扫描、没有可用缓存，或作品没有可对应分类时，本组件记为 `N/A`。此时平台分按其余 80 分重新折算：

`平台匹配度 = round（其余组件得分 ÷ 80 × 100）`

报告必须同时标注“榜单热度未参与计分”和数据置信度低，不得擅自给热度分。

### 3. 频道与题材承诺，满分 25 分

以下五项各 5 分；部分成立给 2 分，不成立给 0 分：

1. 目标频道清楚。
2. 核心题材清楚。
3. 核心情绪或阅读回报符合频道与题材。
4. 主角目标和主要冲突符合该类读者承诺。
5. 大纲方向、简介承诺和前三章实际内容一致。

### 4. 包装与投流表达，满分 20 分

以下四项各 5 分；部分成立给 2 分，不成立给 0 分：

1. 书名能够传达身份、冲突、关系或核心卖点。
2. 简介能够讲清主角处境、核心冲突和阅读回报。
3. 一句话梗概具有明确点击理由。
4. 第一章前部存在可提炼成推荐流文案或短推素材的具体场面。

用户没有提供书名或简介时，对应项只能给 0 分，并在报告中注明材料缺失，不得由审稿人代写后再给自己加分。

### 5. 平台合规，满分 10 分

- 10 分：未发现明显平台安全风险。
- 5 分：存在可修改的边缘风险，但不构成作品核心卖点。
- 0 分：明显依赖露骨色情、现实政治敏感、违法内容美化或其他高危表达作为主要卖点。

频道毒点属于读者体验问题，计入最直接的内容组件；不要再次从平台合规分扣除。

### 6. 数据置信度

数据置信度不计分，只写入报告：

- **高**：本次成功刷新了完整全分类扫描。
- **中**：使用 7 天内的完整扫描缓存；即使强制刷新失败，只要回退缓存仍在 7 天内，也按中置信度处理。
- **低**：使用超过 7 天且不超过 30 天的完整旧扫描、榜单热度记为 `N/A`，或分类仍含推测。

分类目录缓存只用于确认分类名称和 ID，不决定榜单热度置信度。报告必须记录扫描数据的 `fetched_at`、缓存年龄、`scan.complete`、实际扫描分类数和是否触发 `stale_fallback`。

## 三、故事主线，满分 100 分

- **主角核心目标，25 分**：主角想得到、守住、摆脱或改变什么。
- **长线推进发动机，30 分**：目标如何持续产生任务、冲突和下一阶段期待。
- **阶段推进与结果，25 分**：重要节点是否改变资源、能力、身份、局势或关系。
- **大纲与正文一致性，20 分**：前三章是否真正开始兑现大纲和简介承诺。

## 四、前三章节奏，满分 100 分

- **首屏钩子，25 分**：前 200 至 300 字是否出现有效事件、异常、压力或关系变化。
- **第一章题材承诺，25 分**：第一章是否讲清主角处境、近期目标和核心卖点。
- **第二章兑现，20 分**：是否回应第一章承诺，而不是继续重复铺设定。
- **第三章升级，20 分**：是否形成收益、反制、选择、关系变化、身份变化、新证据或新危机。
- **章尾追读，10 分**：三章章尾是否持续提供下一章点击理由。

慢热、持续受压、群像失焦等问题，直接降低对应组件，不再额外扣分。

## 五、核心梗与设定，满分 100 分

- **一句话清晰度，25 分**：核心卖点能否简洁说明。
- **题材新切口，25 分**：熟悉套路是否有身份、关系、职业、现实痛点、能力或世界规则上的新切口。
- **剧情转化能力，30 分**：核心设定是否已经在前三章制造选择、冲突和结果。
- **长期可持续性，20 分**：核心设定能否支撑多个阶段，而不是只够写一个开局噱头。

理解成本高、规则复杂、空有噱头等问题，直接降低最相关组件，不再额外扣分。

## 六、文笔与代入感，满分 100 分

- **信息清晰度，30 分**：读者第一遍能否看懂谁做了什么、为什么、结果怎样。
- **场景与动作结果，25 分**：描写是否落在可见事件上，关键动作是否有结果和反应。
- **人物语言与区分度，20 分**：对话是否符合身份，不同角色是否说话一个腔调。
- **移动端排版，15 分**：段落和信息密度是否适合手机阅读。
- **成稿完成度，10 分**：病句、错别字、标点和明显视角错误是否影响通读。

## 七、五维总分

固定权重：

- 平台匹配度：15%。
- 故事主线：20%。
- 前三章节奏：30%。
- 核心梗与设定：20%。
- 文笔与代入感：15%。

计算公式：

`最终总分 = round（平台匹配度×15% + 故事主线×20% + 前三章节奏×30% + 核心梗×20% + 文笔×15%）`

最终总分必须是 0 至 100 的整数。

## 八、高分复核

最终总分达到 80 分时执行高分复核。复核只检查组件证据是否足够，不增加或扣除隐藏分：

1. 所有达到 80 分以上的维度是否有正文证据。
2. 平台热度分是否来自同频道全分类扫描，而不是单分类榜。
3. 核心梗高分是否已经在正文转化成剧情。
4. 前三章节奏高分是否覆盖三章，而不是只靠一个开头句子。

发现虚高时，回到具体组件修改分数并重新计算；禁止直接从最终总分减 10 分或 20 分。

## 九、评级映射

- 最终总分 < 50 分：**当前版本不建议上线**。
- 最终总分 50 至 64 分：**大幅修改后复审**。
- 最终总分 65 至 79 分：**修改后可测试**。
- 最终总分 80 至 89 分：**具备上线测试价值**。
- 最终总分 90 至 100 分：**高潜力，优先推进**。
