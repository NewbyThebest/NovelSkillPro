# 网文样板书拆解

## 边界与路径

- 将本技能目录记为 `SKILL_DIR`，包含 `.agents/` 的项目根目录记为 `WORKSPACE_ROOT`。技能资源从 `SKILL_DIR` 解析，样板书和输出从 `WORKSPACE_ROOT` 或用户指定路径解析。
- 只拆解用户指定的网文样板书或竞品，不因作品被指定为样板书就宣称其商业成绩、平台表现或写法必然有效。
- 支持本地 `.txt`、`.md`。普通内容摘要和读书笔记不使用本技能；用户稿件质检改用 `webnovel-review`，大纲加完整前三章的番茄综合诊断改用 `ai-editor-fanqie`。
- 综合拆解输出到 `<WORKSPACE_ROOT>/样板书拆解/[书名]/2_样板书拆解报告.md`。全书梗概是中间材料，固定保存为同目录下的 `1_故事梗概.md`。

## 模式与材料

1. 用户要求完整、系统或全书拆解时，使用**综合模式**。至少覆盖开篇、章节目录或全书梗概、一个完整剧情单元；缺少中后期材料时，明确报告范围，不推断全书结构。
2. 用户只要求文风、黄金三章、爽点循环、人物关系或指定章节时，使用**专项模式**。只读取完成该主题所需材料，输出 `<WORKSPACE_ROOT>/样板书拆解/[书名]/[主题]_专项拆解.md`，不强制生成全书梗概。
3. 按“用户明确说明 > 文件元信息 > 书名、简介和正文承诺”判断频道、题材与目标平台。无法确认时使用中性口径；只有结论依赖该信息时才询问。
4. 先运行检查命令，确认编码、字符数、章节数和章节标题识别结果：

```bash
python "<SKILL_DIR>/scripts/summarize.py" "<样板书绝对路径>" --workspace-root "<WORKSPACE_ROOT>" --inspect
```

## 执行流程

### 1. 建立材料地图

- 综合模式优先读取：前三章、章节标题列表、`1_故事梗概.md`、首个完整回报单元、一个中段转折单元和一个后段升级或关系变化单元。根据作品结构调整样本，不固定要求“新手村”“换地图”或战力升级。
- 文风样本至少分布在两个不同阶段；只读开篇时不得把开篇语言直接概括为全书文风。
- 建立证据索引 `E1...En`，记录章节或区间、可核对事实、短引文及其支持的结论。关键结论必须引用证据编号；单段引文控制在 100 个中文字符以内。

### 2. 生成或复用全书梗概

- 综合模式已有可靠章节梗概时直接复用。缺少时运行：

```bash
python "<SKILL_DIR>/scripts/summarize.py" "<样板书绝对路径>" --workspace-root "<WORKSPACE_ROOT>"
```

- 脚本从 `<SKILL_DIR>/.env` 或环境变量读取 `NOVEL_API_KEY`、`NOVEL_API_BASE`、`NOVEL_API_MODEL`。缺少密钥时脚本会创建 `.env` 模板；仅在确实需要全书梗概时提示用户填写并等待确认。
- 专项模式或材料规模可直接处理时，不为调用脚本而调用脚本。脚本失败或报告缺章时不得继续宣称全书拆解完成；保留缓存后修复失败批次再运行。

### 3. 分析作品机制

按材料证据分析以下维度，不存在的项目写“不适用”，不要强行套模板：

1. **定位与读者承诺**：频道、题材、核心受众、开篇承诺和主要阅读回报。
2. **核心驱动器**：高概念、能力机制、身份差、关系矛盾、谜题或现实目标。只有作品实际存在时才分析金手指。
3. **人物与关系结构**：主角欲望、阻碍、关键关系、对手逻辑和角色功能变化。
4. **长线结构**：按作品实际采用的地图、案件、任务、关系阶段、身份阶层、时间或目标划分剧情单元，说明阶段切换的原因。
5. **回报与追读循环**：承诺、阻碍、蓄压、兑现、余波和下一钩如何循环；从证据归纳实际章数，不预设固定周期。
6. **开篇与微观写法**：首屏切口、信息释放、场景推进、章尾问题、完整剧情单元的起承转合。
7. **语言与叙事**：视角、句段、对白、叙述距离、情绪表达、信息密度和不同阶段的稳定性。
8. **题材边界与可迁移性**：哪些机制可复用，哪些依赖作者能力、平台环境或具体角色，哪些只能借鉴原理而不能照搬表达与情节。
9. **平台包装**：仅在用户要求或目标平台明确时，分析书名、简介、导语、首屏和投流场面的可包装信号。

### 4. 生成报告

- 完整读取 `<SKILL_DIR>/references/report_template.md`，按当前模式选择综合或专项模板。
- 先写观察事实，再写机制推断和可迁移规则。不得把单个案例写成普遍规律，也不得用“节奏快”“爽点密集”等空泛结论代替证据。
- 综合报告只保留一份最终报告，不再拆成文风、套路、框架和微观节奏四份重复文件。用户明确要求分文件时再拆分。

## 交付检查

- 报告范围与实际读取材料一致，未读部分没有伪装成全书结论。
- 所有核心结论均能回指章节证据；观察、推断和建议已经区分。
- 分析框架适配当前频道和题材，没有默认套用男频升级流术语。
- 可迁移规则说明适用条件和失效边界，不输出近似原文的仿写段落。
- 对话中提供报告路径、覆盖范围和不超过三条最重要的可迁移规律，交付后结束任务，不强制追加问答。
