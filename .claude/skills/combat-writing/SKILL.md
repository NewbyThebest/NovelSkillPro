# 打斗描写

## 使用原则

每场打斗先锁定一个可见目标：击败、脱身、拖时间、护人、夺物、试探、藏住实力或展示差距。结束时必须改变至少一项局面：位置、伤势、资源、情报、关系、名声或下一步选择。

重要战斗再选择一个主要情绪效果：痛快、紧张、压迫、悲壮、恐惧或悬念。普通交手可以简短，但不能只报出胜负而看不见过程和局面变化。

## 总流程

1. 锁定双方目标、战场、能力与规则边界、当前资源或伤势、不能暴露的信息和胜负条件；直接写作时缺少关键事实，只问一次。
2. 判断战斗类型。
3. 按“按需读取指南”诊断稿子缺什么，据此读取 `references/` 中的对应模板，不只看用户明说了什么。
4. 设计主角策略、对手压力和结算后的局面变化。
5. 用动作链写清交锋因果：进攻 -> 防御/闪避/硬接 -> 反击 -> 结果 -> 反应 -> 调整。
6. 只在关键转折加入有来源、能提供新信息的感官细节。
7. 落到明确的情绪效果、代价或下一步悬念。

## 按需读取指南

只读相关文档，避免一次加载全部模板。但"相关"不由"用户明说了什么需求"决定，而由**稿子本身缺什么**决定——哪怕用户只说"润色"，也要先扫一遍稿子，命中下面哪条缺陷，就去读对应文档再动手。

**动手前先诊断（写正文、润色、诊断都要过这一步）：**

- 有放狠话、立 flag、看轻主角的角色，但结算时没被打疼、没沉默改口付代价？→ 读 [payoff-types.md](references/payoff-types.md)（打脸、结算爽）。
- 金手指、杀招、反杀手段冒得突兀，前文没露过端倪？→ 读 [tactics.md](references/tactics.md)（反制要有铺垫）。
- 对手降智、主角靠对手犯蠢才赢、赢得没道理？→ 读 [tactics.md](references/tactics.md)（避免降智）。
- 主角明明能赢却要装弱、控分、不能暴露实力？→ 读 [hidden-strength.md](references/hidden-strength.md)。
- 主角其实不弱、故意装成废物，最后掀桌碾压打脸（扮猪吃虎）？→ 读 [hidden-strength.md](references/hidden-strength.md)（装弱）+ [payoff-types.md](references/payoff-types.md)（秒杀、打脸）。爽点在揭穿反差和强者破防，不是险死还生。
- 拿不准这场该怎么打、是什么形态？→ 读 [fight-types.md](references/fight-types.md)。
- 想加强某种情绪爽点（打脸、压迫、技巧、藏锋、结算、代价、惨烈）？→ 读 [payoff-types.md](references/payoff-types.md)。

按上面命中的项去读，命中几条读几个，别因为用户没明说就跳过。

常见组合：

- **规则擂台 + 藏拙：** `fight-types.md` + `hidden-strength.md` + `payoff-types.md`。
- **追杀反杀：** `fight-types.md` + `tactics.md`。
- **越级硬战：** `fight-types.md` + `payoff-types.md`。
- **群战：** `fight-types.md` + `tactics.md`。
- **斗法/异能：** `fight-types.md`。
- **枪战/热兵器：** `fight-types.md` + `tactics.md`。

## 基础动作链

优先写清因果，不要只堆华丽招式名。

```text
对手带着明确目的出手。
主角读出压力，或先承受压力。
主角格挡、闪避、硬接、换伤、诱敌，或借地形化解。
交锋造成可见、可听、可感的结果。
对手、系统、裁判、亲友、围观者或高手重新判断局势。
下一拍改变距离、节奏、风险或策略。
```

以下词库是按场景取用的备用，不按顺序堆砌；每个动词都要带来距离、姿势、伤势、武器状态或局势变化。

基础动作语法：

- **攻击：** 劈、斩、刺、挑、扫、砸、贯、钩、踢、肘击、肩撞、掌拍。
- **防御：** 格挡、横架、卸力、侧身、矮身、翻滚、后撤、硬接、借势、以伤换位。
- **反击：** 踹腹、击腕、砸颌、断筋、折兵、锁臂、贴身、攻盲区、借环境反杀。
- **结果：** 血线、手臂发麻、地砖开裂、枯叶倒卷、袖口撕开、耳中嗡鸣、刀锋弯折、满场死寂。

以上是冷兵器/肉搏词库。写枪战或精神战时，换用下面对应的一组：

**枪战/热兵器：**

- **攻击：** 点射、扫射、抬枪、瞄、压制、爆头、盲射、抛壳、拉栓、换弹匣。
- **防御：** 找掩体、贴墙、翻窗、卧倒、滚进死角、压低身位、听枪声判位、等换弹空隙。
- **反击：** 绕侧翼、抢掩体、近身夺枪、贴脸肉搏、打油箱/灯/管线借爆、掐火力间隙突进。
- **结果：** 弹壳跳动、墙面炸出弹孔、木屑飞溅、跳弹尖啸、血雾、枪管发烫、卡壳、弹匣见底的空响。

**精神/意识战：**

- **攻击：** 神识撞入、意志碾压、幻象、心魔、灼烧神魂、精神尖啸、记忆入侵、规则压制。
- **防御：** 稳住心神、筑起识海壁垒、咬破舌尖、以痛清醒、认出幻象破绽、把意志缩成一点。
- **反击：** 反噬入侵者、以幻破幻、抓住对方执念、把痛感反推回去、用一个念头钉死对手。
- **结果：** 太阳穴刺痛、七窍渗血、瞳孔失焦、识海龟裂、幻象崩碎、对方踉跄后退、鼻血砸在地上。

## 感官与结果

感官细节只放在危险、命中、反转或结算等关键点，每次保留一两个，并说明它来自什么、改变了什么：

- 危险前：武器、地形、倒计时或对手动作如何提示压力。
- 命中时：哪处受力、哪件武器或哪段地形发生了什么变化。
- 反转时：哪条规则、哪次失误或哪项资源让局势改写。
- 结算时：伤势、消耗、战场痕迹或制度判定如何确认结果。

## 节奏

- 复杂交锋先交代动作因果，再落结果；关键命中、反转和结算可单独成短句。
- 不要连续用碎短句假装紧张，也不要用长句淹没距离、受力和结果。念出声卡嘴，优先检查动作顺序和信息拥挤，而不只看句长。

## 润色检查

### 删改前的三问（每处修改都要过）

动手改任何词句之前，必须先问：

1. **这个词/句在承载什么？** 节奏/人物气场/画面感/情绪递进/对比反差？
2. **删了，这些会不会一起没了？** 比如"慢慢转一圈"承载的是掌控感，删了等于削气场。
3. **念出声，改后版本卡不卡嘴？** 读出来不顺，就是改错了。

**铁律：功能优先于精简。承载节奏和人物的词，哪怕看着"多余"，也不删。**

### 正文检查（按顺序）

1. 开打前的矛盾是否够热，读者是否觉得必须打。
2. 战斗条件是否清楚：地形、规则、倒计时、胜负标准、不能做什么。
3. 对手是否靠动作显强，而不是旁白说强。
4. 主角是否有策略，而不是被动挨打或机械闪避。
5. 每个动作是否有后果：伤、退、乱、断、招式暴露、气势被压。
6. 节奏是否服务动作因果，关键结果是否有足够停顿，而非连续碎短句或信息拥挤的长句。
7. 结尾是否完成既定目标，并留下明确的情绪效果、代价或下一步悬念。

## 输出方式

- **设计打斗：** 给起因、战场、双方目标、能力与规则边界、战斗节拍、反转、局面变化和余波。
- **直接写正文：** 按用户要求的风格、视角和字数写成可读段落，遵守已有能力、伤势和信息边界。
- **润色原文：** 保留原剧情结果与设定边界，增强动作清晰度、压力、感官结果和结算。
- **诊断问题：** 先指出最大问题，再给具体修法，必要时附示范改写。
