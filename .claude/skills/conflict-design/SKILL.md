# 番茄冲突设计

把“剧情有点平”改成读者看得懂压力、看得见行动、愿意等待结算的冲突。

## 前置确认

设计或改稿前，先确定章节位置、主角和阻碍方各自要什么、已知事实与不能改动的限制，以及本场结束后希望留下什么变化。缺少关键事实时只问一次；结局章或用户明确要求收束时，不强行设计章尾钩子。

## 核心原则

1. **开头不劝退，章尾留钩**：章节开头别用背景、回忆、景物、复述上一章开场，要快速进入正在发生的事；非收束章的章尾留下悬念、威胁或反转预告。是否强冲突开场看章节位置。
2. **冲突来自人和利益**：不要只有“我很难受”，要有某个人、规则或事件正在抢主角的东西。
3. **读者情绪要明确**：愤怒、心疼、好奇、爽、恐惧、期待，至少命中一个。
4. **主角必须行动**：主角可以暂时吃亏，但必须观察、试探、反击、取证、设局或做选择。
5. **冲突必须改变局面**：结束后资源、关系、信息、地位、危险等级至少变一项。
6. **虐点要配结算预期**：可以压主角，但要给读者看到后面怎么反杀、揭穿或打脸。

## 章节位置决定冲突强度

“前 200 字必须出冲突”是**全书开篇的特化规则，不是每章通则**。先判断这是什么位置的章节，再决定开头要多强：

| 位置 | 开头要求 | 说明 |
|---|---|---|
| **全书开篇**（第一章，尤其前 200 字） | **必须强冲突/强异常** | 冷启动，面对零耐心新读者，没有前情可借力，只能靠开头本身瞬间抓人。这条是铁律。 |
| **承接章**（上一章章尾留了钩子） | **直接接住上一章的钩子** | 开头本身已经在张力里，不要另起炉灶新造一个冲突，那样反而打断追读的连贯性。 |
| **常规章 / 过渡章 / 铺垫章 / 情感章** | **不劝退 + 快速进入** | 不必每章前 200 字硬造新冲突。守住两条：开头别铺背景/回忆/景物/复述；章尾留下下一章的阅读理由。 |
| **收束章 / 结局章** | **完成承诺，允许收口** | 重点是结算、余波或情感落点；除非用户要续篇，不强行留下新钩子。 |

- ❌ **不要每章都硬塞“前 200 字新冲突”**：主角不可能每 2000 字被踩一次脸、抢一次东西，硬塞会失真、会让读者疲劳，还会和“承接上一章钩子”打架。
- ✅ **长篇要有呼吸**：铺垫章、过渡章、信息章、情感章都是必要的，它们靠“不劝退的开头 + 留住的章尾钩子”维持追读，而不是靠开头爆冲突。
- ✅ **连载章节的留存点在章尾**：决定读者点不点下一章的，是上一章结尾留没留悬念，不是这一章开头有没有新冲突。

## 番茄冲突七型

按场景选择 1-2 个叠加：

| 类型 | 核心刺激 | 适用场景 | 关键问题 |
|---|---|---|---|
| **羞辱压迫** | 主角被当众踩脸 | 退婚、职场、家族、校园、豪门 | 谁让主角下不来台？ |
| **利益抢夺** | 钱、房、资源、名额、功劳被抢 | 世情、都市、玄幻、末世 | 对方拿走了主角什么？ |
| **关系背叛** | 亲人/爱人/朋友反咬 | 婚恋、宅斗、团队、兄弟 | 背叛证据是什么？ |
| **规则倒计时** | 不做选择就会损失 | 悬疑、末世、系统、竞技 | 主角还剩多少时间？ |
| **秘密揭露** | 一个新信息改写局面 | 真假千金、重生、悬疑、权谋 | 揭露后谁会崩？ |
| **强者碾压** | 强敌或权力压迫主角 | 玄幻、高武、历史、朝堂 | 主角凭什么破局？ |
| **两难抉择** | 两个目标无法兼得 | 救人、站队、婚恋、团队、道德困境 | 选哪边会失去什么？ |

## 冲突升级公式

用这 6 步设计或改写冲突：

1. **主角要什么**：本场景主角的具体目标。
2. **谁来抢/挡/骗/逼**：阻碍方必须有自己的利益。
3. **压迫动作**：对方当场做了什么，而不是只说了什么。
4. **主角代价**：不反击会失去什么？反击会付出什么？
5. **主角动作**：主角如何观察、试探、反击、取证、设局或硬刚。
6. **局面变化**：冲突结束后出现新证据、新敌人、新危险、新爽点或新钩子。

## 打脸/爽点兑现七型

压迫要么在本场兑现，要么给出明确的后续兑现预期。按场景选 1-2 个兑现方式，和压迫型咬合：对方怎么错判、抢夺或逼迫，主角就用相应的实力、信息、证据或布局还回去。

| 兑现型 | 核心快感 | 关键问题 |
|---|---|---|
| **实力揭露**（扮猪吃虎/隐藏境界/越级） | 你以为我弱，其实我碾压你 | 主角藏了什么实力？ |
| **信息差/鉴定** | 我看穿了你不知道的真相（你的宝物是假的、你的靠山要倒） | 主角掌握了什么对方不知道的信息？ |
| **证据反转** | 取证后当众揭穿黑幕/背叛 | 打脸的证据是什么？ |
| **身份揭露** | 你不知道我是谁（主角自己就是大佬/有背景） | 主角真正的身份是什么？ |
| **设局反杀**（请君入瓮/自食其果） | 你的算计正好坑了你自己 | 主角怎么让对方掉进自己挖的坑？ |
| **大佬登场**（有人替主角出面/认出主角分量） | 你惹的人，背后站着你惹不起的 | 谁出面？他为什么替主角出头？ |
| **实力兑现**（硬实力当场解决） | 我就是比你强，当场打服 | 主角凭什么当场赢？ |

- ⚠️ **大佬登场慎用**：主角要先做过求助、结下善缘或展现价值等动作；外援是主角布局或积累的结果，不能替代主角行动。
- ⚠️ **别用主角想不到、做不到的方式兑现**：金手指/底牌/靠山可以有，但要在兑现前埋过线索或立过设定，不能临场硬造一个。读者要的是"原来如此"，不是"凭什么"。

## 爽点兑现公式

设计"糖怎么给到位"，和压迫的冲突升级公式对称：

1. **对方错判了什么**：压迫方基于什么错误认知踩了主角（以为他弱、以为他没背景、以为他的东西是真的）。
2. **主角的底牌**：主角手里那张能反转的牌是什么（实力、信息、证据、身份、局、靠山），这张牌在兑现前有没有埋过线索。
3. **兑现时机**：当众、关键人物在场、对方最得意的那一刻——打脸要打在对方声势最高的时候才爽。
4. **反转动作**：主角当场做了什么，把错判砸回对方脸上（不是主角说"其实我很强"，是用一个动作证明）。
5. **结果反应**：压迫方或受影响者如何重新判断局势。回扣对方的核心判断、赌注或条件，不必逐句复述狠话。
6. **外部放大（可选）**：公开场景可用旁观者、制度或舆论放大反转；私密场景则写关系、资源或信息后果。

## 结算节奏纪律

压迫没有兑现或结算预期，读者容易失去追读；但蓄势段和兑现段不能用同一标准要求。

**先分清这段处在"虐→爽"周期的哪个位置，再决定它该不该自带爽点：**

| 节奏位置 | 这段该有什么 | 别做什么 |
|---|---|---|
| **蓄势段 / 养雷章**（反转、打脸设计在后面章节） | 让读者看到后续兑现的预期，且主角在观察、试探、取证或攒牌，不能纯挨打。 | 别硬在这段塞反转或大爽点，打断蓄势。 |
| **兑现段**（这就是该爽的那一章/那一段） | 用"打脸/爽点兑现七型"检查糖给得够不够狠、够不够解气。 | 别把该炸的爽点写软、写成讲道理或对方良心发现。 |

- ⚠️ **判断依据**：反转/爽点是在这段之内，还是在后面章节？位置不明就问用户一句，别默认"每段都必须自带爽点"——那和"每章硬塞 200 字冲突"是同一种错。
- **蓄势段允许不给当场爽点**，但要给后续兑现预期，且主角不能纯挨打。是否给微小扳回，按本书节奏决定。

以下针对**兑现段**（或明确该给当章小爽点的场景）：

- ✅ **兑现段给足回报**：主角当章被踩、且本章就是兑现段时，至少完成一步反击、拿到一张牌或发生反转，让局面从完全没辙变成有路可走。
- ✅ **大结算给明确预期**：可以养几章再爆，但要让读者看见触发条件、主角的准备或合理的兑现时机；不要求机械承诺具体章数。
- ❌ **虐点悬空**：只压不给糖、也不预告糖，章尾还在纯虐——读者会跑。
- ❌ **爽点迟到**：大结算拖十几章还不炸、期间小爽点也断供，追读会掉。
- 自查：这段是蓄势还是兑现？蓄势段有没有兑现预期和主角动作，兑现段有没有实际回报？

## 输出格式

当用户要求设计冲突时，按以下格式输出：

```markdown

- **主角目标**：
- **阻碍方目标**：
- **冲突本质**：谁要从谁身上拿走什么？

### 2. 冲突类型
- **压迫主类型**：[羞辱压迫/利益抢夺/关系背叛/规则倒计时/秘密揭露/强者碾压/两难抉择]

- **打脸兑现型**：[实力揭露/信息差鉴定/证据反转/身份揭露/设局反杀/大佬登场/实力兑现]——这一场或后续用哪种把压迫还回去
- **读者情绪**：[愤怒/心疼/好奇/爽/恐惧/期待]

### 3. 场景节拍
1. **开头**：全书开篇章前 200 字必须出冲突/异常；常规章快速进入正在发生的事、优先承接上一章的钩子，不硬造新冲突。
2. **第一次压迫**：
3. **主角短暂失衡**：
4. **压迫升级/证据出现**：
5. **主角主动动作**：
6. **本章爆点**：
7. **章尾钩子**（非收束章必填）：留悬念、危险或反转预告；收束章改为结算或余波。

### 4. 爽点/虐点结算
- **本章虐点**：
- **本章回报**：[兑现段写本章小爽点；蓄势段写主角攒到的牌或后续兑现预期]
- **打脸兑现方式**：用哪一型、靠什么兑现（底牌/信息/证据/身份/局/外力）
- **后续大结算预期**：怎么正面结算，以及触发条件或准备动作

### 5. 可直接写入正文的桥段

```

## 改稿模式

如果用户提供了原片段，先确认它在书里的位置（全书开篇 / 常规章 / 承接上一章的章节），再判断问题类型。位置不明就问一句，别默认按开篇处理——常规章开头没有新冲突不是问题，只要它快速进入、承接住了上一章的钩子。

- **没压迫**：对手只是说话，没有实际抢夺/威胁。
- **没代价**：主角输赢都无所谓。
- **没动作**：主角只想不做。
- **没升级**：整场都在重复同一个情绪。
- **没钩子（连载章最致命）**：非收束章的章尾收成感悟或解释，没给读者点下一章的理由。
- **开头劝退**（仅查常规章开头）：用背景、回忆、景物、复述上一章开场，迟迟不进入正在发生的事。
- **只虐不爽**：先分清是不是蓄势段。兑现段缺回报是问题；蓄势段则检查有没有后续兑现预期和主角动作。位置不明时只问一次。
- **打脸软**：反击靠讲道理、靠苦熬、靠对方良心发现，而不是靠主角的底牌/信息/证据/身份/局把对方按下去。

然后输出：

```markdown
### 原问题
[指出 2-4 个具体问题]

### 升级方案
[压迫侧按冲突七型重排；兑现侧按七型定具体方式，并标明蓄势或兑现节奏]

### 改写片段
[给出可直接替换的片段]
```

## 题材校准

每个题材给一组常用的"压迫型 → 兑现型"搭配（压迫怎么来的，糖就怎么还回去）：

- **世情/家庭**：利益抢夺 + 关系背叛 → 证据反转 / 身份揭露；重点写“理所当然的剥削”被当众戳穿。
- **都市/职场**：羞辱压迫 + 利益抢夺 → 证据反转 / 身份揭露 / 大佬登场；重点写身份反差和证据反杀。
- **玄幻/高武**：强者碾压 + 规则倒计时 → 实力揭露 / 实力兑现；重点写境界压迫和即时突破、扮猪吃虎。
- **末世**：规则倒计时 + 利益抢夺 → 设局反杀 / 实力兑现；重点写物资、生存名额、安全区，靠布局和硬实力翻盘。
- **悬疑/灵异**：秘密揭露 + 规则倒计时 → 信息差鉴定 / 证据反转；重点写异常、线索、误判，靠信息差反制。
- **古言/宫斗/宅斗**：关系背叛 + 秘密揭露 → 设局反杀 / 证据反转 / 身份揭露；重点写身份、礼法、证据链。
- **现言/追妻/豪门**：关系背叛 + 羞辱压迫 → 身份揭露 / 大佬登场；重点写误会证据和情绪反噬、隐藏身份亮明。

## 自查清单

- [ ] **先分清这是全书开篇还是常规章**：开篇章看“前 200 字有没有强冲突/强异常”；常规章看“开头有没有快速进入正在发生的事、有没有承接上一章的钩子”，不硬塞新冲突。
- [ ] **非收束章的章尾有没有留住钩子**（反转、证据、威胁、打脸预告、悬念）？
- [ ] 阻碍方是否有明确利益？
- [ ] 主角不行动是否会付出代价？
- [ ] 主角是否做了主动动作？
- [ ] 本场是否有情绪爆点？
- [ ] **虐了主角，有没有配一个明确的兑现方式或后续预期**？
- [ ] **这段的位置是否匹配节奏**：蓄势段有预期和主角动作，兑现段有实际回报？
- [ ] 冲突结束后局面是否发生变化？
