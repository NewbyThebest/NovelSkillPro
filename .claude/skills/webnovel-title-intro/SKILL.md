# 书名与简介

## 目标与边界

让读者在移动端快速看清题材、主角处境或核心关系，以及持续追读的理由；不把书名和简介写成世界观说明书、谜语或逐章梗概。

本技能只处理书名、简介和按需的推广短文案。用户选定书名后，只有明确要求封面、封面方案或绘图提示词时，才读取 `bookcover-design`。

## 输入与续写

按形成当前任务所需的范围读取：

1. `1-预期/预期.md`：平台、频道、核心读者、题材、读者承诺和内容边界。
2. `2-设定/2.2_新书设定案.md`：开局处境、核心矛盾、主线和关系或世界规则。
3. `2-设定/2.3_金手指设定.md`、`2.4_主要角色设定表.md`：只读取能支撑卖点的核心机制、身份、关系信息。
4. `3-大纲/3.1_全书结构总纲.md`：首屏与前三章承诺、第一阶段近端追读节点和可包装卖点。

缺少总纲时可基于预期和设定生成，但必须说明后续走向的依据较弱；缺少会改变书名或简介方向的关键资料时，只问一次。

已有 `2-设定/候选书名与简介.md` 时先读取。用户要求改现有候选时，保留已选方案和确认过的限制，只新增、替换或修订相关部分，不整份覆盖。

## 任务模式

先识别用户实际需要什么，不强制同时生成书名和简介：

1. **组合模式**：默认生成 3 组“书名 + 简介”，每组主打不同的读者承诺。
2. **书名模式**：用户只要书名时，生成 3-5 个方向明确、互不只是同义词替换的书名，并标注各自的题材信号。
3. **简介模式**：用户已有书名或只要简介时，围绕已定书名生成 1 个主版本；只有存在明显定位取舍时，再补 1-2 个不同方向的版本。
4. **改版模式**：用户提供现有书名、简介或数据反馈时，先指出题材信号、卖点和追读承诺哪里不足，再给针对性的改版，不凭空重做整套方案。
5. **推广模式**：仅当用户明确需要投流、短推或导语时，额外提供 1-3 条短文案。

## 生成流程

### 1. 提炼定位

先用一句话明确：频道、核心题材、主角或关系的开局处境、持续回报，以及本次最该放大的一个卖点。已有信息冲突时先说明，不把未定设定写入文案。

### 2. 生成候选

- 候选之间必须在主打信号上有真实差异，例如主角身份、核心关系、特殊机制、危机事件或情绪承诺，而不是只换几个词。
- **书名**：从题材、主角或关系、关键机制、目标或代表事件中选择一至两个最强信号。无需把所有元素塞进标题；核心词尽量靠前，避免生僻字、泛化大词和看不出题材的空泛抒情。
- **简介**：番茄默认约 100-200 字，用户或目标平台另有要求时以其为准。前半段落到具体开局压力、变化或关系问题，后半段亮出主线将如何持续推进。男频可突出目标、资源、能力、竞争或责任；女频可突出命运、关系、身份、职业、利益处境或情绪边界。两者都要服从作品实际卖点。
- 不只复述激励事件，不堆世界观，不用“阴谋交织”“宿命”“梦想”等空词替代故事。除非用户明确要这种口吻，否则不以反问句作为简介结尾。
- 推广短文案只提炼已经存在的强冲突、反差或情绪场面，不新造剧情和设定。

### 3. 写入与确认

写入 `2-设定/候选书名与简介.md`。组合模式使用以下结构；其他模式删除不适用的小节。

```markdown
# 书名与简介候选

## 定位摘要
- 频道与核心读者：
- 题材与开局处境：
- 核心读者承诺：
- 本次主推卖点：

## 候选 A
- 主打信号：
- 书名：
- 简介：
- 适合的读者预期：

## 候选 B

## 候选 C

## 已选方案与后续调整
```

交付时只说明主推方向和不超过三条关键判断。等待用户选择或给出修改方向后，再更新候选文件。

## 自查

- 书名和简介至少有一处让目标读者快速识别题材或核心阅读体验。
- 简介既有开局抓手，也有后续追读理由；不能只讲“发生了什么”。
- 不夸大未定设定，不承诺作品没有的爽点、关系或结局。
- 候选有不同定位，且与目标平台、频道和内容边界一致。
