# 人物活人感

## 材料与边界

- 本技能不另建角色卡或写入设定文件。新书完整人设由 `webnovel-ideation` 建档；这里只提供可合入角色表的补强项。
- 用户只要补一个新核心角色的活人感时，按下文五项输出，不要求先提供剧情片段。
- 诊断已有角色时，读取角色设定与相关正文；主要角色读取 `2-设定/2.3_主要角色设定表.md`，阶段或功能角色读取 `2-设定/2.3.1_阶段与功能角色记录.md`，问题涉及已确认关系时再读取 `2-设定/2.3.2_核心关系.md`。缺少关键材料时，只问一次“要补哪个角色，问题出现在哪段剧情？”

## 角色层级

| 层级 | 判断 | 处理 |
|---|---|---|
| 核心角色 | 主角、核心反派，或反复影响主线、关系的人 | 完整检查五项。 |
| 功能配角 | 有固定场景作用，但不承担长线变化的人 | 只补当下得失和差异化反应；反复影响关键决定时升为核心角色。 |
| 工具人、过场角色 | 一次性完成场景动作的人 | 够用即止，行为符合身份；不补身世、欲望或成长。 |

工具人写得扁平并不自动构成问题，不为活人感强行加戏。

## 核心角色检查

每个核心角色都要落到以下五项，不能只给性格标签：

1. **当前目标与代价**：他现在想得到什么，失去会怎样。
2. **压力下的选择**：在风险、羞辱、利益冲突或关系拉扯下，他优先保什么、舍什么；给一个可写进场景的决定或动作。
3. **过去的外露方式**：过去只保留会改变当下选择的部分，通过习惯、偏见、禁忌、熟练度或回避动作露出，不集中倾倒身世。
4. **独立利益**：核心配角和反派必须有不依附主角的目标、底线或算盘；主角要有明确的个人得失。
5. **变化链**：用 `困境 -> 选择 -> 代价 -> 新的行为` 表示成长，不用总结代替事件。

记忆点应是可复现但不机械重复的行为模式、决策倾向或情绪触发点；“冷、狠、傲、温柔”不是记忆点本身。主角代入感则检查读者能否看懂他想要什么、怕失去什么，以及为何必须自己做这个选择。

## 交付

被 `webnovel-ideation` 读取时，在生成核心角色前按本技能检查。核心角色补强项合入 `2.3_主要角色设定表.md`；功能角色只给当下得失和差异化反应，由创意流程写入 `2.3.1_阶段与功能角色记录.md`。

单独诊断或改稿时，依次输出：

1. 角色层级，以及是否需要补强。
2. 最关键的一至三个问题，并引用设定或正文证据。
3. 对应的补强动作，以及应放入哪类事件。
4. 一个场景节拍：`压力 -> 选择 -> 结果`。需要时再给短改写，保持已有事实、视角和关系不变。

优先补目标与代价、压力反应、变化链；不要用额外悲惨经历、无关秘密或大段内心独白代替人物行动。
