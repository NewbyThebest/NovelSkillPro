# 提示词-04-章节创作

用于逐章创作番茄短故事正文。

## 输入

```text
本章章纲：

角色语言风格摘要：

前文内容：
第二章起粘贴前 1-2 章正文或关键摘要。
```

## 写作硬规则

1. **目标字数**：默认 800-1500 字，以 `scripts/count_words.py` 为准。
2. **首屏钩子**：本章前 200 字必须出现冲突、异常、威胁、羞辱、证据、反转或强问题。
3. **对话推进**：优先用对话和行动推进剧情，少写解释。
4. **第一人称优先**：若项目已定第一人称，严格只写“我”能看到、听到、感受到的内容；如用户指定第三人称，则保持单章视角稳定。
5. **段落短**：移动端阅读，段落尽量短，关键句单独成段。
6. **每章有爆点**：不能只有过渡；本章必须有情绪刺激或新信息。
7. **章尾有钩子**：用动作、台词、证据、电话、门铃、消息、反转收尾；禁止感悟式总结。
8. **虐爽比可控**：如果本章主虐，章尾必须给反击预期或真相钩子。

## 禁止项

- 连续大段背景介绍
- “第二天/三天后/一周后”式机械时间戳开头
- 连续 3 句以上心理独白
- 章末哲理总结、作者感悟、空泛升华
- 重复直述年龄、职业、家庭背景
- 角色为了剧情需要突然降智
- “空气仿佛凝固”“眼底闪过一丝复杂”等套路句

## 正文输出

只输出正文，不在正文文件里加入解释或审核。

正文完成后，在对话中附快速审核：

```text
【第 X 章快速审核】
字数：运行 python scripts/count_words.py 章节/第X章.md
首屏钩子：通过/需改
章纲符合：通过/需改
情绪爆点：通过/需改
主角主动性：通过/需改
章尾钩子：通过/需改
反 AI 表达：通过/需改
人物真实感：通过/需改
结论：可进入下一章 / 建议局部修改 / 建议重写
```

## 自查清单

- [ ] 前 200 字是否抓人？
- [ ] 本章是否真的推进了冲突？
- [ ] 是否有新证据、新威胁、新反转或新行动？
- [ ] 主角是否至少做了一个主动动作？
- [ ] 章尾是否让人想点下一章？
- [ ] 是否符合角色语言风格？
