# 提示词-05-章节审查

用于审查番茄短故事单章或多章正文，找出影响点击、完读、情绪推进和反转结算的问题。

## 输入

```text
审查章节：

本章章纲：

角色语言风格摘要：

正文：

前文摘要：
多章联审时填写。
```

## 审查维度

### 1. 字数与移动端可读性

- 是否 800-1500 字，或符合用户指定字数？
- 段落是否短，关键句是否能被手机读者扫到？
- 是否有大段说明、长段心理、密集背景？

### 2. 首屏钩子

- 前 200 字是否出现冲突、异常、威胁、羞辱、证据、反转或强问题？
- 如果删掉前 200 字，读者是否会失去点击理由？
- 开头是否直接进入人物关系矛盾？

### 3. 章纲符合度

- 情节点是否按顺序落实？
- 情绪爆点是否写出来，而不是只在章纲里存在？
- 主角动作是否清晰？
- 章尾钩子类型是否与章纲一致？

### 4. 情绪推进与虐爽比

- 本章主情绪是什么？
- 是否只有压迫没有反击预期？
- 是否有小爽点、真相推进、证据出现或打脸预告？
- 情绪是否递进，还是原地重复吵架？

### 5. 反转和信息差

- 本章是否新增信息？
- 反转是否有伏笔？
- 悬念是否明确，还是靠含糊拖延？
- 读者看完是否知道“下一章想看什么”？

### 6. 人物真实感

- 角色行为是否有利益、恐惧、欲望或软肋支撑？
- 对手是否只会发疯，还是有操控逻辑？
- 主角是否主动，还是一直被动挨打？
- 对话是否符合角色语言风格？

### 7. 反 AI 表达

- 是否有套路句、机械排比、过度修饰？
- 是否有“空气凝固、眼神复杂、心如刀割”等泛化表达？
- 是否有无源监控镜头或越视角？
- 是否存在重复话术？

## 输出格式

```markdown
## 【章节审查】第 X 章

### 总体结论
[通过 / 局部修改 / 建议重写]

### 分项评分
- 字数与移动端可读性：[X/10]
- 首屏钩子：[X/10]
- 章纲符合度：[X/10]
- 情绪推进与虐爽比：[X/10]
- 反转和信息差：[X/10]
- 人物真实感：[X/10]
- 反 AI 表达：[X/10]

### 关键问题
1. **问题**：[具体问题]
   **位置**：[引用原句或段落]
   **修改建议**：[怎么改]

### 可直接替换示例
原句：
> ...

改法：
> ...

### 下一步
[继续下一章 / 修改后再写 / 重排章纲]
```

## 多章联审额外检查

- 反转是否太密或太少？
- 虐点是否连续堆叠导致读者疲惫？
- 打脸和结算是否拖太久？
- 角色话术是否重复？
- 伏笔是否遗忘？

## 检查清单

- [ ] 是否明确指出首屏问题？
- [ ] 是否给出具体原句或段落？
- [ ] 是否区分“执行问题”和“章纲问题”？
- [ ] 是否给出可操作改法？
