# 克制与入稿压缩模板

用于把“火力全开”的打斗改成更成熟、可入稿的正文。克制不是削弱打斗，而是让力量集中到最该爆的位置。

## 先定篇幅等级

根据场景在章节中的功能控制密度。

- **过场冲突，300-600 字：** 只保留一个动作亮点和一个结算点。
- **小高潮，800-1500 字：** 一个主爽点，两个次爽点，三处关键动作。
- **章节高潮，1500-3000 字：** 一个主爽点，二到三个次爽点，可有完整升级和余波。
- **大高潮，3000 字以上：** 允许多阶段战斗，但每阶段仍只服务一个明确目标。

如果用户没有指定，训练、测试、路边冲突默认按“小高潮”处理。

## 主爽点优先

每场战斗只选一个主爽点，其他爽点只服务它。

- **藏锋爽为主：** 压迫、技巧、结算都要服务“主角能赢但不赢”。
- **打脸爽为主：** 藏拙和技巧不要盖过最终打脸。
- **压迫爽为主：** 不要让主角太早显得游刃有余。
- **技巧爽为主：** 不要用纯力量结算破坏聪明感。
- **结算爽为主：** 系统、裁判、惩戒、积分要短而冷，不要解释太多。

检查句：

```text
这一段最想让读者爽的是什么？
其他描写有没有抢它的戏？
```

## 删除规则

正式入稿前删掉这些内容：

- 同一心理判断重复表达两次以上。
- 连续三段都用单行短句制造紧张。
- 每个动作都配五感。
- 对手强度已经由动作证明后，又用旁白解释一遍。
- 主角策略已经被动作展示后，又用内心独白完整复述。
- 系统/教官/围观者反应过多，打断主战斗节奏。
- 为了展示模板而加入的多余招式、术语、判断。

## 保留规则

这些内容不要轻易删：

- 战斗规则和胜负条件。
- 第一次证明对手很强的动作。
- 主角第一次有效反击。
- 主角看出规律的关键证据。
- 主角能赢却不赢的理由。
- 系统、裁判或局势的最终结算。
- 会影响后续剧情的伤、记录、误判、惩罚、奖励。

## 短句密度

短句只留给四类时刻：

- 出手：`拳到面前。`
- 命中：`砰。`
- 反转：`他没有退。`
- 判定：`时间到。`

普通交锋用中句承接，避免整段都像鼓点。

压缩前：

```text
拳到面前。
祁临只偏了半肩。
没躲净。
衣袖炸开。
手臂火辣。
下一拳又来。
```

压缩后：

```text
拳风擦裂衣袖，手臂侧面立刻烧起一线痛。祁临没退远，只借这半步递出一拳。
```

## 心理压缩

重要心理只保留最有力的一次。若必须重复，第二次必须推进新信息。

低效重复：

```text
不能太亮，也不能太假。
他不能暴露。
江照野已经在明处，他不能也进去。
```

压缩为：

```text
江照野已经在明处。
他要的不是胜，是刚好不输。
```

## 五感压缩

五感只放在关键点：

- 对手第一次显强。
- 主角第一次受伤。
- 主角第一次反击成功。
- 最后结算或余波。

其他交锋只写动作结果，不必每下都写疼、响、风、血。

## 入稿检查

提交前做一遍：

1. 主爽点是否唯一明确。
2. 篇幅是否符合场景等级。
3. 是否可删 10%-20% 而不损失剧情信息。
4. 是否有至少一处安静或停顿，让高潮更突出。
5. 是否把解释尽量藏进动作、规则和结算。
6. 结尾是否干净，不拖尾，不重复点题。
