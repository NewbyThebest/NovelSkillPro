# 大纲 (Outline)

## Usage
触发指令：`加载大纲`、`写大纲`、`大纲阶段`、`番茄大纲`

## 目标
将设定落地为可执行的方向大纲：开书阶段只生成全书总纲/方向大纲，不要求完整卷纲或批量章纲；真正进入正文时，再由正文技能按“写哪一章，补哪一章章纲”的方式即时生成章节施工图。重点锁定题材承诺、主线方向、阶段爽点、换图逻辑、前 30-50 章留存爆点与可动态调整的近期锚点，支持动态修订。

## 📌 阶段开始前：回顾预期

**【立即提示用户】**：
> 进入大纲阶段前，建议先打开 `1-边界/预期.md`，确认大纲方向与你的初始预期（题材边界、核心剧情执念等）是否一致，再开始规划。

---

> **流程原则**：每一步的输出都必须经用户确认后，再进入下一轮或写入文件。
> **设定回溯**：撰写大纲时若发现与既有设定不一致或需要细化，须回到 `2-设定/` 更新对应文件，再继续或落稿。设定服务于剧情，不是一成不变。

---

## 📋 设定回溯时机与范围

| 时机 | 建议检视并可能更新的设定文件 |
|------|------------------------------|
| **总纲完成后** | `2.2_新书设定案.md`（主线推演、地图路线图、换图逻辑与总纲对齐） |
| **进入新阶段/新地图前** | `2.2_新书设定案.md`（阶段目标、地图、关系博弈）、`2.3_金手指设定.md`、`2.4_主要角色设定表.md` |
| **每章正文前** | 由正文技能即时生成本章章纲；若本章即时章纲或正文结果反推总纲/设定需要调整，则只修订受影响的总纲锚点和设定文件。 |

**说明**：`2.1_创意脑暴.md` 为方向层，一般不在大纲阶段修改；仅当总纲与核心梗/类型边界严重冲突时再考虑回溯。

## 📚 步骤 1: 全书总纲规划 (Master Outline)

### 输入
- `2.2 新书设定案.md`（含**核心创作指导**）
- **`1.4 全书框架.md`（⚠️ 必读参考）** - 样板书的节奏模板

### 参考标准

> **🎯 强制对标原则**：新书的总纲必须在格式、结构、节奏密度上严格对标 `1.4 全书框架.md`。（如果有）

**包含的核心要素**：
1. **全书剧情单元总览**：按【地图 + 身份/阶级 + 核心目标】划分阶段
2. **核心节奏公式**：样板书的标准循环公式（如：发现危机→试错→重开→布局→收网）
3. **换地图逻辑**：何时换地图、为什么换（战力崩坏/资源枯竭/主动扩张）
4. **节奏密度统计表**：样板书各卷的字数占比、章节范围、核心事件密度
5. **番茄留存节点**：首屏承诺、前三章兑现、前 30-50 章追读爆点、可包装投流素材

### 执行方式
**交互式共创（Iterative Dialogue）** - 这是一场持续的深度对话

> **⚠️ 重要**：
> - 不要一次性生成完整总纲。必须逐项与用户讨论、确认后再继续。
> - 每一轮提议时，必须明确说明「如何对标 1.4 全书框架.md」。

### 动作流程

#### 前置步骤：读取并展示参考标准
- **AI 动作**：
  - 读取 `1-边界/1.4_全书框架.md`（如有）
  - 向用户展示样板书的：总卷数与字数分布、各卷核心主题、节奏密度统计表、换地图逻辑
- **目的**：让用户清楚看到样板书的结构模板

#### 第1轮：分卷数量与字数规划
- **AI 提议**：根据全书目标字数（询问用户）提出分卷方案，并对标说明
- **等待用户确认** → 确认后进入下一轮

#### 第2轮：各卷核心主题与节奏对标
- **AI 提议**：定义每一卷的核心主题，强制对标样板书节奏分布
- **等待用户确认** → 确认后进入下一轮

#### 第3轮：大事件锚点分配与换地图逻辑
- **AI 提议**：确立全书超级高潮点，分配到具体卷中，并说明换地图逻辑、追读钩子和情绪回报
- **等待用户确认** → 确认后进入下一轮

#### 第4轮：核心节奏公式提炼
- **AI 提议**：根据 `1.4 全书框架.md` 提炼新书的标准循环公式
- **等待用户确认** → 确认后进入下一轮

#### 第5轮：创作原则检查
- **AI 检视**：检查每一卷的核心高潮是否符合项目宪法
- **等待用户确认** → 确认后生成文件

### 输出
- `3-大纲/3.1_全书结构总纲.md`

#### 输出格式要求

**必须包含以下四大板块**（对标 `1-边界/1.4_全书框架.md`）：

1. **全书剧情单元总览**：表格形式（章节区间 | 地图 | 身份/阶级 | 核心目标 | 故事梗概）
2. **核心节奏公式**：新书的标准循环公式与可复用节奏模板
3. **节奏密度统计表**：各卷字数占比、章节范围、核心事件密度
4. **番茄留存设计**：首屏承诺、前三章兑现、前 30-50 章追读爆点、投流素材
5. **对新书的启示**：开局节奏、中期布局、后期格局

> **CRITICAL**: 必须将总纲注册到 `SOLOENT.md §4.2 Outline Index`：
> `[总纲] -> 3-大纲/3.1_全书结构总纲.md`

### 用户确认
- **【强制暂停】** 输出以下提示后，**禁止继续执行任何后续步骤，等待用户明确回复**：
  > 「已生成 `3-大纲/3.1_全书结构总纲.md`，请打开文件仔细阅读，确认全书结构与节奏符合预期后回复『继续』。」
- 收到用户『继续』（或等效肯定）后，方可进入步骤 2。

### 设定回溯（可选）
总纲落定后，检视 `2-设定/2.2_新书设定案.md` 中的主线推演、地图路线图、换图逻辑是否与总纲一致；若有细化或修正，则更新后再进入步骤 2（卷纲）。

## 📖 步骤 2: 当前阶段方向锚点规划 (Optional Stage Anchors)

### 目标
在不预写完整卷纲的前提下，为即将进入的阶段/地图提供方向锚点：核心矛盾、关键人物链、阶段爽点、风险边界和近期 3-5 章的追读问题。它只负责“方向”，不负责把整卷事件流排死。

### 触发时机
- 用户明确需要阶段规划
- 正文即将进入新地图、新身份、新大矛盾时
- 总纲方向足够，但下一阶段缺少可执行锚点时

### 参考依据
- `3-大纲/3.1_全书结构总纲.md`
- `1-边界/1.4_全书框架.md`（如有）
- `2-设定/2.2_新书设定案.md` - **重点参考核心创作指导**
- `1-边界/1.5_微观节奏拆解.md`（如有）

### 执行方式
**交互式共创（Iterative Dialogue）** - 这是一场持续的深度对话

> **⚠️ 重要**：本步骤可跳过。开书阶段不得要求用户先完成完整卷纲；如执行本步骤，只能生成当前阶段方向锚点，避免提前锁死后续章节。

### 动作流程

#### 第1轮：阶段核心设计
- **AI 提议**：当前阶段目标、核心矛盾（明暗双线）、主角短期目标、阶段爽点曲线和近期 3-5 章追读问题
- **等待用户确认** → 确认后进入下一轮

#### 第2轮：人物链
- **AI 提议**：本阶段的 BOSS/阻力源、盟友/渠道、关键配角及功能定位
- **等待用户确认** → 确认后进入下一轮

#### 第3轮：近期事件锚点
- **AI 提议**：只规划近期 3-5 章可能触及的事件锚点、反转方向、阶段爽点兑现和追读问题；不得细排整卷章纲
- **等待用户确认** → 确认后进入下一轮

#### 第4轮：金手指植入
- **AI 提议**：明确每个关键剧情节点主角如何利用金手指破局，精确到词条功能与代价
- **等待用户确认** → 确认后进入下一轮

#### 第5轮：质检与创作原则
- **AI 检视**：对照项目宪法确保符合项目专属创作指导
- **等待用户确认** → 确认后生成文件

### 输出
- 可选：`3-大纲/XX阶段_方向锚点.md`

> 若生成阶段锚点，必须注册到 `SOLOENT.md §4.2 Outline Index`：
> `[第XX阶段方向锚点] -> 3-大纲/XX阶段_方向锚点.md`

### 用户确认
- **【强制暂停】** 输出以下提示后，**禁止继续执行任何后续步骤，等待用户明确回复**：
  > 「已生成 `3-大纲/XX阶段_方向锚点.md`，请打开文件仔细阅读，确认当前阶段方向无误后回复『继续』。」
- 收到用户『继续』（或等效肯定）后，方可进行设定回溯或进入正文。

### 设定回溯
阶段锚点确认后，须检视并必要时更新：
- `2-设定/2.2_新书设定案.md`（阶段目标、地图、关系博弈等与当前方向一致）
- `2-设定/2.3_金手指设定.md`（本阶段金手指植入是否需补丁或新功能）
- `2-设定/2.4_主要角色设定表.md`（本阶段新配角、主角成长或关系变化）

更新完毕并经用户确认后，进入正文技能；章纲由正文技能在每章写作前即时生成。

## 🔨 步骤 3: 即时章纲说明 (Just-in-time Chapter Beats)

### 目标
本大纲技能不在开书阶段批量生成章纲。章纲改为正文阶段的前置动作：写第 X 章前，依据总纲/阶段锚点、设定、上一章正文、角色状态、伏笔进度和用户本章意图，生成并确认第 X 章即时章纲。

### 参考依据
- `3-大纲/3.1_全书结构总纲.md`
- `3-大纲/XX阶段_方向锚点.md`（如有）
- `1-边界/1.5_微观节奏拆解.md` - 场景与写法原子级对标（如有）

### 执行方式
**每章即时细化** - 写哪一章，只补哪一章章纲

> **⚠️ 重要**：禁止在开书阶段要求用户先完成完整卷纲或批量章纲。若用户明确要求批量规划，可以作为例外执行，但默认流程必须保持即时章纲。

### 动作流程

#### 步骤 1: 单章结构对标与场景拆解

**约束**：**单章字数锚定 2300-3300 字**，每章必须有明确推进、情绪波动和章尾追读点；开篇章纲必须标注前 200 字首屏抓力。

**AI 提议本章即时章纲**
- **章节范围**：只覆盖第 X 章
- **结构对标**：参考样板书的单章节奏和当前总纲/阶段锚点
- **本章即时章纲**：章节梗概、本章番茄目标、场景拆解、爽点/虐点、伏笔与线索、结尾钩子
- **场景拆解要求**：**尽可能多地拆解场景**，不要把一章写成两三个大场景

**等待用户确认本章章纲** → 确认后进入正文镜头稿与正文撰写

#### 步骤 2: 设定回溯与动态修正 (Feedback Loop)

**触发时机**：每次即时章纲确认后、正文完成后，按需执行。

**AI 检视**（以下设定文件若有与本章章纲或正文不一致、需细化的地方，须更新后经用户确认再继续）：
- **设定案检视**：本章剧情是否导致 `2.2_新书设定案.md` 需要补充或修正？
- **金手指检视**：现有金手指机制能否支撑本章爽点？若不能，需修改 `2.3_金手指设定.md`。
- **人设检视**：本章是否需要新配角？主角性格是否有成长？若有，需更新 `2.4_主要角色设定表.md`。

若发现本章即时章纲与总纲严重冲突，优先微调本章章纲；若冲突来自总纲方向错误，再跳转至 **步骤 5: 动态修订**。

**等待用户确认** → 确认后继续正文或进入下一章即时章纲。

**用户确认（最终产出）**：
- **【强制暂停】** 输出以下提示后，**禁止继续执行任何后续步骤，等待用户明确回复**：
  > 「第 X 章即时章纲已生成，请确认本章方向、场景和结尾钩子无误后回复『继续』进入正文。」
- 收到用户『继续』（或等效肯定）后，进入正文技能的镜头稿与正文撰写。

### 输出
- `3-大纲/XX卷_章纲.md`（按章追加；不存在则创建）

#### 输出格式示例

```markdown
第X章：[章节标题]

**章节梗概**：[一句话概括]

**本章番茄目标**：[首屏问题/冲突升级/爽点兑现/追读钩子]

**场景拆解**：
- 场景1：[发生了什么] + [写法指导：情绪基调、细节要求]
- 场景2：[发生了什么] + [写法指导：情绪基调、细节要求]
- 场景3：[发生了什么] + [写法指导：情绪基调、细节要求]
- 场景N：...

**伏笔与线索**：
- 埋入：[新伏笔内容]（预计第Y章收回）
- 推进：[推进的已有线索]
- 收回：[收回的伏笔]（呼应第Z章）

**结尾钩子**：[悬念点]
```

> **CRITICAL**: 必须将章纲注册到 `SOLOENT.md §4.2 Outline Index`：
> `[第XX卷章纲] -> 3-大纲/XX卷_章纲.md`

## 📌 步骤 4：书名与简介生成

**触发时机**：已有总纲，且至少完成当前卷的前三章章纲后，在进入写正文前可执行本步。

| 项目 | 内容 |
| :--- | :--- |
| **输入** | `2-设定/2.2_新书设定案.md`、`3-大纲/3.1_全书结构总纲.md`、第一卷前三章章纲 |
| **输出** | 多组「书名 + 简介」供用户选择 |

**动作**：
1. 读取上述三份材料，提炼核心卖点、类型与开局钩子。
2. 生成 **3–5 组** 候选「书名 + 简介」（简介每条约 100–200 字，突出卖点与期待感）。
3. 将结果写入 `2-设定/候选书名与简介.md`，并提示用户阅读，选定或修改后确定最终书名与简介。
4. 用户确认后，可将最终书名与简介同步到 `2.2_新书设定案.md` 或 SOLOENT.md §1。

**步骤 4 完成后**：
- **【强制暂停】** 输出以下提示后，**禁止继续执行任何后续步骤，等待用户明确回复**：
  > 「已生成 `2-设定/候选书名与简介.md`，请打开文件选定或修改书名与简介，确认后回复『继续』。」
- 收到用户『继续』（或等效肯定）后，同步最终书名与简介，再进入写正文或继续章纲。

## 🔄 步骤 5: 动态修订 (Dynamic Revision)

> **场景**：正文写作中涌现新灵感、剧情自然偏离原定大纲，或发现原大纲逻辑不通时使用。

### 目标
保持"大纲-正文"的一致性，确保后续剧情在新的逻辑上继续展开，防止崩书。

### 触发时机
- **每章正文完成后**：若实际剧情与本章即时章纲差异较大，按需回溯检视
- **正文创作里程碑**（阶段完结或每 10 章节点）：自动检视正文走向
- **用户主动指令**："修订大纲"、"剧情偏了"

### 执行流程

#### 情况 A：正文跑偏，回溯大纲 (Bottom-Up)
1. 读取最新完成的正文章节、总纲、阶段锚点（如有）和当前 `XX卷_章纲.md` 中已确认的即时章纲
2. AI 总结"原定走向"与"实际走向"的差异点
3. 推演蝴蝶效应（对本卷结局及后续卷的影响）
4. 执行修订：优先更新下一章即时章纲 → 必要时更新阶段锚点 → 再必要时更新总纲

#### 情况 B：总纲调整，重做细纲 (Top-Down)
1. 读取 `3.1_全书结构总纲.md` 和下一卷的旧大纲（如有）
2. 按"步骤 2：当前阶段方向锚点规划"重新生成下一阶段锚点
3. 确认后覆盖旧文件，并更新 SOLOENT 索引

### 输出
- 更新后的 `3-大纲/XX卷_章纲.md`
- 更新后的 `3-大纲/XX阶段_方向锚点.md`（如有）
- 更新后的 `3-大纲/3.1_全书结构总纲.md`

### 用户确认
- **【强制暂停】** 输出以下提示后，**禁止继续执行任何后续步骤，等待用户明确回复**：
  > 「大纲修订已完成，请打开对应文件确认修订内容无误后回复『继续』。」
- 收到用户『继续』（或等效肯定）后，方可继续创作。
