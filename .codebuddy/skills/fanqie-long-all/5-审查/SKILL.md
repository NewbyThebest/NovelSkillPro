# 审查 (Review)

## Usage
触发指令：`加载审查`、`审稿`、`审查阶段`、`番茄审查`

## 目标
在设定、大纲、正文等关键节点进行番茄向质量把控，防止问题累积；重点检查首屏留存、爽点兑现、章尾追读、题材适配与投流可包装性，并支持设定回溯。

## 📌 阶段开始前：回顾预期

**【立即提示用户】**：
> 进入审查阶段前，建议先打开 `1-边界/预期.md`，以初始预期为基准来审视当前产出，检查方向是否跑偏。

---

> **⚠️ 说明**：本质检流程面向番茄长篇网文，以下标准需结合具体题材、男频/女频方向与用户创作目标灵活调整。
> **原则**：早发现，早修正。设定服务于剧情，剧情服务于读者情绪，读者情绪服务于追读。

---

## 🛠️ 模块 1: 设定质检 (Setting Review)

**触发时机**：完成创意与设定流程，生成设定文件后。
**检查对象**：
- `2-设定/2.1_创意脑暴.md`
- `2-设定/2.2_新书设定案.md`
- `2-设定/2.3_金手指设定.md`
- `2-设定/2.4_主要角色设定表.md`

**质检标准 (Checklist)**：

1. **逻辑自洽性 (Internal Logic)**
   - [ ] **力量体系闭环**：金手指的代价与收益是否平衡？后期是否会战力崩坏？
   - [ ] **世界观支撑**：地图广度与等级上限是否匹配？换地图逻辑是否顺畅？

2. **爽点支撑度 (Pleasure Points)**
   - [ ] **金手指独特性**：是否只有主角能用？是否具备"人无我有，人有我优"的绝对优势？
   - [ ] **期待感设计**：金手指的进化路线是否清晰？是否有足够的升级期待？
   - [ ] **前期兑现频率**：前三章是否能看到承诺或初次效果？前 30-50 章是否有持续回报？

3. **番茄适配度 (Fanqie Fit)**
   - [ ] **题材卖点**：一句话能否说清本书爽点/虐点/关系爆点？
   - [ ] **投流素材**：是否有适合做书名、简介、导语和短视频投放的强情绪场面？

4. **利益驱动逻辑 (Motivation)**
   - [ ] **主角驱动力**：主角的目标是否足够强烈（生存/复仇/长生）？
   - [ ] **反派智商**：反派的行为是否符合其利益最大化？拒绝无脑送经验。

5. **人物立体度 (Character Depth)**
   - [ ] **反差萌/记忆点**：主角和核心配角是否有独特的标签或反差（A面/B面）？

**执行动作**：
- 若发现问题，直接在对话中指出，并协助用户修改对应设定文件。
- 修改后，须再次确认是否引入新矛盾。
- **【强制暂停】** 质检完成后，**禁止继续执行任何后续操作，等待用户明确回复**：
  > 「设定质检已完成，请查看以上问题与建议。确认修改完毕或选择忽略后回复『继续』。」
- 收到用户『继续』（或等效肯定）后，方可进入下一步（加载 3-大纲）。

## 📋 模块 2: 大纲质检 (Outline Review)

**触发时机**：完成大纲规划流程，生成总纲、卷纲或章纲后。
**检查对象**：
- `3-大纲/3.1_全书结构总纲.md`
- `3-大纲/XX卷_完整卷纲.md`
- `3-大纲/XX卷_章纲.md`

**质检标准 (Checklist)**：

1. **节奏密度 (Pacing)**
   - [ ] **黄金节奏**：是否形成 5-10 章短循环、20-30 章阶段高潮？
   - [ ] **垃圾时间**：是否存在连续 2 章以上的铺垫或说明性文字？（须压缩或删除）
   - [ ] **前期留存**：前 30-50 章是否有足够密集的反转、收益、关系爆点或目标升级？

2. **期待感管理 (Hooks)**
   - [ ] **钩子有效性**：每一章（特别是章末/卷末）是否留下了强悬念、强危机、新收益或反转？
   - [ ] **伏笔埋设**：高潮前的铺垫是否足够？是否有生硬的机械降神？

3. **爽点逻辑 (Payoff)**
   - [ ] **情绪释放**：压抑后的爆发是否足够酣畅？
   - [ ] **主角作用**：核心转折点是否由主角主导？（拒绝主角划水）
   - [ ] **结算感**：阶段高潮后是否有明确收获、打脸、升级或关系推进？

4. **一致性检查 (Consistency)**
   - [ ] **设定对齐**：大纲剧情是否与 `2.2_新书设定案.md` 中的主线推演一致？
   - [ ] **战力检查**：本卷BOSS战力是否在主角当前等级的可应对范围内？

**执行动作**：
- 模拟【主编】视角，对大纲进行辣评。
- 提出具体的修改方案（如：调整事件顺序、增加危机感、强化金手指作用）。
- **【强制暂停】** 质检完成后，**禁止继续执行任何后续操作，等待用户明确回复**：
  > 「大纲质检已完成，请查看以上问题与建议。确认修改完毕或选择忽略后回复『继续』。」
- 收到用户『继续』（或等效肯定）后，方可进入下一步（加载 `.agents/skills/fanqie-long-zhengwen/SKILL.md`）。

## ✍️ 模块 3: 正文质检 (Draft Review)

**触发时机**：完成正文写作流程，生成单章正文草稿后。
**检查对象**：`4-正文/第X章_草稿.md`

**质检标准 (Checklist)**：

1. **黄金三章法则 (Opening)**（仅限开篇）
   - [ ] **切入点**：是否开篇即危机/冲突？（拒绝慢热）
   - [ ] **金手指/核心卖点**：金手指或核心卖点是否在合适时机（通常前3章）出现、承诺或发挥作用？
   - [ ] **代入感**：主角的处境是否能引起读者共情？

2. **沉浸感 (Immersion)**
   - [ ] **Show, Don't Tell**：是否用动作、对话、细节描写代替了枯燥的设定说明？
   - [ ] **五感描写**：是否调动了视觉、听觉、嗅觉等感官体验？

3. **情绪调动 (Emotion)**
   - [ ] **爽点/憋屈点**：读完本章，读者是感到爽、期待，还是憋屈？（憋屈不能过夜）
   - [ ] **装逼打脸**：如果有装逼情节，是否铺垫到位？打脸是否响亮？
   - [ ] **章尾追读**：结尾是否制造“下一章必须看”的问题、危机、反转或收益期待？

4. **文风与红线 (Style & Constraints)**
   - [ ] **AI味去除**：检查是否存在翻译腔、重复词汇、过度书面语。

**执行动作**：
- 模拟【番茄读者】或【资深编辑】视角，进行段落级批注。
- 重点标注"出戏"、"啰嗦"、"逻辑不通"的地方，并提供改写示范。
- **【强制暂停】** 质检完成后，**禁止继续执行任何后续操作，等待用户明确回复**：
  > 「正文质检已完成，请查看以上批注与建议。确认修改完毕或选择忽略后回复『继续』。」
- 收到用户『继续』（或等效肯定）后，方可继续下一章 `.agents/skills/fanqie-long-zhengwen/SKILL.md` 循环。

## 🔄 模块 4: 设定回溯 (Feedback Loop)

**定位**：防止"剧情设定两张皮"，确保设定随剧情动态演进。

**触发时机**：
- 正文/大纲写作中涌现新灵感，与旧设定冲突时。
- 发现旧设定无法支撑当前剧情爽点时。

**执行动作**：
1. **更新 Memory Bank**：
   - 补充新配角 -> `SOLOENT.md §3` & `§7.2`
   - 修正地图 -> `SOLOENT.md §2`
   - 更新伏笔 -> `SOLOENT.md §7.4`
2. **更新设定文档**：
   - `2-设定/2.2_新书设定案.md`（主线/地图）
   - `2-设定/2.3_金手指设定.md`（补丁/进化）
   - `2-设定/2.4_主要角色设定表.md`（性格成长/关系变化）
