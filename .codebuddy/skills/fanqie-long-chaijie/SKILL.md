# 番茄边界确定 (Boundary) — 样板书拆解


## Usage
触发指令：`加载边界确定`、`边界阶段`、`样板书拆解`、`番茄边界`、`番茄样板书拆解`
---

## 目标
有样板书时，拆解其文风、套路、首屏钩子、爽点循环、追读结构、题材边界与投流素材，产出 `1-边界/` 下系列文件，为后续番茄长篇创意与设定提供边界参考。

---

## 初始化（最先执行）

在**项目根目录**创建目录 `1-边界`（若不存在）。

---

## 📖 步骤 0：准备工作

| 项目 | 内容 |
| :--- | :--- |
| **输入** | 用户提供的样板书文件路径（或已放入 `1-边界/` 的约定） |
| **动作** | 1）确认样板书在 `1-边界/`，格式为 `.txt`、编码正确；2）若尚未有该目录，先创建 |
| **输出** | 无；为后续步骤准备好可读的样板书路径 |

## 📊 步骤 1：故事梗概提取

| 项目 | 内容 |
| :--- | :--- |
| **输入** | 样板书 `.txt` 路径（步骤 0 已确认） |
| **输出** | `1-边界/1.1_全书故事梗概.md`（全书或开篇梗概） |

**⚠️ 执行前须先与用户确认**：本步有两种方式，**二选一**。在跑脚本或读原文之前，必须先明确询问用户：
- **方式 A**：用脚本批量生成全书梗概（需配置 API，有 Token 成本）；
- **方式 B**：由 Agent 读取样板书前 N 章生成开篇梗概（无需 API）。
待用户选择后，再按对应方式执行。无 API 或用户不想用脚本时用方式 B；后续步骤 3、4 可照常进行（基于开篇梗概 + 分段读原文）。

### 方式 A：脚本批量生成全书梗概（需 API）

按以下顺序在**对话中**完成，不要跳过步骤。

**1. 提醒用户注意**
- 样板书很长时 Token 消耗大，可优先处理黄金三章、前 30-50 章和代表性高潮单元，或选用低成本 API。
- 使用 `.env` 配置 API 最安全：密钥仅存于本地文件、不会出现在命令行或聊天记录中，也不会被提交到仓库。

**2. 若用户尚未配置 API：先让脚本创建 `.env` 模板**
- 在**项目根目录**执行下面的命令（可由 AI 通过 Shell 代为执行一次）：
  ```bash
  python3 .agents/skills/fanqie-long-chaijie/scripts/summarize.py 1-边界/[书名].txt
  ```
- 脚本检测到缺少 API Key 时，会**自动在项目根创建 `.env` 文件**（模板）。
- 执行后，**在对话中明确提示用户**：「已在项目根创建 `.env` 文件，请打开该文件（路径：`项目根/.env`）查看并准备填写。」

**3. 提示用户打开并填写 `.env`**
- 在对话中说明：请打开项目根目录下的 **`.env`** 文件，在 `NOVEL_API_KEY=` 后面填入你的 API 密钥；如需更换服务商，可改 `NOVEL_API_BASE` 和 `NOVEL_API_MODEL`。**仅限本地使用，请勿将 `.env` 提交到仓库。**
- 请用户填写并保存后，在对话中回复「已填写」或「没问题」。

**4. 确认后执行脚本**
- 确认样板书已放入 `1-边界/` 下，且用户已确认 API 填写无误。
- **由 AI 在项目根代为执行脚本命令**（或给出命令供用户执行）：
  ```bash
  python3 .agents/skills/fanqie-long-chaijie/scripts/summarize.py 1-边界/[书名].txt
  ```
- 脚本会从项目根或 `.novelkit/.api.env` 读取 `.env` 配置，请勿在命令行中填写或粘贴 API Key。

**结果**：生成 `1-边界/1.1_全书故事梗概.md`。

### 方式 B：Agent 生成开篇梗概（无需 API）
- **执行**：不跑脚本，由 Agent 读取样板书前 N 章（如前 15 章或前 3000 行），在对话中生成开篇梗概。
- **输出**：写入 `1-边界/1.1_全书故事梗概.md`，可在文件中注明「开篇梗概，非全书」。后续若用户提供 API，可再跑脚本覆盖或追加为全书梗概。

### 步骤 1 完成后
- **【强制暂停】** 输出以下提示后，**禁止继续执行任何后续步骤，等待用户明确回复**：
  > 「已生成 `1-边界/1.1_全书故事梗概.md`，请打开文件校验是否有明显断点或缺章，确认无误后回复『继续』。」
- 收到用户『继续』（或等效肯定）后，方可进入步骤 2。

## 📊 步骤 4：全书框架与节奏拆解

| 项目 | 内容 |
| :--- | :--- |
| **输入** | `1-边界/1.1_全书故事梗概.md`；可选：样板书章节标题列表（用于统计节奏） |
| **输出** | `1-边界/1.4_全书框架.md` |

- **输出格式（固定四大板块）**：

  1. **全书剧情单元总览**
     - 按【地图 + 身份/阶级 + 核心目标】复合维度划分阶段
     - 以表格形式呈现：章节区间、地图、身份、目标、故事梗概
     - **换地图逻辑**：分析为什么要换地图（战力崩坏/资源枯竭/主动扩张）

  2. **核心节奏公式**
     - 提炼出本书的标准循环公式
     - 例：发现危机→试错→重开→布局→收网
     - 总结可复用的叙事节奏模板，尤其是 5-10 章短循环与阶段爽点兑现

  3. **换地图逻辑分析**
     - 作者是何时换地图的？（主动 vs 被动）
     - 换地图的前兆是什么？（资源枯竭/战力崩坏/追杀）

  4. **对新书的启示**
     - **开局节奏**：如何在前 200 字建立危机感、题材承诺和主角处境
     - **中期布局**：如何维持 5-10 章短循环与 20-30 章阶段高潮
     - **后期格局**：如何实现身份跃迁、权限升级和持续追读期待

  5. **番茄留存与投流素材**
     - 前三章承诺与兑现点
     - 前 30-50 章追读爆点
     - 可用于书名、简介、导语或短视频投流的强情绪场面

- **禁止事项**：不要画曲线图等冗余内容，专注以上四大核心板块。

## 🔍 步骤 5：微观节奏与剧情单元拆解

| 项目 | 内容 |
| :--- | :--- |
| **输入** | 样板书指定章节正文（前三章 + 一个完整高潮单元约 3–10 章）、可选 `1.1_全书故事梗概.md` 用于选段 |
| **输出** | `1-边界/1.5_微观节奏拆解.md` |

### 子步骤 5.1：单剧情单元拆解

#### 目标
分析作者如何用几章的篇幅讲好一个完整的小故事（小高潮）。

#### 选材方法
1. **先看梗概**: 浏览 `1.1_全书故事梗概.md`
2. **寻找样本**: 字数占比适中（约3-6章）、起承转合明显的完整事件
3. **推荐样本**: 优先选择“新手村第一次大危机/大收获”（通常在第10-20章左右）
4. **提取正文**: 根据梗概锁定的章节号，去样板书原文中提取对应章节

#### 分析模型：标准网文四步法

**1. 起 (Setup)** — 欲望与阻碍、拉仇恨、给承诺  
**2. 承 (Build-up)** — 试探与压抑、情绪积蓄、制造追读问题  
**3. 转 (Climax)** — 爆发与反转、释放、主角主导破局  
**4. 合 (Resolution)** — 收获与余波、落袋为安、抛出下一钩  

### 子步骤 5.2：单章精密拉片——以「黄金三章」为例

- **输入**：样板书第 1、2、3 章（逐章分析）
- **分析维度**：前 200 字首屏钩子、题材承诺、主角处境、信息密度控制、场景结构（场景组合/写法/镜头语言/缝合/视角游移）、爽点兑现、断章钩子
