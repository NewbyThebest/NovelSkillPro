# 番茄新手避雷

把番茄课堂《新手写作五大避雷针》以及相关新手避坑课，转成一个可执行的诊断与改稿流程，重点解决新手最常见的问题：

1. 题材与内容脱节
2. 题材缺乏辨识度
3. 跨题材强行混搭
4. 设定看似新，实际只是旧框架换壳
5. 人称选择和人称混乱
6. 流水账
7. 开篇太寡淡
8. 开篇太复杂
9. 人设扁平化或前后不统一
10. 世界观模糊或强行灌输
11. 视角杂乱或叙事方式不当
12. 主线不清、剧情混乱、爽点突兀
13. 故意秀文笔、词藻堆砌、排版不规范
14. 钩子、书名、简介不够抓人
15. 过度准备、越准备越不敢写
16. 写到中段开始崩文、视角乱、读者差评集中

## 适用场景

当用户出现下面这些需求时触发：

- 我适合写什么题材，不知道该不该追热点
- 这个设定是不是太老了，怎么做微创新
- 这个设定够不够新，会不会只是换皮
- 我准备得很多，但越写越不顺
- 为什么我一写就崩，或者越写越乱
- 让我按番茄课堂的方法看看这段有没有问题
- 这段是不是流水账
- 帮我把开头改得更上头
- 看看我这个书名和简介能不能点进去
- 按番茄新手避雷标准审一下
- 帮我把这一章改得更像番茄平台读者会吃的写法

## 输入类型

优先识别用户要处理的是哪一类材料：

- 开篇正文
- 单章正文
- 书名 + 简介
- 故事设定 / 人设 / 大纲
- 写作卡点 / 差评反馈 / 崩文症状
- 选题方案 / 题材定位 / 标签方向
- 用户只描述问题，没有给正文

如果输入不足以落地，就先点明缺口，再告诉用户最少需要补什么。

## 工作流

### 1. 先判断任务模式

先判断本次任务属于哪一种：

- `诊断模式`：判断问题、指出雷点、给修改方向
- `改稿模式`：直接改写已有文本
- `生成模式`：基于设定生成书名、简介、开头方案、钩子方案
- `混合模式`：先诊断，再给改稿样例

如果用户没明确说，只要给了现成文本，默认走 `混合模式`。

### 2. 按避雷清单逐项排查

检查时使用 [docs/CHECKLIST.md](docs/CHECKLIST.md) 中的标准，不要泛泛而谈，必须落到具体文本。

尤其注意：

- 题材、书名、标签和正文承诺是否一致
- 作品是否有足够明确的题材辨识度，而不是写成泛类型
- 设定是否保留了有效期待线，同时做了可感知的微创新
- 创新是不是安全创新，还是把读者原本会吃的点一起推翻
- 跨题材混搭有没有逻辑纽带，还是只是把元素硬拼在一起
- 人称是否统一，第一人称是否真的适合长篇
- 情节是不是只在记流水，而没有冲突或强情绪
- 开篇是不是把最无聊的准备段落放在最前面
- 开篇场景、人物、故事线是不是过多过杂
- 人设是不是只能靠标签和台词撑着，缺少事件中的行为体现
- 世界观是不是一上来猛灌设定，而不是跟着行动自然露出
- 视角是不是乱飞，叙事方式是不是把新手自己也绕进去了
- 主线目标是否明确，剧情是否有阻碍、递进和合理铺垫
- 用户当前的“准备工作”是在帮助开书，还是已经反过来压制落笔
- 差评、崩文、视角混乱究竟是包装问题、结构问题，还是叙事控制问题
- 文风是不是在讲故事之外，额外展示作者自己，排版是否影响手机阅读
- 每章末尾、每个关键场景后，是否留下继续读下去的理由
- 书名是否一眼能看出题材和卖点，简介是否像宣传语一样勾人

### 3. 输出方式

#### 诊断模式

输出顺序固定为：

1. `一句话总评`
2. `命中的雷点`
3. `最影响读者继续看的 1-3 个问题`
4. `修改方向`
5. `如有必要，给一小段改写示例`
6. 如果用户给了差评反馈或崩文症状，再补 `问题归因`：包装 / 设定 / 视角 / 节奏 / 主线

建议使用下面的表格格式：

| 项目 | 结论 | 说明 |
|---|---|---|
| 题材契合 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 题材辨识度 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 跨题材混搭 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 微创新 / 安全创新 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 人称 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 流水账 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 开篇抓力 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 开篇复杂度 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 人设 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 世界观 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 视角/叙事 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 主线/剧情 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 文风/排版 | 通过 / 预警 / 命中 | 只写最关键的问题 |
| 钩子/书名/简介 | 通过 / 预警 / 命中 | 只写最关键的问题 |

#### 改稿模式

改稿时不要只做字面润色，必须优先处理结构问题：

1. 先补冲突或强情绪
2. 再修正题材偏差和核心承诺
3. 再压缩无效说明
4. 再收束人物、场景、设定信息和视角数量
5. 再拉直主线与事件因果
6. 再补章节末尾钩子
7. 最后再处理措辞精简和排版

#### 生成模式

如果用户要求生成书名、简介、开头方案或钩子方案：

- 书名：给 5 组，优先简单直接、题材明确、卖点前置
- 简介：给 3 组，分别偏 `主线概括型`、`冲突开场型`、`片段小剧场型`
- 开头方案：给 3 个开篇切口，优先“从出事处切入”，不要从砌第一块砖开始；优先小而狠、接地气、容易代入的人性冲突
- 钩子方案：给每章末尾可落地的小卡点，不写空泛口号

如果用户要求判断 `该写什么题材`：

- 先区分 `大题材` 和 `细分梗`
- 优先建议新手从自己长期爱看、爱聊、能持续产出灵感的大题材入手
- 不默认鼓励盲追热点，除非用户明确对热点题材本身就有兴趣和积累
- 输出时至少给出 `首选题材 / 可尝试题材 / 暂不建议题材` 三档

如果用户要求判断 `设定是否有拉新感 / 是否够新`：

- 先判断市场上是否已有大量同类直写
- 如果框架已老，优先寻找 `微创新`，而不是强行推翻全部框架
- 微创新重点看：身份变化、关系变化、信息差变化、掉马方式变化、目标变化
- 安全创新的底线：保留读者熟悉的期待线，再替换其中一个关键变量
- 输出时至少说明 `保留的期待线` 和 `新增的新鲜点`

如果用户问的是“这个题材还能不能写”：

- 不直接回答能或不能，先拆成 `底盘期待线` 和 `当前变化点`
- 如果底盘还成立，但变化点太弱，就优先补变化点
- 如果变化点很花，但底盘期待线被掏空，就优先把读者熟悉的爽点、情绪点、关系点补回去

如果用户问的是“为什么总觉得准备了很多却写不出来”：

- 明确告诉用户：大纲、设定、人设表是辅助，不是枷锁
- 如果准备工作已经开始替代真正写作，就先压缩为 `主线一句话 + 第一卷目标 + 开篇切口`
- 允许先写一小段高压场景，再反推修正大纲和人物表
- 不鼓励为了追求完整，把开篇写成说明书

### 4. 改稿原则

- 不把“问题”说成抽象概念，必须指出是哪一句、哪一段、哪一类安排在拖后腿
- 不为了“有文采”牺牲可读性
- 不强行把所有问题一次改完，先改最影响点击和追读的问题
- 如果用户已经写崩，不要先安慰式泛谈，先判断是 `设定失速`、`视角失控`、`主线断裂` 还是 `爽点承诺失约`
- 如果用户是番茄平台导向，优先保点击、保读下去、保节奏，不走纯文学化处理
- 如果用户的问题已明显转成开篇重做、单章重做或人物塑造，不要硬留在本技能里，及时分流

## 与仓库内其他技能的协同

- 做选题定位、题材方向判断时，可先配合 `wuhang-long-ideation`
- 只审开篇质量时，可配合 `golden-three-chapters-review`
- 如果确认问题集中在开篇切入、钩子、前三章布局，可转交 `fanqie-opening-hook`
- 如果确认问题集中在节奏、冲突、爽点，可转交 `fanqie-pacing-conflict`
- 如果确认问题集中在人物总像工具人、情绪反应假、配角无魂，可转交 `fanqie-character-vitality`
- 如果确认问题集中在倒叙、循环、并行线、多时间层的组织问题，可转交 `fanqie-timeline-architecture`
- 只做书名简介时，可配合 `wuhang-long-title-intro`
- 进入正式正文写作时，继续遵循项目里的反 AI 表达、反 AI 韵律、人物真实性规则
