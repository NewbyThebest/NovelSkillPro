# 番茄时间线架构

这是一个只处理 `时间线结构` 的技能，重点不是文风，而是帮助用户决定：

- 该不该顺叙
- 该不该倒着讲
- 循环、读档、圆环、多线并行该怎么选
- 这种结构会不会把读者绕晕

## 适用场景

- 时间线怎么设计
- 倒叙会不会太乱
- 循环文怎么排信息
- 读档流怎么避免重复
- 多线并行怎么控节奏
- 这个结构适不适合番茄连载

## 输入类型

- 故事设定
- 大纲
- 卷纲
- 已写章节
- 用户想尝试的结构类型

## 核心任务

1. 先判断故事真正需要哪类时间线
2. 避免为了花而花
3. 确保信息释放顺序服务追读
4. 让复杂结构也能保住清晰度和钩子

## 五类常用时间线

### 1. 顺叙时间线

适合：

- 大多数长篇网文
- 升级、冒险、成长、经营等强调过程推进的故事

优点：

- 最稳
- 最清楚
- 最适合持续追更

风险：

- 容易写成平推流水

补法：

- 用阶段目标、节点爆点、阶段结算来撑节奏

### 2. 圆环时间线

含义：

- 开头先抛一个高压结果
- 中间回到起点，解释角色如何走到那里
- 结尾与开头形成照应或闭环

适合：

- 结果极有吸引力
- 结局画面本身就能拉出巨大好奇

风险：

- 开头抛得太大，后续解释跟不上

### 3. 读档制时间线

含义：

- 主角可以保留部分认知、经验或资源，重来某段过程

适合：

- 试错、破局、逆天改命、战术成长

风险：

- 重复感重
- 代价感不足

补法：

- 每次读档都必须带来新信息、新选择或新代价

### 4. 纯循环时间线

含义：

- 角色不断困在同一段时间中，核心爽点在于破局和认知升级

适合：

- 强悬念
- 强规则
- 强解谜

风险：

- 容易写成重复执行

补法：

- 每轮循环至少改变一个关键变量：认知、人际、资源、风险、规则理解

### 5. 多线并行时间线

含义：

- 同时推进两条及以上重要叙事线

适合：

- 群像
- 权谋
- 战争
- 大格局追查

风险：

- 切线太勤，主线感消失
- 各线信息价值不均

补法：

- 每条线都必须回答：它在推进什么、制造什么悬念、何时与主线交汇

## 选型流程

### 1. 先问四个问题

1. 这个故事最强吸引力在 `过程` 还是 `结果`
2. 主角是否需要反复试错
3. 读者的核心快感来自 `清晰推进` 还是 `结构解谜`
4. 连载时，复杂结构会不会压垮追读清晰度

### 2. 默认判断

- 新手优先顺叙
- 只有当结构本身能显著增强吸引力时，才上圆环、循环、读档、多线并行

### 3. 清晰度底线

不管哪种结构，都要保证读者随时知道：

- 现在在哪个时间层
- 这段目标是什么
- 切线或回溯为什么现在必须看

## 输出方式

### 诊断模式

1. `当前时间线类型`
2. `它适不适合这个故事`
3. `最大风险是什么`
4. `该保留、该删减、该改顺的部分`

### 设计模式

1. `推荐时间线类型`
2. `为什么选它`
3. `前三个关键节点怎么排`
4. `信息什么时候放`
5. `怎么避免读者看晕`

### 重构模式

如果用户已写乱，优先输出：

1. `哪几处时间跳转最伤理解`
2. `新的时间层顺序`
3. `哪些信息前置，哪些后置`
4. `一个更稳的章节推进法`

## 改稿原则

- 结构必须服务吸引力，不服务作者炫技
- 番茄连载场景下，清晰度优先级很高
- 所有复杂时间线都要靠明确节点和重复提示维持可读性
- 如果复杂结构没有明显增益，就退回更直接的讲法

## 与其他技能协同

- 开篇结构和第一页抓手问题，可接 `fanqie-opening-hook`
- 中段节奏、高潮余波、单章拖沓问题，可接 `fanqie-pacing-conflict`
- 如果问题还混着题材、人称、视角、新手通病，可先接 `fanqie-newbie-guardrails`
