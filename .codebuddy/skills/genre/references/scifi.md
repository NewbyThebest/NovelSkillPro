# 番茄科幻题材

## 一、边界与易混淆

服务科幻。适用：

- 黑科技文
- 星际文
- 软科幻升级文
- 科技文明升级

如果主轴已经变成末世求生，转 `apocalypse.md`。

## 二、流派/分类选型

两条常用主线：

1. **黑科技都市**：科技升级 → 社会变化 → 力量碾压
2. **星际扩图**：文明升级 → 开新星球 → 新种族 / 新副本

## 三、核心爽点/抓手

科幻的核心是 **新奇感 + 把新东西转成升级爽点**。不需要把学术解释写成正文，但科技规则必须自洽。

- 让读者看到新东西
- 让这些新东西形成升级爽点

不要用学术说明或术语堆砌替代剧情；现实合理服务于科技树的自洽和读者预期，不必写成论文。

## 四、开篇切口

- 用新奇的科技/设定事件切入，快速让读者看到"这书有别处没有的东西"。
- 星际文要尽快开出有差异的地图，别只有背景没有地图差异。

## 五、人设与金手指

科技树本质就是玄幻里的升级体系，至少要有：

- 层级
- 前置科技
- 解锁回报
- 力量转化
- 新地图

每一级科技升级都要能兑现成读者可感的变强。

## 六、常见误区

- 只会堆术语，不会转成爽点
- 科技升级后读者感受不到变强
- 星际只有背景，没有地图差异
- 用术语和资料考据替代科技升级带来的具体变化

## 七、题材特有约束

- **科技树要自洽**：前置—解锁—回报的链条别断，读者能顺着往下预期，才有升级的确定爽感。

## 设定阶段必备维度

按顺序确认：

1. 核心科技或科学前提，哪些规则需要读者理解，哪些只作为背景。
2. 前置技术、解锁条件、实际回报和升级后的可见变化。
3. 能源、材料、生产、组织或文明资源如何限制科技扩张。
4. 科技改变社会、职业、伦理和权力结构的具体方式。
5. 地图、文明或敌我层级的扩展逻辑，以及科技树不能越过的边界。
