# 节奏控制 (Pacing Control)

## 触发条件 (Trigger Scenarios)
当用户提出以下需求时，主动调用本技能：
- 询问故事进展是否“太快”、“太慢”、“太水”或“太拖沓”。
- 提到读者反馈“看累了”、“审美疲劳”，需要调整剧情模块。
- 要求规划一个大剧情/大事件的章节分配和字数比例（起承转合）。
- 连续写了多场相似剧情（如连续战斗），需要建议如何过渡、收尾或插入日常缓冲。

## 核心理念 (Core Concepts)

在辅助用户调整小说节奏时，必须遵循以下原则：
1. **节奏的本质是管理读者的疲劳度**：最高级的节奏是在“读者大脑的疲惫曲线上跳舞”。不能一直高潮，也不能一直平淡。
2. **黄金阅读周期**：读者完整的阅读周期约2万字（7-14章）。必须在这个篇幅内完成一个有效的情绪起伏循环。
3. **防同质化**：连续使用相同的剧情模式（如连续打比赛、连续下副本）会迅速积累疲劳，必须交替使用不同的“剧情块”。

## 节奏核心结构：起承转合 (The 4-Act Arc)

当用户提供一段大纲或数章内容要求分析节奏时，将其映射到以下结构中检查比例和功能：

1. **起（开局与铺垫，1~3章）**
   - **功能**：交代主角当前面临的极端压力、核心目的，建立读者的预判。
   - **节奏要求**：必须紧凑。超过三章还没把压力和目标写清楚，就会显得拖沓。
2. **承（发展与挣扎，4~14章）**
   - **功能**：展现主角在压力下的挣扎、收集资源、逐步成长。
   - **节奏要求**：篇幅最长，负责积累情绪和剧情张力。
3. **转（高潮与爆发，1~3章）**
   - **功能**：剧情爆发、反转。主角解决难题，完成身份跃迁或能力展现。
   - **节奏要求**：释放积压的情绪，给读者带来极致的“爽感”。
4. **合（收尾与缓冲，2~4章）**
   - **功能**：整理战利品、交代余波、处理配角互动，引出下一次事件。
   - **节奏要求**：情绪平缓落地，但必须埋下“钩子”（悬念）为下一个“起”做准备。

## 剧情块交替策略 (Plot Module Mixing)

为了防止读者疲劳，建议用户在“起承转合”的框架内交替使用以下剧情块：
- **竞技场**：明牌博弈，硬实力对拼（适合“转”部）。
- **大逃杀**：资源匮乏下的逃亡与反击（适合“承”部求生或“转”部反杀）。
- **迷雾局**：悬疑探秘，敌暗我明（适合长篇幅铺垫）。
- **基建局/种田**：发展实力，极强的反馈感。
- **欺诈局**：心理博弈，做局欺骗。
- **公路游**：转移地图，温馨日常（极佳的缓冲模块，适合放在两个宏大事件之间防通胀）。

*注意：幕间剧（反派视角等）绝对不可插在“转”与“合”之间打断高潮，必须放在“合”之后作为下一个事件的预警。*

## 节奏分析与调整工作流 (Workflow)

当用户要求分析或调整节奏时，按以下步骤执行，并严格按照要求的格式向用户输出：

1. **结构拆解**
   - **动作**：将用户提供的内容按“起、承、转、合”进行拆解，指出各部分所占的篇幅比例。
   - **输出**：一份【结构映射表】，列出当前剧情在“起承转合”各阶段的篇幅比例和核心事件。
2. **疲劳度诊断**
   - **动作**：分析是否铺垫太长、是否缺乏缓冲、剧情块是否同质化。
   - **输出**：一段【疲劳度诊断报告】，明确指出哪里容易让读者感到拖沓或疲劳（如：“起”超过3章，或连续高潮缺乏“公路游”缓冲）。
3. **提供调整方案**
   - **动作**：根据诊断结果，建议在哪里砍掉冗余，在哪里插入缓冲日常，或在哪里更换剧情块类型。
   - **输出**：一份【节奏优化建议清单】，具体到建议替换的“剧情块”类型（如建议将某段改为“公路游”缓冲或“迷雾局”铺垫）。

## 节奏自查清单 (Validation Checklist)

在输出节奏调整建议后，使用以下清单进行自我校验：

- [ ] “起”的部分是否在3章内明确了核心压力和目标？
- [ ] “承”的部分是否形成了资源消耗与成长的逻辑闭环？
- [ ] 剧情块的切换是否平滑？（上一模块的战利品/代价，是否是下一模块的诱因？）
- [ ] 宏大叙事之间是否插入了微观/日常叙事来缓解紧绷感？
- [ ] 每一段剧情结束（合）时，是否留下了足够的“钩子”拉动阅读期待？
