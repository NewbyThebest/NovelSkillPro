## 触发条件 (Trigger Scenarios)
当用户提出以下需求时，主动调用本技能：
- 要求“创建一个主角”、“构思男主/女主的人设”。

## 核心理念 (Core Concepts)

在辅助用户设计小说主角时，必须遵循以下“工程学”创作原则：
1. **角色比剧情更重要**：读者最终记住的是角色。设置好角色就是设置好读者。
2. **唯一接口**：主角是读者体验故事世界的唯一载体和外壳。
3. **工程学思维**：主角设计不是纯粹的感性艺术，而是可量化、可拆解的逻辑结构。

## 主角设计三要素 (The 3 Core Elements)

构建主角时，必须清晰定义以下三个核心支柱，这是建立读者信任（理解 -> 代入 -> 认同）的关键：

1. **欲望 (Desire)**：主角想要什么？（驱动力，让读者知道主角的目标）
2. **难点 (Difficulty)**：主角面临什么阻碍？（困境与冲突，让读者产生共情）
3. **成就 (Achievement)**：主角获得了什么？（成长与爽点，让读者感受到价值）

*进阶设定（视需求添加）：*
- **底色**：幽灵（过去的创伤/潜意识动机）与谎言（主角自以为是的错误认知）。
- **四大原型**：攀登者（求道）、复仇者（公义）、归乡者（羁绊）、守护者（秩序）。

## 工作流与 SOLOENT.md 指针联动 (Workflow & Pointer Linkage)

当用户要求创建一个主角或生成角色文档时，**必须**严格执行以下联动工作流，并向用户输出对应内容：

### 第一步：概念构思与确认
- **动作**：根据用户提示，构思主角的核心三要素（欲望、难点、成就）及进阶设定。
- **输出**：在对话框中向用户展示【主角设定草案】，并询问用户是否满意或需要调整。

### 第二步：生成角色独立文档
- **动作**：在用户确认后，在合适的目录（如 `characters/` 或用户指定目录）下创建角色的独立 Markdown 文件（如 `主角_林某某.md`）。文档内容必须包含核心三要素、角色原型、性格底色等。
- **【关键】** 在文档顶部或底部添加指向全局实体库的返回指针：
  `> [!note] 实体归属`
  `> 本角色已注册于全局实体库：[[SOLOENT.md]]`（或使用相对路径 `[SOLOENT](../../SOLOENT.md)`）
- **输出**：执行文件写入操作，并在对话框中明确回复：“✅ 已生成角色独立文档：`[文件路径]`”。

### 第三步：更新 SOLOENT.md 实体库 (指针联动)
- **动作**：检查根目录或工作区是否存在 `SOLOENT.md`（角色/实体总集注册表）。如果不存在，则主动创建它。在 `SOLOENT.md` 的“主角/核心角色”区块中，新增该角色的条目。
- **【关键】** 必须使用 Markdown 链接（指针）指向刚刚创建的独立文档，并附带一句话核心摘要。
  *示例格式：*
  `- **[林某某](characters/主角_林某某.md)**：【攀登者】核心欲望是复兴家族，面临的难点是天生绝脉。`
- **输出**：执行文件修改操作，并在对话框中向用户展示：“✅ 已在 `SOLOENT.md` 中完成指针联动，新增条目如下：`[展示新增的单行链接内容]`”。

## 主角自查清单 (Validation Checklist)

在输出角色设计和完成文档联动后，进行以下自查：

- [ ] 主角的“欲望、难点、成就”三要素是否清晰且逻辑自洽？
- [ ] 读者能否通过这个主角的视角，快速理解并信任故事世界？
- [ ] 是否已生成角色的独立文档？
- [ ] 是否已在 `SOLOENT.md` 中添加了指向该角色文档的指针（Markdown链接）？
- [ ] 角色文档中是否包含了返回 `SOLOENT.md` 的反向指针？
