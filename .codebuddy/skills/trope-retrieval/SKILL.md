# 套路推荐与检索 (Trope Retrieval)

>> SoloEnt 官方免费 Skill，欢迎使用、分享并提出优化建议。如建议被采纳，我们会在优化版本中为您署名。免费资料仅供交流学习，请勿商用，违者必究。

## 触发条件 (Trigger Scenarios)
当用户提出以下需求时，主动调用本技能：
- 明确表示“卡文了”、“不知道接下来怎么写”。
- 要求“帮我找个xx套路”、“推荐几个装逼/打脸/相遇的桥段”。
- 明确要求提供剧情灵感或特定场景的设计思路。

## 核心理念 (Core Concepts)
在辅助用户检索和使用套路时，必须遵循以下原则：
1. **套路是工具，不是目的**：套路的作用是提供经过市场验证的情绪释放机制，必须服务于角色的核心动机。
2. **拒绝生搬硬套**：绝对禁止直接复制粘贴素材库的原文。所有的套路必须经过“定制化改编”，融入用户当前的世界观和角色设定中。
3. **情绪价值优先**：推荐套路时，必须向用户解释该套路背后的“底层爽点/机制”（即满足了读者的什么心理）。

## 上下文与前置条件 (Prerequisites)
在执行此技能前，Agent 必须：
1. **理解当前上下文**: 如果工作区存在 `SOLOENT.md`，必须读取该文件以了解小说的流派、核心主题、主角人设以及当前剧情进度。
2. **识别用户需求**: 明确用户当前需要的情绪节点或剧情功能（例如：复仇、感情升温、化解危机）。

## 执行步骤与工作流 (Execution Steps)

### 第一步：检索套路素材库
- **动作**：使用 `Read` 或 `Grep` 工具，在与本 `SKILL.md` 同目录下的本地数据库文件中搜索相关关键词：
  - `docs/tropes_library.md` (核心套路素材总库)
- **检索策略**：使用与用户需求相关的宽泛关键词（如：“退婚”、“扮猪吃虎”、“英雄救美”、“拍卖会”、“绝境反杀”）。

### 第二步：筛选与匹配
- **动作**：从检索结果中筛选出 3 到 5 个最符合用户小说流派和当前上下文的套路。
- **约束条件**：**绝对禁止**选择与小说核心设定相冲突的套路（例如：不要给传统古言推荐“系统面板”套路）。

### 第三步：定制化改编与输出（核心价值）
- **动作**：将选出的套路与用户的具体故事进行深度结合与改编。
- **输出格式**：请使用以下 Markdown 格式向用户展示推荐结果：

```markdown
基于您的需求和当前作品的设定（[简述提取到的作品风格/主角人设]），我从素材库中为您匹配了以下 3 个套路方向：

**🎯 推荐套路 1：[套路名称]**
- **底层逻辑**：[一句话说明为什么这个套路管用，满足了什么读者心理]
- **在本作中的应用建议**：[结合用户角色和世界观的具体剧情推演，例如：男主可以利用XX道具，在XX场合下完成反杀...]

**🎯 推荐套路 2：[套路名称]**
- **底层逻辑**：...
- **在本作中的应用建议**：...

**🎯 推荐套路 3：[套路名称]**
- **底层逻辑**：...
- **在本作中的应用建议**：...

---
**💡 下一步**：这几个方向哪个比较符合你的预期？需要我把其中一个展开写成具体的【章节大纲】吗？
```

## 自查清单 (Validation Checklist)
在输出推荐结果前，进行以下自查：
- [ ] 是否已经读取了 `SOLOENT.md` 或向用户确认了当前的小说背景？
- [ ] 推荐的套路是否与小说的核心流派和设定相符？
- [ ] 是否提供了具体的“定制化落地建议”，而不是仅仅复述套路概念？
- [ ] 是否解释了该套路的“底层逻辑/爽点”？
