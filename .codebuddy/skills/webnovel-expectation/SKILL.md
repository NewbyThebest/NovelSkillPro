# 预期确定

## 执行流程

### 1. 聊方向

逐项跟用户聊，一次聊一个，不扔问卷：

- **平台与频道**：全流程默认番茄平台，询问用户是否修改。番茄作品明确男频/女频。
- **核心题材与主线**：一句话说清主角从哪出发、想达成什么、阻力是什么。
- **读者承诺与爽点**：读者为什么追读，回报来自什么（成长/破局/复仇/博弈等）。
- **单章字数与人称**：每章字数范围，第一/第三人称。

### 2. 确认

全部项聊完后，向用户复述结论，用户确认无误。

### 3. 写文档

按模板生成 `1-预期/1.1_预期.md`。更新已有文档时先说明改了什么，用户确认后再写入。

```markdown
# 新书预期

## 1. 定位与读者

- **目标平台**：[番茄 / 其他]
- **频道**：[已确认的频道]
- **核心题材**：[题材]
- **叙事人称**：[第一人称或第三人称]
- **单章字数**：[每章字数范围，如2000-4000字]

## 2. 主线与读者承诺

- **一句话主线**：[主角处境、目标与主要阻力]
- **核心读者承诺**：[读者持续追读的主要理由]
- **核心爽点或情绪回报**：[内容]
```

## 执行约束

- 输出文档只包含模板中的章节，不加多余板块或占位项。

**下一步建议**：完成后，执行 `webnovel-ideation` 进入设定阶段。
