# 创意与设定

## 执行流程

先读取 `1-预期/1.1_预期.md`。文件不存在时停止，说明缺少方向锚点。

## 步骤 1：新书设定案

先执行 `genre` 识别题材，读取对应参考。逐项讨论，一次聊一项，每项深入聊透再确认：

- **主角处境与目标**：主角当前的处境、目标和不能失去的对象
- **开篇承诺与读者留存**：读者最先看到什么，为什么继续看
- **力量体系（根据题材按需）**：能力来源、境界、战力构成等
- **核心矛盾与冲突**：最持续的冲突是什么
- **资源与组织规则**：物资、组织、利益分配规则
- **核心关系与势力**：势力层面的合作、竞争、对立，个人关系留到角色设定表

全部确认后按模板写入 `2-设定/2.1_新书设定案.md`：

```markdown
# 新书设定案

## 主角处境与目标

## 开篇承诺与读者留存

## 力量体系（按需）

## 核心矛盾与冲突

## 资源与组织规则

## 核心关系与势力
```

完成后进入下一步。

## 步骤 2：核心机制或金手指（按需）

先从预期和设定案判断是否有独立金手指。没有则创建 `2-设定/2.2_金手指设定.md`，注明"无独立金手指"及核心驱动。有则逐项讨论，一次聊一项，每项深入聊透再确认：

- **本质与来源**：它是什么，主角为何拥有
- **可见规则**：触发条件、作用范围、已知效果
- **限制与风险**：成本、条件、副作用、暴露风险
- **演变或揭示路线（按需）**：成长、升级或逐层揭谜

题材已有公共力量体系时，金手指只写相对公共体系的特殊部分。全部确认后按模板写入 `2-设定/2.2_金手指设定.md`：

```markdown
# 金手指或核心机制设定

## 状态

## 核心驱动

## 规则与限制（按需）

## 演变或揭示路线（按需）

## 剧情影响与不可违背的边界
```

## 步骤 3：主要角色设定

先读取 `character-vitality`，按角色层级区分核心角色和功能配角。一次聊一个角色，每个角色逐项确认后再进入下一个：

- **基本信息**：年龄、外貌
- **身世与过去**：出身、关键经历、塑造他的事件
- **可识别特征与风格**：能力、性格基调、说话风格、行为习惯
- **写作红线**：角色不能被写成什么

所有角色确认后，再讨论核心关系。功能配角只记录场景功能、当下得失和差异化反应。

写入角色条目时按字段聚合：

- 同一角色下的同名字段只能出现一次，不得通过重复字段名追加内容。
- 字段只有一项内容时，直接写在字段名后。
- 字段有两项及以上内容时，字段名单独成行，内容使用 `1. 2. 3.` 编号，每项只记录一条独立信息。

全部确认后拆分写入以下角色文件，并在主要角色表开头写明文件索引、读取时机和更新规则：

- `2-设定/2.3_主要角色设定表.md`：主角、核心配角、重要配角、核心反派和未出场的重要预备角色。
- `2-设定/2.3.1_阶段与功能角色记录.md`：当前阶段仍会出场、影响局面或需要复用的阶段反派、一次性功能角色和岗位角色；没有这类角色时可以不创建。
- `2-设定/2.3.2_核心关系.md`：只记录双方均已写入主要角色表的已确认关系基础、矛盾和发展边界；没有已确认关系时可以不创建。
- `2-设定/角色归档/[阶段名].md`：只在确有复用可能的低频角色退出当前工作集时按需创建，新书建档时不预先创建空归档。

主要角色表使用以下结构：

```markdown
# 主要角色设定表

## 角色资料索引

## 更新规则

## 主角

### [角色名]

- **基本信息**：[单项内容]
- **身世与过去**：
  1. [第一项]
  2. [第二项]
- **可识别特征与风格**：[单项内容]
- **写作红线**：
  1. [第一项]
  2. [第二项]

## 核心配角

### [角色名]

- **基本信息**：[单项内容]
- **身世与过去**：
  1. [第一项]
  2. [第二项]
- **可识别特征与风格**：[单项内容]
- **写作红线**：
  1. [第一项]
  2. [第二项]

## 核心反派

### [角色名]

- **基本信息**：[单项内容]
- **身世与过去**：
  1. [第一项]
  2. [第二项]
- **可识别特征与风格**：[单项内容]
- **写作红线**：
  1. [第一项]
  2. [第二项]

```

写入主要角色表后，运行 `python <本技能目录>/scripts/validate_character_profiles.py <工作区>/2-设定/2.3_主要角色设定表.md`。校验发现同一角色存在重复字段，或多项字段没有使用编号列表时，先修正再进入下一步。

阶段与功能角色记录是当前阶段的活跃工作表，不保存逐章历史。角色的“当前得失”或“当前场景结果”只保留最新快照，新结果覆盖旧结果；完成场景作用且没有明确复用价值的角色不建档或直接删除，不对应固定个人的岗位与群体写入新书设定案。角色开始拥有独立长期目标、反复影响主线或形成持续关系时，将条目移入主要角色表；确有复用可能但近期不再使用时移入对应阶段归档。移动后删除旧位置，不保留重复版本。

核心关系文件只记录双方均已写入主要角色表的已确认合作、依赖、竞争、对立和感情方向。写入前核对双方都有主要角色条目；阶段与功能角色、归档角色和未进入主要角色表的人物不得在此建立条目。主要角色表中的人物尚未形成核心关系时，不补空白条目，也不强行建立联系。

## 步骤 4：种子文风文档

先询问用户有没有可以参考的小说素材，直接复制粘贴到窗口发送过来，素材越多越好，最好能有 3 章以上正文。没有完整素材时，可提供一段自己认为不错的段落。都没有则从零开始讨论，给出风格选项供选择（如简洁快速型、细腻画面型、轻松吐槽型等）。

有素材时，先按素材风格预填各维度，再逐项询问用户是否可行。无素材时逐项讨论，一次聊一项，每项深入聊透再确认：

- **用词与句段节奏**：用词偏好、句子长短、段落节奏
- **场景与信息释放**：场景描写密度、信息释放节奏
- **对白与人物语言差异**：核心人物的说话风格见角色设定表，文风只记录整体对白原则
- **情绪基调与爽点表达**：整体情绪氛围、爽点的表达方式
- **本书特有的避免项**：不用的表达方式、不出现的套路

`2.4_文风.md` 是后续正文每章都会读取的高频文档，必须保持轻量：

- 每个维度只保留两至三条最稳定、最能改变成稿手感的倾向，不写成逐项检查表。
- 以正向写法为主，不放示例句，不设没有必要的句长、段长、次数或比例指标。
- 不重复项目根目录 `AGENTS.md` 已经规定的排版、去 AI 味和人物真实性硬规则。
- 有样本文本时只提炼跨多章反复出现的稳定特征，单章场景需要、偶然句式和重复性小动作不升级为文风。
- 多条表述指向同一写法时合并，宁可少而明确，不为填满维度增加通用写作常识。

全部确认后按模板写入 `2-设定/2.4_文风.md`，写入时只保留维度标题和内容，不保留模板标记和举例：

```markdown
# 文风

## 用词与句段节奏

## 场景与信息释放

## 对白与人物语言差异

## 情绪基调与爽点表达

## 本书特有的避免项
```

**下一步建议**：完成后，执行 `webnovel-outline` 写大纲。
